## 4. 快捷方式专项复查

四个快捷方式已经是圆形图标按钮，图标分别为 `search`、`auto_mode`、`swap_horiz`、`dark_mode`，按钮大小均为 `w-12 h-12`，横向间距为 `gap-[20px]`，视觉尺寸基本一致。

问题在于每个按钮下方仍有正式文字标签：`搜索`、`自动翻页`、`内容替换`、`夜间模式`。这些文字不是轻量 handoff 标注，而是按钮组内部的 UI 文本节点。结合截图，文字标签处于正文段落上方，破坏阅读主体并造成可读性干扰。

判断：不通过，P0。建议只保留 48px 或 44px 圆形图标按钮，移除正式文字标签；若需要交付说明，使用文档说明或不压正文的轻量标注。

## 5. 底部三个控制部分间距复查

- 圆形快捷方式到页内控制条距离：源码中快捷区 `bottom-[156px]`，页内控制条高度 `52px` 且 `bottom-[88px]`，两者结构间距约 16px；但快捷区包含文字标签后，整体视觉高度增加，控制区显得更松散。
- 页内控制条到底栏距离：页内控制条底部为 `88px`，底栏高度为 `68px`，两者间距约 20px，超过 8px-14px 建议范围。
- 底栏高度：`h-[68px]`，在建议区间上限内，可以接受。
- 三者整体高度：约覆盖底部 220px 以上区域，正文容器使用 `pb-64` 避让，但视觉重心仍偏高，主要由快捷标签和 20px 间距造成。
- 是否需要 token 调整：需要。当前为分散硬编码定位，未体现由底栏高度推导页内控制条、再由页内控制条推导快捷按钮的统一 token 体系。

判断：存在 P1。若移除快捷文字并改用 token 推导，底部控制区可明显收紧。

## 7. 建议布局 token

建议将当前硬编码 bottom 值收敛为以下 token：

```css
--reader-bottom-bar-height: 68px;
--reader-bottom-safe-gap: 8px;
--reader-floating-page-control-height: 52px;
--reader-floating-page-control-bottom: calc(var(--reader-bottom-bar-height) + var(--reader-bottom-safe-gap) + 10px);
--reader-quick-circle-size: 48px;
--reader-quick-actions-bottom: calc(var(--reader-floating-page-control-bottom) + var(--reader-floating-page-control-height) + 10px);
--reader-quick-circle-gap: 20px;
--reader-brightness-left: 12px;
--reader-control-layer-z-index: 50;
```

执行修正时需避免 `bottom-21` 等无效 token；快捷方式位置应由页内控制条位置推导，页内控制条位置应由底栏高度和安全区推导。
