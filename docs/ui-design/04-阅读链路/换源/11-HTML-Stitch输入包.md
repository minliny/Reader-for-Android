## 4. 页面生成约束

换源为 P1 阅读链路能力，不进入 P0 主流程；列表、检测中、检测结果必须保持同一外壳。

生成时必须保持当前页面视觉基准的结构层级和组件关系。多状态图片只用于派生状态，不得把多个状态合并成一个页面。历史、旧版、废弃、候选图片不能反向覆盖当前页面视觉基准。

响应式以当前页面视觉基准尺寸为基准。小屏优先保留顶部结构、核心内容、主要操作和安全区；短屏允许内容区滚动，不通过缩小字号破坏可读性。

## 5. 验收清单

| 项目 | 通过标准 |
| --- | --- |
| 输出状态 | 初次生成只能标记 `HTML_DRAFT` |
| 截图尺寸 | 与当前页面视觉基准尺寸一致 |
| 主模块位置 | 顶部、核心内容、主要操作目标误差 `±2px` |
| 视觉 token | 圆角、边框、字号、行高目标误差 `±1px` |
| 状态一致性 | 状态参考只替换对应状态区域，不重画页面外壳 |
| 人工复核 | 截图叠放后记录差异，未复核前不得升级为 `STITCH_READY` |
