# 换源

## 页面内容
### 页面目标

在阅读中检查和切换可用来源，处理来源失效与检测结果。

### 页面区域

1. 顶部标题或控制区。
2. 主内容区，承载当前任务的核心列表、表单或阅读覆盖层。
3. 操作反馈区，展示 loading、empty、error、offline 和 permission。
4. 返回区，必须保留来源页上下文。

### 组件清单

来源列表、当前来源标记、检测状态、检测结果、切换按钮、错误说明。

### 核心文案

- `换源`
- `当前来源`
- `检测中`
- `可用`
- `失效`
- `切换来源`
- `重试`

### 通用反馈

- `正在加载`
- `加载失败，请重试`
- `网络不可用，请稍后重试`
- `取消`
- `确认`

## 交互逻辑
### 主要操作

检测来源、选择来源、确认切换、查看失败原因、返回阅读页。

### 返回与反馈

- 系统返回回到来源页。
- 弹层优先关闭当前层。
- 操作失败保留用户上下文并提供重试。
- 提交中禁止重复点击。

### default

展示完整可操作内容。

### loading

使用骨架或局部加载，不清空已存在内容。

### empty

说明空原因，并给出下一步。

### error

展示失败原因和重试入口。

### offline

保留本地可用内容，阻断联网动作。

### permission

需要权限时说明用途并提供授权入口。

- 不新增未声明入口。
- 不引入账号、社区、广告、会员或推荐流。
- 不压缩触控区来适配小屏。
- 不在提交中允许重复点击。
