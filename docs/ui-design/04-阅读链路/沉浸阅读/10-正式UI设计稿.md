# 沉浸阅读正式 UI 设计稿

状态：正式稿
等级：P0 / READING_TEXT_MASTER
适用页面：阅读正文默认沉浸态
视觉参考图片：`图片/01-沉浸阅读-默认态.png`
夜间参考图片：`图片/02-夜间模式状态.png`

## 1. 设计目标

沉浸阅读是阅读链路的基础层，目标只有一个：让用户长时间稳定阅读正文。

本页不是 App Shell 页面，不显示主 Tab 顶栏、底部导航、书架卡片、书源管理、推荐流或广告。

## 3. 页面分层

从低到高：

1. 阅读背景层。
2. 正文排版层。
3. 阅读信息层。
4. 透明点击热区层。

四角弱信息不属于正文母版，只属于信息层。点击热区不属于可见 UI。

## 5. 正文排版规则

- 章节标题在正文排版层内展示。
- 正文段落使用稳定行高和段距。
- 正文不进入卡片容器。
- 正文不因控制层出现而重排。
- 长章节、网络刷新、亮度变化不改变当前分页位置，除非用户主动修改字体、字号、行距或版式。

## 6. 信息层规则

默认信息层包含：

- 左上：书名与章节。
- 右上：时间。
- 左下：阅读百分比。
- 右下：当前章节信息，只显示当前章节名或当前章节序号；不得显示总章节数、`第 N / M 章` 或全书目录总量。

信息层是弱提示，控制层显示时隐藏。它不得挤压正文，也不得被 Stitch 当成顶部栏或底部导航。

## 7. 交互定义

| 区域 | 行为 |
| --- | --- |
| 左侧透明热区 | 翻上一页 |
| 中间透明热区 | 呼出或隐藏阅读控制层 |
| 右侧透明热区 | 翻下一页 |
| 系统返回 | 回到进入阅读页的来源页 |
| 长按正文 | 后续文本选择/批注入口，优先级高于翻页热区 |

透明热区不显示可见按钮。

## 8. 状态设计

| 状态 | 设计要求 |
| --- | --- |
| 默认态 | 使用 `图片/01-沉浸阅读-默认态.png` 的阅读密度，以视觉 token 校准 |
| 夜间态 | 使用夜间背景和正文颜色，不显示控制层 |
| loading | 保留阅读背景，可显示轻量加载，不出现主 Tab 外壳 |
| error | 保留返回能力和重试入口，不清空阅读上下文 |
| offline | 已缓存章节继续阅读，联网动作提示网络不可用 |

## 10. 明确排除

- 不显示 App 主底部导航。
- 不显示控制按钮、底表、卡片或广告。
- 不显示书源列表或换源流程。
- 不把四角弱信息归入正文母版。
- 不在右下弱信息显示总章节数；右下只允许当前章节信息。
- 不让控制层覆盖导致正文重新分页。

## 11. 验收标准

- 阅读正文可长期阅读，背景和文字对比克制。
- 四角信息弱化且不挤压正文。
- 控制层隐藏时页面不出现任何显性按钮。
- 日间和夜间属于同一阅读排版系统。
