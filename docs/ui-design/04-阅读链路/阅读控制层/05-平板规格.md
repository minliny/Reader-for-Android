# 平板阅读控制层设计规格

本文是侧置悬浮阅读控制层在 Android 大屏与横向窗口中的专项规格。布局模式选择与
尺寸分级以 `READER_CONTROL_RESPONSIVE_RULES.md` 为准。本文继承手机端阅读控制层的
功能结构、动作语义和视觉语言，但不修改或覆盖手机端结构规格。具体坐标、尺寸、圆角、
字体、图标和颜色以 `READER_CONTROL_GEOMETRY_SPEC.md` 为唯一参数源。

## 1. 设计目标

平板阅读控制层是在手机端已冻结体系上的设备适配，不是重新设计一套控制层，也不是
把手机控制层等比例放大。

设计目标：

- 直接复用手机端控制面板的功能结构、控件顺序和视觉语言；
- 保持接近手机端的物理尺寸和操作密度；
- 将侧置模式的顶部控制条适配为全屏通栏；
- 让控制面板以侧置悬浮或停靠方式出现，保留主要正文阅读空间；
- 横屏和竖屏使用同一套控制面板内容；
- 控制层出现、切换和关闭时不重排正文；
- 完整控制面板与自动翻页、朗读运行态胶囊互斥。

## 2. 与手机控制层的关系

手机端权威来源：

- `READER_CONTROL_LAYER_SPEC.md`：结构与层级；
- `READER_CONTROL_ACTION_FLOWS.md`：动作、覆盖和返回规则；
- `READER_CONTROL_RESPONSIVE_RULES.md`：手机竖屏适用范围；
- `READER_CONTROL_IMAGE_USAGE.md`：图片角色和使用边界。

平板端继承：

- 搜索、自动翻页、替换三个阅读工具；
- 当前章节、上一章、下一章和本章进度；
- 目录、朗读、界面、设置四个模块入口；
- 独立右侧竖向亮度栏；
- 四个模块只替换控制面板左侧主内容区的规则；
- 专项浮层、完整功能页和运行态胶囊的动作语义；
- 手机端的色彩、图标、字体层级、圆角、描边和阴影语言。

平板端改变：

- 手机顶部胶囊改为平板顶部通栏；
- 手机底部悬浮面板改为平板侧置悬浮或停靠面板；
- 只进行必要的尺寸和位置适配，不改变功能分类。

本文件不得反向修改手机端固定结构、`390×884` 基准或手机端图片优先级。

## 3. 适用范围

本规格覆盖 `READER_CONTROL_RESPONSIVE_RULES.md` 定义的侧置悬浮模式：

- 竖向窗口且 `W >= 600dp`：固定在右下侧；
- 横向窗口：固定在左下侧；
- `S >= 600dp` 使用标准密度；
- 横向窗口且 `S < 600dp` 使用紧凑密度。

设备名称不参与模式判断。折叠屏和自由窗口按当前可用窗口尺寸判断，但分离式铰链、
可用面板高度低于 `280dp` 的超矮窗口，以及桌面专用交互不在本规格内。

## 4. 平板沉浸式阅读页

沉浸页只包含：

- 正文阅读区；
- 四角弱信息；
- 透明点击热区。

四角信息固定贴合屏幕安全区和屏幕边缘，不跟随正文列宽：

| 位置 | 内容 |
| --- | --- |
| 左上 | 书名，或书名与当前章 |
| 右上 | 时间 |
| 左下 | 阅读百分比 |
| 右下 | 当前章节信息；只显示当前章节名或当前章节序号，不显示总章节数 |

透明点击热区不是显性 UI：

| 区域 | 默认占比 | 动作 |
| --- | --- | --- |
| 左侧 | 约 30% | 上一页 |
| 中间 | 约 40% | 显示或隐藏控制层 |
| 右侧 | 约 30% | 下一页 |

平板可根据握持模式调整热区边界，但默认动作语义与手机一致。

### 4.1 竖屏正文

- 单栏居中；
- 正文列宽固定参考值为 `560dp`；
- 在 `800×1280dp` 画布中，正文列固定为
  `left 120dp / top 112dp / width 560dp / bottom 1216dp`；
- 正文字号 `22sp`，行高 `41dp`，首行缩进 `44dp`，段后间距 `20dp`；
- 章节标题位于 `left 120dp / top 120dp`，正文首行从 `top 188dp` 开始；
- 控制面板出现时正文不重排；
- 控制面板出现时正文容器、坐标、列宽和分页结果完全不变；
- 不允许为了避让控制面板动态左移正文。
- 正文先按完整 `560dp` 居中列排版到页面底部，控制面板最后叠加并遮挡其覆盖范围；
- 不得在右侧控制面板周围预留空白列或缩窄正文。

### 4.2 横屏正文

- 在 `1280×800dp` 画布中，正文组固定为
  `left 148dp / top 108dp / width 984dp / bottom 752dp`；
- 左栏 `left 148dp / width 460dp`，右栏 `left 672dp / width 460dp`，
  栏间距 `64dp`；
- 也可由阅读版式设置切换为单栏或双页；
- 具体模式由阅读版式设置决定；
- 正文字号、行高和字重与竖屏完全一致；
- 默认双栏是一个连续文本流：章节标题只在左栏出现一次，左栏写满后续到右栏；
- 不得复制章节标题、重复正文或把两栏当成两个独立页面；
- 控制面板位于左侧时只覆盖既有正文或留白；
- 控制面板出现不得触发正文移动、重新分页或生成新的内容布局。
- 左栏文字必须连续排版到页面底部；不得因为左下控制面板而提前结束或留下空白块。

## 5. 顶部通栏

平板控制态顶部栏必须全屏通栏，不使用手机端居中胶囊。

结构：

```text
返回 / 书名 · 当前章 / 来源 / 换源 / 更多
```

| 属性 | 规则 |
| --- | --- |
| 宽度 | 屏幕可用宽度 100% |
| 高度 | `60dp` |
| 左右内边距 | `32dp` |
| 背景 | 米白、暖白或半透明纸张色 |
| 底部分割线 | `1dp` 极浅暖灰 |
| 阴影 | 极轻、低对比暖灰 |
| 返回 | 最左侧 |
| 标题 | 左对齐；书名为主标题，来源与当前章为副标题 |
| 换源、更多 | 右侧 |

顶部栏是阅读控制态的上层 UI，不改变正文内容。顶部返回按钮退出阅读页，不只是关闭
控制层。

## 6. 底部显示规则

阅读页底部不显示通栏进度、细进度线或额外状态条。沉浸态的阅读百分比和章节序号使用
四角弱信息；控制态的本章进度只显示在控制面板内部。

## 7. 控制面板本体

平板控制面板直接复用手机端 `04` 的默认功能结构和 `04-2` 的视觉语言，不新增分类。

固定四区：

| 区域 | 内容 |
| --- | --- |
| 阅读工具区 | 搜索、自动翻页、替换 |
| 章节定位区 | 当前章节、上一章、下一章、本章进度条、本章百分比 |
| 阅读能力区 | 目录、朗读、界面、设置 |
| 亮度调节区 | 亮度标题、太阳图标、竖向滑杆、A、系统/手动状态 |

结构关系：

```text
┌──────────────────────┬──────┐
│ 搜索   自动翻页   替换 │ 亮度 │
│                      │  ☀   │
│ 第 32 章 雨夜         │  │   │
│ 上一章       下一章    │  │   │
│ ━━━━━●──── 本章 38%   │  │   │
│                      │  A   │
│ 目录   朗读   界面   设置│ 系统 │
└──────────────────────┴──────┘
```

亮度保持为独立竖向调节区，不能改成普通入口按钮或水平滑杆。

## 8. 控制面板尺寸

标准密度：

```text
controlPanelWidth = 390dp
controlPanelHeight = 343dp
```

横向紧凑密度：

```text
controlPanelWidth = min(360dp, W - 32dp)
controlPanelHeight = min(317dp, H - safeTop - safeBottom - 24dp)
```

解释：

- 标准密度固定为 `390×343dp`；
- 紧凑密度只用于短边不足 `600dp` 的横向窗口；
- 横向窗口不随长边拉伸；
- 竖向宽屏不随屏幕高度持续增高；
- 极大平板也不放大为大型控制台；
- 控制面板不得铺满屏幕宽度。

尺寸公式表达的是平板控制面板的物理尺寸约束，不允许用导出图片像素直接反推 dp。

## 9. 控件分区和视觉

### 9.1 区域比例

百分比均相对于控制面板内部可用尺寸，用于约束密度，不用于改变四区关系。

| 区域 | 比例 | 布局 |
| --- | --- | --- |
| 阅读工具区 | 左侧内容区高度约 24% | 三等分横排，图标在上、文字在下 |
| 章节定位区 | 左侧内容区高度约 36% | 标题、章节切换、本章进度 |
| 阅读能力区 | 左侧内容区高度约 26% | 始终一行四等分 |
| 亮度调节区 | 面板宽度约 16%～18% | 位于右侧并贯穿主要高度 |

剩余高度用于区域间距和面板内边距。

具体规则：

- 搜索、自动翻页、替换的文字不得换成多行；
- 章节标题单行省略；
- 上一章、下一章保持同组对称关系；
- 本章进度条可拖动并显示百分比；
- 目录、朗读、界面、设置名称和顺序固定；
- 模块选中项使用浅蓝底与蓝色文字或图标；
- 阅读能力区必须保持手机端一行四项，不允许改成 `2×2`。

### 9.2 视觉语言

- 面板背景：米白、暖白或纸张色；
- 圆角：`18dp`；
- 阴影：柔和、低对比、暖灰；
- 描边：`1dp` 极浅暖灰；
- 内部分区：轻描边、轻圆角；
- 操作按钮：浅色胶囊或浅底圆角块；
- 选中态：浅蓝底配蓝色文字或图标；
- 主强调色：继承手机端蓝色；
- 主文字：接近黑色但不使用纯黑；
- 辅助文字：暖灰；
- 图标：继承手机端线性图标语义、描边重量和尺寸层级。

## 10. 竖屏布局

控制面板固定在屏幕右下侧，不铺满底部、不居中放大，也不自由悬浮。

```text
panelRight = safeRight + 24dp
panelBottom = safeBottom + 24dp
panelTop = panelBottom - controlPanelHeight
```

定位时应同时满足：

- 控制面板完整位于安全区内；
- 面板底部距离安全区固定为 `24dp`；
- 面板不横跨屏幕底部；
- 顶部不遮挡顶部通栏；
- 正文保持稳定，不因面板出现而重新排版。

左手模式镜像到左侧：

```text
panelLeft = safeLeft + 24dp
```

镜像只改变面板停靠侧，不改变内部四区、按钮顺序、亮度栏内部结构或动作语义。

## 11. 横屏布局

控制面板固定在屏幕左下侧：

```text
panelLeft = safeLeft + 24dp
panelBottom = safeBottom + 24dp
panelTop = panelBottom - controlPanelHeight
```

横屏规则：

- 控制面板尺寸与竖屏一致；
- 不因长边增加而拉伸；
- 不改成超宽左右分栏面板；
- 不改成只有图标的左侧竖栏；
- 不删除功能，不改变按钮顺序；
- 正文可显示单栏、双栏或双页，仍由阅读版式设置控制；
- 面板作为临时浮层覆盖侧边区域，不为面板重新分配正文空间，也不触发正文重新分页。

## 12. 动作流

| 入口或动作 | 平板行为 |
| --- | --- |
| 点击沉浸页中间区域 | 显示或隐藏顶部通栏和控制面板 |
| 点击控制层外正文空白 | 关闭控制层，回到沉浸页 |
| 系统返回 | 按当前最上层状态逐级关闭，最终关闭控制层 |
| 顶部返回 | 退出阅读页 |
| 搜索 | 打开当前书内容搜索浮层 |
| 自动翻页 | 打开自动翻页快捷配置浮层 |
| 替换 | 打开内容替换快捷浮层，只影响显示，不修改原文 |
| 上一章 / 下一章 | 直接切章，控制层保持显示 |
| 拖动本章进度 | 显示临时进度提示，松手跳转 |
| 目录 | 切换到目录/书签快捷模块 |
| 朗读 | 切换到朗读快捷模块 |
| 界面 | 切换到界面快捷模块 |
| 设置 | 切换到设置快捷模块 |
| 亮度滑杆 | 即时调整亮度 |
| A / 系统 | 切换跟随系统或手动亮度 |
| 换源 | 打开换源底表或平板适配浮层 |
| 更多 | 打开顶部锚定菜单 |
| 开始自动翻页或朗读 | 关闭完整控制层，进入对应运行态胶囊 |

目录、朗读、界面、设置仍是模块切换入口。模块态只替换控制面板左侧主内容区，
控制面板外壳、右侧亮度栏和模块导航保持稳定。

控制态四角信息和顶部通栏的显示互斥关系以
`READER_CONTROL_RESPONSIVE_RULES.md` 的信息状态矩阵为准。

## 13. 专项浮层关系

第一版优先复用手机端专项浮层的内容和动作语义，只调整尺寸、锚点和最大宽高。

### 13.1 竖屏

- 可从控制面板附近弹出或在屏幕安全区内居中浮出；
- 不铺满全屏；
- 不改变正文排版；
- 不强制采用“控制面板 + 详情区”结构；
- 专项浮层打开时，完整控制面板收起或被替换，避免层级冲突。

### 13.2 横屏

- 可从左侧控制面板右侧展开窄浮层；
- 也可在安全区内居中浮出；
- 不覆盖整个正文区域；
- 不把控制面板扩展成大型详情面板；
- 浮层关闭后返回触发前的控制面板状态或沉浸页。

### 13.3 完整功能页

内容搜索完整页、自动翻页完整设置、替换规则管理、朗读设置、字体选择、主题选择、
版式高级等仍是完整页面。本规格只定义其入口和与控制层的返回关系，不定义这些页面的
平板最终布局。

## 14. 禁止事项

1. 不允许把平板控制层做成超宽左右分栏大控制台。
2. 不允许使用“总览 / 导航 / 查找 / 运行 / 外观 / 内容 / 设置”等后台式抽象分类。
3. 不允许把手机端按钮无分组平铺成按钮堆。
4. 不允许把横屏控制层改成只有左侧竖向图标栏。
5. 不允许让竖屏控制层铺满底部或按平板宽度等比例放大。
6. 不允许顶部栏使用手机端居中胶囊；平板顶部栏必须通栏。
7. 不允许在阅读页底部添加通栏进度、细进度线或额外状态条。
8. 不允许把 `34-reader-appearance-main-panel-standard.png` 或 `28–33` 作为最终平板视觉、尺寸或布局依据。
9. 不允许使用生成图、低保真草图、线框图或失败 demo 反推正式规格。
10. 不允许在平板第一版强行加入“控制面板 + 右侧详情区”的大型结构。
11. 不允许把亮度变成普通入口按钮或水平滑杆。
12. 不允许把自动翻页和朗读合并为“运行”。
13. 不允许把替换合并为“内容”，或把目录合并为“导航”。
14. 不允许横屏删减功能、改变按钮顺序或重分类。
15. 不允许平板规格反向覆盖手机端冻结规格。
16. 不允许控制面板出现时动态移动、缩窄或重新分页正文。
17. 不允许侧置模式保留手机底部 sheet 的拖动柄。
