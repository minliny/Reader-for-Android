# 阅读控制层响应式规格

本文定义阅读控制层在不同窗口比例下的布局模式。功能结构、模块关系和动作语义仍以
`READER_CONTROL_LAYER_SPEC.md` 与 `READER_CONTROL_ACTION_FLOWS.md` 为准。正文、
控制层、模块态、浮层和胶囊的具体坐标与视觉参数统一以
`READER_CONTROL_GEOMETRY_SPEC.md` 为准。

## 1. 统一变量

```text
W = 当前窗口可用逻辑宽度
H = 当前窗口可用逻辑高度
R = H / W
S = min(W, H)
```

手机竖屏设计工具基准仍为：

```text
W = 390dp
H = 884dp
R = 2.267
```

现有高分辨率 PNG 按相对比例映射，不直接使用导出像素反推全部 dp。

## 2. 两种布局模式

阅读控制层只定义两种外部布局：

| 条件 | 模式 | 默认位置 |
| --- | --- | --- |
| `H >= W` 且 `W < 600dp` | 底部悬浮模式 | 屏幕底部居中 |
| `H >= W` 且 `W >= 600dp` | 侧置悬浮模式 | 屏幕右下侧 |
| `H < W` | 侧置悬浮模式 | 屏幕左下侧 |

设备名称不参与判断。手机、平板、折叠屏和自由窗口均以当前窗口的 `W`、`H` 为准。

两种模式只改变：

- 控制面板外部尺寸；
- 控制面板在窗口中的锚点；
- 顶部控制条采用手机悬浮条或宽屏通栏。

两种模式均不得改变：

- 搜索、自动翻页、替换的顺序；
- 当前章节、上一章、下一章和本章进度；
- 目录、朗读、界面、设置的一行四项结构；
- 右侧竖向亮度栏；
- 四个模块只替换左侧主内容区的关系；
- 正文容器、正文列宽、分页结果和滚动位置。

## 3. 底部悬浮模式

适用条件：

```text
H >= W
W < 600dp
```

规则：

- 继承 `04-2-reader-controls-opaque-icon-aligned.png` 的底部悬浮结构；
- 控制面板水平居中；
- 面板底部与安全区保持 `8dp～16dp` 间距；
- 面板不贴屏幕边缘，不出现额外屏幕底部进度线；
- 顶部控制条继续采用手机端悬浮条；
- 控制面板出现时正文不移动、不缩窄、不重新分页。

尺寸：

```text
panelWidth = min(W - 10dp, 520dp)
panelHeight = clamp(317dp, panelWidth × 0.88, 380dp)
panelLeft = (W - panelWidth) / 2
panelBottom = safeBottom + 8dp～16dp
```

当窗口不足以容纳最小高度时，只压缩模块内部垂直间距或启用左侧模块内部滚动，不改变
固定四区顺序，也不改为按钮墙。

## 4. 侧置悬浮模式

适用条件：

```text
H < W
或
H >= W 且 W >= 600dp
```

侧置面板是纯覆盖层。“停靠”只表示视觉锚点，不表示参与正文布局。

### 4.1 标准密度

满足以下条件时使用：

```text
S >= 600dp
```

```text
panelWidth = 390dp
panelHeight = 343dp
```

### 4.2 紧凑密度

横向窗口且 `S < 600dp` 时使用：

```text
panelWidth = min(360dp, W - 32dp)
panelHeight = min(317dp, H - safeTop - safeBottom - 24dp)
```

紧凑密度只允许压缩间距、辅助说明和模块内部可见行数。不得删除功能、改成竖向图标栏、
改变模块顺序或把模块导航改成 `2×2`。

若可用面板高度低于 `280dp`，当前规格不再保证完整控制面板可用，应进入后续超矮窗口
专项适配；不得擅自切换成第三种结构。

## 5. 侧置位置

### 5.1 竖向宽屏

```text
panelRight = safeRight + 24dp
panelBottom = safeBottom + 24dp
panelTop = panelBottom - panelHeight
```

- 默认固定在右下侧；
- 左手模式镜像到左侧，`panelLeft = safeLeft + 24dp`；
- 面板不居中放大，不横跨屏幕底部；
- 面板不支持自由悬浮或上下拖动。

### 5.2 横向窗口

```text
panelLeft = safeLeft + 24dp
panelBottom = safeBottom + 24dp
panelTop = panelBottom - panelHeight
```

- 面板默认固定在左下侧；
- 不随窗口长边拉伸；
- 不为面板重新分配正文列；
- 不支持自由悬浮或上下拖动；
- 正文可在面板下方被视觉覆盖，但其布局和分页始终保持原状态。

## 6. 正文不变量

控制层显示、隐藏、切换模块或改变停靠侧时：

- 正文容器坐标不变；
- 正文宽度和列数不变；
- 行高、段落位置和分页结果不变；
- 滚动位置不变；
- 不允许为控制面板动态左移、右移或缩窄正文；
- 双栏或双页模式仍由阅读版式设置决定。
- 正文必须先按沉浸态完整排版到页面底部，再叠加控制层；
- 控制面板覆盖范围内的正文仍然存在，只是被不透明面板遮挡；
- 不得在控制面板周围生成空白避让区、截断正文列或提前结束段落；
- 验收图中应能从面板边缘看到被遮挡文字的连续上下文，证明面板未参与正文布局。

## 7. 顶部信息

### 7.1 顶部控制条

| 模式 | 顶部控制条 |
| --- | --- |
| 底部悬浮模式 | 继承手机 `04-2` 悬浮控制条 |
| 侧置悬浮模式 | 使用全屏通栏，高度固定 `60dp` |

侧置通栏结构固定为：

```text
返回 / 书名（主标题）+ 来源 · 当前章（副标题） / 换源 / 更多
```

标题区左对齐，不再保留“居中或偏左均可”的不确定规则。

阅读页不显示任何贴屏幕底部的通栏进度、细进度线或状态条。本章进度只存在于控制面板
内部；沉浸态阅读百分比与章节序号只通过四角弱信息表达。

## 8. 信息状态矩阵

| 状态 | 四角弱信息 | 顶部控制条 |
| --- | --- | --- |
| 沉浸态 | 显示 | 隐藏 |
| 默认控制态 | 隐藏 | 显示 |
| 目录/朗读/界面/设置模块态 | 隐藏 | 显示 |
| 专项浮层 | 隐藏 | 由浮层规则接管 |
| 自动翻页/朗读运行胶囊态 | 显示 | 隐藏 |

同一信息不得同时在四角信息和顶部控制条中重复显示。

## 9. 画布、密度与正文基准

侧置模式的横竖屏设计稿必须使用同一组旋转画布和同一导出倍率：

```text
竖向逻辑画布 = 800 × 1280dp
横向逻辑画布 = 1280 × 800dp
建议导出倍率 = 2x
竖向导出尺寸 = 1600 × 2560px
横向导出尺寸 = 2560 × 1600px
```

若生成工具无法输出精确像素，仍必须保证：

- 横竖图互为旋转比例；
- 同一个 `390×343dp` 控制面板在两张图中的像素尺寸完全一致；
- 顶部栏、图标、字号、描边和圆角使用同一 dp→px 比例；
- 输出必须包含完整画布四边，不得裁切顶部栏、正文、控制面板或底部页面区域。

正文使用统一排版基准：

```text
正文字号 = 22sp
正文行高 = 41dp
首行缩进 = 44dp
段后间距 = 20dp
字重 = 400
字间距 = 0
```

横竖屏只改变正文列数，不改变正文字号、行高、字重、首行缩进或段间距。

### 9.1 竖向正文固定坐标

基于 `800×1280dp` 完整画布：

```text
正文列：
  left = 120dp
  top = 112dp
  width = 560dp
  bottom = 1216dp

章节标题：
  left = 120dp
  top = 120dp
  fontSize = 28sp
  lineHeight = 42dp

正文首行：
  top = 188dp
```

正文列水平居中，占画布宽度 `70%`。正文必须持续排版到 `bottom = 1216dp`，不得因为
右下控制面板而缩窄、左移、提前换页或在右侧形成空白栏。

### 9.2 横向正文固定坐标

基于 `1280×800dp` 完整画布：

```text
正文组：
  left = 148dp
  top = 108dp
  width = 984dp
  bottom = 752dp

左栏：
  left = 148dp
  width = 460dp

栏间距：
  width = 64dp

右栏：
  left = 672dp
  width = 460dp
```

横屏使用一个连续的双栏文本流：

1. 章节标题只出现在左栏顶部一次；
2. 左栏正文从 `top = 180dp` 开始；
3. 左栏写满至 `bottom = 752dp` 后，正文继续流入右栏；
4. 右栏从 `top = 108dp` 开始承接后续正文；
5. 不得在右栏重复章节标题；
6. 不得把左右栏生成为两份重复正文或两个独立页面。

### 9.3 正文层与控制层的合成顺序

必须按以下顺序生成或实现：

1. 绘制完整纸张背景；
2. 按上述固定坐标完成整页正文排版；
3. 绘制控制态轻遮罩；
4. 隐藏四角弱信息并绘制顶部控制条；
5. 最后绘制不透明控制面板。

禁止先放置控制面板，再让正文布局系统围绕面板排版。

竖向控制面板矩形：

```text
left = 386dp
top = 913dp
width = 390dp
height = 343dp
```

横向控制面板矩形：

```text
left = 24dp
top = 433dp
width = 390dp
height = 343dp
```

这些矩形只用于最终覆盖。它们不得成为正文的 margin、padding、避让区、障碍区域或
分页边界。

### 9.4 正文层验收

隐藏控制面板和遮罩后：

- 竖屏必须恢复一列完整、居中、连续到页面底部的正文；
- 横屏左右两栏必须连续且左栏下半部有正常正文；
- 控制面板原覆盖区域内不得出现空白矩形；
- 正文行不能在控制面板边缘主动换行；
- 面板边缘外露的文字必须能与面板另一侧被遮挡的上下文连续对应。

## 10. 遮罩与关闭

- 控制态使用全屏暖黑 `6%～10%` 的轻遮罩；
- 遮罩只弱化正文，不改变正文布局；
- 顶部控制条、控制面板和辅助提示位于遮罩上方；
- 点击控制层外正文区域关闭最上层控制 UI；
- 点击不能穿透并触发正文翻页。

## 11. 控制面板固定规则

- 底部与侧置模式复用同一功能组件；
- 目录、朗读、界面、设置始终一行四项；
- 亮度栏始终位于控制面板右侧并使用竖向滑杆；
- 侧置模式取消手机底部 sheet 的拖动柄；
- 侧置面板第一版固定在规定的侧边底部锚点，不支持自由拖动；
- 竖向宽屏可通过左手模式镜像到左下侧；
- 模块内容超出时只允许左侧主内容区内部滚动；
- 运行态胶囊与完整控制面板互斥。

## 12. 退出条件

当前规格不覆盖：

- 可用面板高度低于 `280dp` 的超矮窗口；
- 需要跨越或避让分离式铰链的特殊折叠屏；
- 桌面端多窗口专用交互。

这些场景进入独立适配，不得从当前两种模式自动推导第三种控制层结构。
