## 3. Stitch 输入顺序

每个页面生成时按以下顺序输入：

1. `01-主标签页范围与剔除清单.md`
2. `../01-全局规范/02-应用外壳组件视觉规格.md`
3. 目标页面包全部 Markdown
4. 目标页面图片示例
5. 若目标为书架首页，同时读取 `02-主标签页/书架/08-书架首页组件视觉规格.md`

## 4. 不合格判定

以下任一情况出现时，页面只能算候选，不能冻结：

- 底部导航出现 `我的` 或 `书源`。
- 主标签页 根页面中直接展开次级链路完整页面。
- 发现/RSS 被补成完整功能闭环，超出当前暂缓范围。
- 设置首页出现账户头像、昵称、登录卡片。
- 卡片、chip、底部导航、顶部栏 token 与 `02-应用外壳` 不一致。
- 图片与文字稿冲突但未按文字稿修正。
