# 书架首页正式 UI 设计稿

状态：正式稿
等级：P0 / PAGE_VISUAL_BASELINE
适用页面：底部主导航 `书架` 根页面
视觉基准图片：`图片/01-书架首页-封面模式.png`
组件对照图片：`图片/02-书架首页-列表模式.png`

## 1. 设计目标

书架首页是 App 启动后的核心阅读入口，优先服务三个任务：

- 快速回到最近阅读位置。
- 清晰浏览和管理书架内书籍。
- 在封面模式和列表模式之间稳定切换，不产生两套视觉系统。

本页不是书源管理页、账户页、营销页或配置页。配置型修改进入书籍详情页完成。

## 3. 页面信息架构

页面从上到下固定为：

1. 系统状态栏安全区。
2. 顶部栏：页面标题 `书架`，右侧搜索图标和更多图标。
3. 分组 chip 横向滚动区：`全部 / 在读 / 已完结 / 本地 / 自定义分组`。
4. 顶部信息区：`继续阅读` 和 `最近更新` 两个同级卡片。
5. 书架标题行：左侧 `我的书架`，右侧排序/筛选与封面/列表模式切换。
6. 书籍内容区：封面网格或列表行。
7. 底部主导航：`书架 / 发现 / RSS / 设置`，当前高亮 `书架`。

底部主导航固定在屏幕底部；其余内容在同一个纵向滚动容器内。内容底部必须预留底部导航高度和系统手势安全区。

## 4. 页面组件树

```text
ShelfRoot

    IconButton(More)
  ShelfGroupChipRail
    ShelfGroupChip(active="全部")
    ShelfGroupChip("在读")
    ShelfGroupChip("已完结")
    ShelfGroupChip("本地")
    ShelfGroupChip(custom)
  ShelfHeroPanels
    ContinueReadingCard
    RecentUpdateCard
  ShelfSectionHeader
    Title("我的书架")
    SortFilterButton
    ViewModeToggle
  ShelfContent
    BookGrid / BookList
  BottomNav(active="书架")
```

组件命名和 token 命名以 `08-书架首页组件视觉规格.md` 为准，不允许在实现中为封面模式和列表模式各自创建一套颜色、圆角、边框或阴影。

## 5. 封面模式正式设计

封面模式是本页的视觉基准。

- 书籍以网格展示，卡片宽度由可用空间和 `Shelf.BookCard.MinWidth / MaxWidth` 决定。
- 书籍封面固定 `2:3` 比例，圆角和占位样式复用 `Shelf.Cover.*`。
- 书名最多两行；作者、章节、更新状态最多一行。
- 阅读进度条放在卡片内容区底部，宽度跟随卡片内容宽度。
- 卡片之间保持稳定横向和纵向间距，切换分组或刷新数据时不得出现布局跳动。

封面模式首屏优先露出至少一行书籍卡片。短屏设备上允许压缩顶部信息卡高度，但不允许隐藏顶部栏、分组 chip 或底部导航。

## 6. 列表模式正式设计

列表模式是同一页面的密集阅读状态，不是另一套视觉设计。

- 书籍以纵向列表展示，每行一个 `BookListItem`。
- 列表行复用 `Shelf.Card.*` 的背景、边框、圆角和阴影。
- 列表封面仍为 `2:3`，使用 `Shelf.Cover.ListWidth / ListHeight`。
- 书名、作者、章节和进度信息的层级必须与封面模式一致。
- 右侧更多按钮只打开书籍操作底表。

封面模式和列表模式切换时，顶部栏、分组 chip、顶部信息卡、标题行、底部导航不改变位置和视觉 token。

## 7. 自适应规则

| 屏幕条件 | 展示规则 |
| --- | --- |
| 宽度 `<360dp` | 页面左右边距使用 `Shelf.Page.CompactHorizontalPadding`；封面网格固定 2 列；顶部信息卡纵向堆叠 |
| 宽度 `360-430dp` | 封面网格按最小卡片宽自动 2 或 3 列；顶部信息卡优先按图片母版展示 |
| 宽度 `431-600dp` | 网格最多 3 列；卡片可变宽但不得超过 `Shelf.BookCard.MaxWidth` |
| 宽度 `>600dp` | 内容最大宽 `720dp` 并居中；网格最多 4 列 |
| 高度 `<760dp` | 压缩顶部信息卡高度，优先保证书籍内容首行可见 |
| 横屏 | 保留顶部栏和底部导航；网格最多 4 列，内容区居中 |

自适应优先级：

1. 调整列数。
2. 调整顶部信息卡排布。
3. 增加滚动。
4. 保持字号不随屏幕宽度缩放。

不得通过缩小字号、压扁封面、隐藏底部导航来适配小屏。

## 8. 交互定义

| 入口 | 行为 |
| --- | --- |
| 搜索图标 | 进入 `../../03-书架链路/书籍搜索/00-说明.md` |
| 更多图标 | 打开书架级更多菜单；P0 只保留入口，不在本稿扩展菜单内容 |
| 分组 chip | 当前页内切换分组，保持当前封面/列表模式 |
| 继续阅读卡 | 进入最近阅读书籍的上次阅读位置 |
| 最近更新卡 | 进入对应书籍详情或最新章节入口 |
| 排序/筛选 | 进入 `../../03-书架链路/排序与筛选/00-说明.md` |
| 封面/列表切换 | 只替换书籍内容区布局 |
| 书籍卡/行 | 进入 `../../03-书架链路/书籍详情/00-说明.md` |
| 书籍更多按钮 | 打开 `../../03-书架链路/书籍操作底表/00-说明.md` |
| 底部主导航 | 切换到对应主标签页根页面 |

书籍操作底表只允许 `修改` 和 `删除`：

- `修改`：进入书籍详情页，在详情页修改书籍配置。
- `删除`：从书架移除该书。

不得在底表放置换源、缓存、分组、导出、详情配置等操作。

## 9. 状态设计

| 状态 | 设计要求 |
| --- | --- |
| 默认封面模式 | 使用 `图片/01-书架首页-封面模式.png` 的结构和密度，以文字 token 校准细节 |
| 默认列表模式 | 使用 `图片/02-书架首页-列表模式.png` 的结构和密度，以文字 token 校准细节 |
| 空书架 | 保留 App Shell、顶部栏、分组区和底部导航；内容区显示空态，不改变页面视觉系统 |
| Loading | 保留页面框架；书籍区使用骨架屏；顶部信息卡可使用卡片骨架 |
| Error | 保留页面框架；内容区提供错误说明和 `重试` |
| 无封面 | 使用 `Shelf.Cover.Placeholder*`，不得显示破图或空白色块 |
| 长书名 | 书名最多两行，作者和章节最多一行，尾部省略 |

状态页只能替换内容区，不能重画底部导航或引入新的卡片样式。

## 10. 视觉一致性要求

以下 token 在封面模式、列表模式、空态、加载态、错误态中必须一致：

- `Shelf.Page.*`
- `Shelf.TopBar.*`
- `Shelf.Chip.*`
- `Shelf.Card.*`
- `Shelf.PanelCard.*`
- `Shelf.Cover.Radius`
- `Shelf.Cover.AspectRatio`
- `Shelf.Progress.*`
- `Shelf.SectionTitle.*`
- `Shelf.BottomNav.*`

允许变化的只有：

- 书籍内容区的布局方向。
- 封面展示尺寸。
- 书籍项高度。
- 进度条宽度。
- 每屏可见书籍数量。

## 12. 明确排除

- 不新增 `我的` 主标签页。
- 不在书架首页展示账户头像、登录入口、会员卡片或广告横幅。
- 不把书源管理、换源、缓存、导出放进书架首页。
- 不把阅读器控制层、搜索结果页、书籍详情页混入书架首页。
- 不把历史图或候选图页面包作为 P0 实现输入。
- 不在书籍操作底表增加 `修改`、`删除` 之外的操作。

## 13. 验收标准

正式 UI 设计通过标准：

- 封面模式截图和列表模式截图看起来属于同一套视觉系统。
- 首屏能同时识别页面标题、分组、顶部信息卡、书架内容和底部主导航。
- 小屏、常规屏、大屏均不依赖缩小字号适配。
- 书籍封面不拉伸，进度条、卡片边框、圆角和阴影一致。
- 书籍操作底表只包含 `修改` 和 `删除`。
- Stitch 使用本页面包能复原出书架根页面，而不是拼出多个相互冲突的草稿版本。
