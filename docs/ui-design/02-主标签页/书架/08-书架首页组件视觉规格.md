# 书架首页组件视觉规格

状态：正式视觉 token
适用范围：书架首页封面模式、书架首页列表模式、书架首页 loading / empty / error 框架
正式页面入口：`10-正式UI设计稿.md`

## 1. 完整性评估

用户提出的视觉规格项是必要项，覆盖了页面还原的主要组件 token：

- 页面背景色 token。
- 卡片背景色、边框颜色、边框粗细、圆角、阴影层级。
- 分组 chip 高度、padding、圆角、激活/未激活颜色、文字样式。
- 书籍封面比例、圆角、最小/最大尺寸。
- `BookCard` 和 `BookListItem` 的内边距、最小高度、标题/作者/章节字号。
- 进度条高度、圆角、轨道色、填充色、长度规则。
- 顶部继续阅读/最近更新模块的统一卡片规格。
- 底部导航高度、圆角、图标尺寸、文字字号、激活态。
- 封面模式和列表模式切换时哪些组件必须保持同一 token。

但如果目标是让 Stitch 和实现都稳定复原，还需要补充以下执行约束：

- 安全区、页面边距、底部导航避让。
- 滚动容器和固定容器的视觉边界。
- 响应式断点、网格列数、最小卡片宽度。
- 图标线宽、图标按钮触控区。
- 文本行数、截断、最长词/长书名处理。
- 图片加载失败、无封面占位、封面裁切规则。
- loading / empty / error 状态是否复用同一组件 token。
- 模式切换时哪些尺寸保持，哪些允许重排。

## 2. Token 命名

本稿使用语义 token。具体数值是书架主 Tab 的正式视觉基线；如果与图片冲突，以本稿为准。

## 3. 页面与安全区

| Token | 建议值 | 说明 |
| --- | --- | --- |
| `Shelf.Page.Background` | `#F8F5EF` | 书架首页统一暖白背景 |
| `Shelf.Page.HorizontalPadding` | `16dp` | 手机常规左右边距 |
| `Shelf.Page.CompactHorizontalPadding` | `14dp` | 小屏宽度低于 360dp 时使用 |
| `Shelf.Page.MaxContentWidth` | `720dp` | 折叠屏/大屏内容最大宽度，超过后居中 |
| `Shelf.SafeArea.TopBehavior` | follow system inset | 顶栏下移，不遮挡状态栏 |
| `Shelf.SafeArea.BottomBehavior` | follow system inset + nav height | 内容底部避让主导航 |

页面不允许使用多套相近背景色。封面模式、列表模式、空态、加载态必须使用同一个页面背景 token。

## 7. 顶部信息模块

顶部 `继续阅读` 和 `最近更新` 复用 `Shelf.PanelCard`。

| Token | 建议值 | 说明 |
| --- | --- | --- |
| `Shelf.PanelCard.Background` | `Shelf.Card.Background` | 复用通用卡片背景 |
| `Shelf.PanelCard.BorderColor` | `Shelf.Card.BorderColor` | 复用通用边框 |
| `Shelf.PanelCard.BorderWidth` | `1dp` | 统一边框 |
| `Shelf.PanelCard.Radius` | `16dp` | 统一圆角 |
| `Shelf.PanelCard.Padding` | `16dp` | 统一内边距 |
| `Shelf.PanelCard.MinHeight` | `168dp` | 常规手机并排时 |
| `Shelf.PanelCard.CompactMinHeight` | `132dp` | 短屏/窄屏堆叠时 |
| `Shelf.PanelTitle.Size` | `16sp` | 分区标题 |
| `Shelf.PanelTitle.Weight` | `600` | 分区标题字重 |

### 自适应

- 宽度小于 `390dp`：顶部信息模块纵向堆叠。
- 宽度大于等于 `390dp` 且高度充足：两个模块可并排。
- 高度小于 `760dp`：压缩顶部模块高度，优先让封面网格首行可见。

## 14. 响应式和自适应

| 条件 | 规则 |
| --- | --- |
| 宽度 `<360dp` | 使用紧凑边距，网格 2 列，顶部信息模块纵向堆叠 |
| 宽度 `360-430dp` | 网格根据 `BookCard.MinWidth` 自动 2 或 3 列 |
| 宽度 `431-600dp` | 网格最多 3 列，增加卡片宽度或内容边距 |
| 宽度 `>600dp` | 内容最大宽 `720dp`，网格最多 4 列，页面居中 |
| 高度 `<760dp` | 压缩顶部信息模块，优先显示书架首行 |
| 横屏 | 保留顶部栏和底部导航；网格最多 4 列，避免卡片过宽 |

自适应不允许通过缩小字号解决。优先级为：减少列数、重排顶部信息模块、增加滚动。

## 15. 文本溢出

- 书名最多两行，尾部省略。
- 作者最多一行，尾部省略。
- 当前章节最多一行，尾部省略。
- 分组名最多一行，尾部省略。
- 按钮文案不得截断；空间不足时使用图标按钮或重排。

## 16. 状态复用

Loading、empty、error、offline 状态必须复用：

- `Shelf.Page.Background`
- `Shelf.TopBar.*`
- `Shelf.Chip.*`
- `Shelf.BottomNav.*`
- `Shelf.Card.*`

状态页只能替换内容区域，不得重画 App Shell。

## 17. 封面模式和列表模式必须保持一致的 token

以下 token 在两种模式中必须相同：

- 页面背景。
- 顶部栏高度、标题样式、图标样式。
- 分组 chip 样式。
- 顶部信息模块卡片样式。
- 通用卡片背景、边框、圆角、阴影。
- 封面圆角和封面比例。
- 进度条颜色、高度、圆角。
- 书架标题行字号和右侧控件样式。
- 底部导航全部 token。

以下内容允许随模式变化：

- 书籍内容布局方向。
- 封面尺寸。
- 卡片高度。
- 进度条宽度。
- 每屏可见书籍数量。

## 18. Stitch 使用要求

用于 Stitch 时必须明确：

- 本稿优先级高于截图里的偶然色值和尺寸。
- `01-书架首页-封面模式.png` 和 `02-书架首页-列表模式.png` 是同一页面的两个状态，不是两套视觉系统。
- Stitch 不得从两张图分别学习两套 chip、卡片、底栏或进度条样式。
- 如果图片与本稿冲突，以本稿为准。
