# 书架

## 页面内容
### 页面定位

书架首页是 App P0 根页面，用于查看本地书架、继续阅读、切换分组、切换封面/列表模式，并进入搜索、书籍详情和阅读。

### 页面结构

1. 系统状态栏安全区。
2. 顶部栏：标题 `书架`，右侧搜索、更多。
3. 分组 chip：`全部 / 在读 / 已完结 / 本地 / 自定义分组`。
4. 顶部信息模块：`继续阅读`、`最近更新`。
5. 标题行：`我的书架`，右侧排序/筛选、封面/列表切换。
6. 书籍内容区：封面网格或列表。
7. 底部主导航：`书架 / 发现 / RSS / 设置`，当前高亮 `书架`。

### 滚动与固定

- 顶部栏可随页面滚动保持常规位置，不做沉浸吸顶要求。
- 底部主导航固定。
- 内容底部必须避让底部主导航和系统手势区。
- 封面模式和列表模式只改变书籍内容区布局，不改变 App Shell。

## 交互逻辑
| 元素 | 交互 |
| --- | --- |
| 搜索图标 | 进入书籍搜索首页 |
| 更多图标 | 打开书架级更多菜单，后续定义 |
| 分组 chip | 当前页内切换书籍分组，保留模式 |
| 继续阅读卡 | 进入最近阅读位置 |
| 最近更新卡 | 进入对应书籍详情或章节 |
| 排序/筛选 | 打开排序与筛选底表 |
| 封面/列表切换 | 当前页内切换展示模式 |
| 书籍卡/行 | 进入书籍详情 |
| 书籍更多按钮 | 打开书籍操作底表 |
| 底部发现/RSS/设置 | 切换到对应主标签页 根页面 |

### 书籍操作底表

书架页书籍操作底表只允许：

- `修改`：进入书籍详情页，在详情页修改配置。
- `删除`：对应移除书架。

不得在底表中放置书源、缓存、分组、导出等配置型操作。

| 状态 | 要求 |
| --- | --- |
| 默认封面模式 | 展示顶部模块和封面网格 |
| 默认列表模式 | 同一数据切换为列表行 |
| 空书架 | 保留顶部栏和底部导航，内容区显示空态 |
| 首次 loading | 保留页面框架，继续阅读、最近更新和书籍区都使用骨架；不显示旧内容遮罩 |
| 刷新中 | 保留已有内容，顶部或内容区显示轻量刷新反馈；不得清空书架 |
| 分组切换 loading | 保留顶部栏、分组 chip 和底部导航，只替换当前分组书籍区骨架 |
| 封面加载中/失败 | 只替换单本书封面区域，不触发整页 loading |
| 部分刷新失败 | 保留已有书籍，用轻量反馈说明失败数量，可重试 |
| error | 保留页面框架，完整替换内容区为错误说明和重试；不叠加半透明蒙层，不露出旧书籍内容 |
| 无封面 | 使用封面占位 |
| 长书名 | 标题最多两行，尾部省略 |

状态页不得改变底部导航结构。

- 不要在底部导航中添加 `我的` 或 `书源`。
- 不要在书籍操作底表中添加除 `修改`、`删除` 之外的操作。
- 不要在书架首页展示账户头像、登录入口、会员卡片。
- 不要把搜索结果、书籍详情、阅读器控制层直接混进书架首页。
- 不要使用营销式大横幅替代继续阅读和最近更新模块。
