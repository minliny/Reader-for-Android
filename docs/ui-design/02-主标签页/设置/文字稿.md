# 设置

## 页面内容
### 页面定位

设置首页是底部主导航 `设置` 对应的根页面，用于聚合 App 级管理入口。它不是账户页，也不是阅读器内设置面板。

### 页面结构

1. 系统状态栏安全区。
2. 顶部栏：标题 `设置`，右侧搜索、更多。
3. 本地概览卡：本地书籍、订阅源、书源可用、最近备份。
4. 常用管理入口：`书源管理`、`RSS/订阅管理`、`阅读偏好`、`缓存管理`。
5. 分组列表：通用、阅读、书架与搜索、书源与订阅、同步与备份、隐私与权限、关于与反馈。
6. 底部主导航：当前高亮 `设置`。

### 范围边界

- 设置首页只作为入口聚合页。
- 设置二级页不在本页面包内展开。
- 不展示账户头像、昵称、登录入口。

| 组件 | 说明 |
| --- | --- |
| `AppTopBar` | 标题、搜索、更多 |
| `LocalOverviewCard` | 本地状态概览 |
| `QuickEntryGrid` | 常用管理入口 2×2 |
| `SettingsSection` | 设置分组 |
| `SettingsListItem` | 设置行 |
| `BottomNav` | 底部主导航 |

设置首页应保持管理型页面的扫描效率，避免营销页式 hero 和超大装饰卡。

- 页面标题：`设置`
- 概览字段：`本地书籍`、`订阅源`、`书源可用`、`最近备份`
- 常用入口：`书源管理`、`RSS/订阅管理`、`阅读偏好`、`缓存管理`
- 分组：`通用`、`阅读`、`书架与搜索`、`书源与订阅`、`同步与备份`、`隐私与权限`、`关于与反馈`
- 状态：`未设置`、`待授权`、`可用`、`需要处理`
- 底部导航：`书架`、`发现`、`RSS`、`设置`

## 交互逻辑
| 元素 | 当前阶段交互 |
| --- | --- |
| 搜索图标 | 保留入口，设置内搜索后续设计 |
| 更多图标 | 保留入口，导入导出/恢复默认后续设计 |
| 书源管理 | 进入书源管理首页 |
| RSS/订阅管理 | 保留入口，次级页暂缓 |
| 阅读偏好 | 进入阅读设置或阅读偏好页 |
| 缓存管理 | 进入缓存管理页 |
| 分组列表行 | 进入对应设置二级页 |
| 底部导航 | 切换主标签页 |

所有破坏性操作必须在二级页中确认，设置首页不直接执行清空、删除、恢复默认。

| 状态 | 当前要求 |
| --- | --- |
| 默认态 | 使用 |
| 局部数据加载 | 概览卡显示骨架或占位，入口列表保持可见 |
| 无备份 | `最近备份` 显示 `未设置` |
| 权限缺失 | 对应行显示状态提示，不弹全屏错误 |
| 搜索/更多 | 当前只保留入口，不补弹层状态 |

设置首页不需要全屏空态；如果本地统计为空，也应显示可进入的管理入口。

- 不要展示账户头像、昵称、登录、个人主页。
- 不要把 `我的` 做成设置首页的替代入口。
- 不要把 `书源` 放回底部主导航。
- 不要使用营销式 hero 或装饰大图。
- 不要在设置首页直接执行删除、清空、恢复默认等破坏性操作。
- 不要沿用人工 FAIL/EXCLUDE 的 `48` 组图片作为当前设置二级页 source。
