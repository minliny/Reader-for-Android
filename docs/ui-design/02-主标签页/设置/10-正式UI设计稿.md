# 设置首页正式 UI 设计稿

状态：正式稿
等级：P0 / MAIN_TAB_ROOT
适用页面：底部主导航 `设置` 根页面
视觉参考图片：`图片/01-设置首页-管理入口.png`

## 1. 设计目标

设置首页用于聚合 App 级管理入口，服务三个任务：

- 快速查看本地阅读环境状态。
- 快速进入常用管理功能。
- 以稳定分组进入设置二级页。

本页不是账户页、个人主页、会员页或阅读控制层。所有配置修改在二级页完成，设置首页不直接执行清空、删除、恢复默认等破坏性操作。

## 3. 页面信息架构

页面从上到下为：

1. 系统状态栏安全区。
2. 顶部栏：标题 `设置`，右侧搜索图标和更多图标。
3. 本地概览卡：`本地书籍 / 订阅源 / 书源可用 / 最近备份`。
4. 常用管理入口：`书源管理 / RSS/订阅管理 / 阅读偏好 / 缓存管理`。
5. 设置分组列表：`通用 / 阅读 / 书架与搜索 / 书源与订阅 / 同步与备份 / 隐私与权限 / 关于与反馈`。
6. 底部主导航：`书架 / 发现 / RSS / 设置`，当前高亮 `设置`。

内容区为纵向滚动容器，底部必须避让主导航和系统手势安全区。

## 5. 交互定义

| 入口 | 行为 |
| --- | --- |
| 搜索图标 | 保留设置内搜索入口，P0 不展开搜索页 |
| 更多图标 | 保留导入导出/恢复默认入口，P0 不展开菜单 |
| 书源管理 | 进入 `../../05-设置链路/书源管理/00-说明.md` |
| RSS/订阅管理 | 保留入口，订阅管理 P1 |
| 阅读偏好 | 进入阅读设置或阅读偏好二级页 |
| 缓存管理 | 进入 `../../05-设置链路/缓存管理/00-说明.md` |
| 设置分组行 | 进入对应设置二级页 |
| 底部主导航 | 切换主 Tab 根页面 |

设置首页不直接执行破坏性操作。

## 6. 状态设计

| 状态 | 设计要求 |
| --- | --- |
| 默认态 | 使用 `图片/01-设置首页-管理入口.png` 的结构，以视觉 token 校准细节 |
| 概览加载 | 概览字段骨架化，快捷入口和分组列表保持可见 |
| 无备份 | `最近备份` 显示 `未设置` |
| 权限缺失 | 对应设置行展示 `待授权` 或 `需要处理` |
| 搜索/更多 | 当前只保留入口，不生成弹层 |

## 7. 自适应规则

- 小屏优先让顶部栏、本地概览、至少两个快捷入口和底部导航可见。
- 快捷入口在窄屏可由 2×2 改为单列，但不得缩小图标和文字。
- 大屏内容宽度不超过 `AppShell.Page.MaxContentWidth`。
- 所有二级页入口保持单列扫描，不使用营销式卡片墙。

## 9. 明确排除

- 不展示账户头像、昵称、登录入口。
- 不新增 `我的` 主标签。
- 不把书源管理放回底部主导航。
- 不使用营销 hero、会员横幅或广告入口。
- 不在首页直接执行删除、清空、恢复默认。

## 10. 验收标准

- 与书架首页共享顶部栏、底部导航、背景、卡片和图标线性语言。
- 首屏可识别本地概览、常用管理入口和底部主导航。
- 设置入口密度适合反复使用，不像营销页。
- Stitch 输出不包含账户、登录、会员或多余主标签。
