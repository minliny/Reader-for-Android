# 设置首页视觉规格

状态：正式视觉 token
适用范围：底部主导航 `设置` 根页面
正式页面入口：`10-正式UI设计稿.md`

## 1. 继承关系

设置首页是主 Tab 根页面，优先继承 `../../01-全局规范/02-应用外壳组件视觉规格.md`：

- 页面背景、顶部栏、底部导航、主标签页安全区使用 `AppShell.*`。
- 卡片、列表行、图标和状态组件使用 `AppShell.*`，但内容密度参考 `../../05-设置链路/00-设置页面视觉规范.md`。
- 设置首页不得使用设置二级页的返回顶栏；它必须保留底部主导航。

如果图片与本文件冲突，以本文件和 App Shell token 为准。

## 7. 底部导航

底部导航完全继承 `AppShell.BottomNav.*`。当前激活项为 `设置`，顺序必须为：

```text
书架 / 发现 / RSS / 设置
```

不得出现 `我的` 或 `书源` 主标签。

## 8. 响应式规则

| 条件 | 规则 |
| --- | --- |
| 宽度 `<360dp` | 页面左右边距使用紧凑值；概览卡仍为 2 列；快捷入口可降为单列 |
| 宽度 `360-430dp` | 概览 2 列，快捷入口 2×2，设置列表单列 |
| 宽度 `>430dp` | 内容最大宽居中；快捷入口保持 2×2，不拉成 4 列 |
| 高度 `<760dp` | 概览卡压缩字段间距，优先露出至少两个快捷入口 |

不通过缩小字号适配小屏；优先增加滚动。

## 9. 状态复用

- 局部数据加载：概览字段用骨架，不隐藏入口。
- 无备份：最近备份显示 `未设置`。
- 权限缺失：对应列表行显示状态，不弹全屏错误。
- 搜索/更多：当前只保留入口，不在根页生成弹层。
