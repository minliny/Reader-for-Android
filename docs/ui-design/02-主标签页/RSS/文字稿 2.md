# RSS

## 页面内容
### 页面定位

RSS 首页是底部主导航 `RSS` 对应的根页面，用于查看订阅源最新内容流。当前阶段只保留首页结构。

### 页面结构

1. 系统状态栏安全区。
2. 顶部栏：标题 `RSS`，右侧搜索、更多。
3. `订阅概览` 卡片：订阅源数量、未读数量、更新时间、刷新入口。
4. 内容筛选 chip：`全部 / 未读 / 收藏 / 稍后读 / 书单`。
5. 订阅源筛选 chip：`全部来源 / 小说更新 / 技术文章 / 书单推送`。
6. `最新订阅` 列表。
7. 底部主导航：当前高亮 `RSS`。

### 范围边界

- 当前只生成 RSS 首页。
- RSS 搜索、订阅管理、添加订阅源、条目详情、分组管理暂缓。
- 行内三点菜单只保留入口，不展开菜单设计。

| 组件 | 说明 |
| --- | --- |
| `AppTopBar` | 标题、搜索、更多 |
| `SubscriptionSummaryCard` | 订阅概览 |
| `FeedStatusChips` | 全部/未读/收藏等 |
| `FeedSourceChips` | 订阅源分组 |
| `RssEntryItem` | 订阅条目行 |
| `UnreadIndicator` | 未读状态 |
| `BottomNav` | 底部主导航 |

RSS 列表可以比设置列表更高，但图标、字号和触控区必须遵循 AppShell。

- 页面标题：`RSS`
- 概览标题：`订阅概览`
- 概览字段：`订阅源`、`未读`、`最近更新`
- 操作：`刷新`
- 内容筛选：`全部`、`未读`、`收藏`、`稍后读`、`书单`
- 来源筛选：`全部来源`、`小说更新`、`技术文章`、`书单推送`
- 列表标题：`最新订阅`
- 空态标题：`还没有订阅内容`
- 空态说明：`添加订阅源后，最新内容会显示在这里。`
- 错误标题：`订阅流加载失败`
- 错误操作：`重试`

## 交互逻辑
| 元素 | 当前阶段交互 |
| --- | --- |
| 搜索图标 | 保留入口，RSS 搜索暂缓 |
| 更多图标 | 保留入口，菜单暂缓 |
| 刷新 | 刷新当前订阅流 |
| 内容筛选 chip | 当前页内过滤列表 |
| 来源筛选 chip | 当前页内切换订阅源分组 |
| 订阅条目行 | 后续进入条目详情，当前只保留语义 |
| 行内三点菜单 | 后续用于标记已读/收藏/稍后读，当前不展开 |
| 底部导航 | 切换主标签页 |

当前阶段不生成订阅管理闭环。

| 状态 | 当前要求 |
| --- | --- |
| 默认态 | 使用 |
| 刷新中 | 保留订阅概览和筛选，列表局部 loading |
| 无订阅 | 使用 AppShell 空态，主操作 `添加订阅源` 后续定义 |
| 无未读 | 保留筛选上下文，提示当前筛选无内容 |
| 网络失败 | 保留上次内容或框架，提供 `重试` |

当前阶段不补 RSS loading/empty/error 图片；后续补图时必须复用 `AppShell.State.*`。

- 不要把 RSS 做成发现页的内容模块。
- 不要在当前阶段生成添加订阅、订阅管理、条目详情完整页面。
- 不要出现账户、登录、我的入口。
- 不要把书源管理作为 RSS 主页面功能。
- 不要改变底部主导航顺序。
