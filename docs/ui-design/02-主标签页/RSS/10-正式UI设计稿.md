# RSS 首页正式 UI 设计稿

状态：正式稿
等级：P0 / MAIN_TAB_ROOT
适用页面：底部主导航 `RSS` 根页面
视觉参考图片：`图片/01-RSS首页-订阅流.png`

## 1. 设计目标

RSS 首页用于查看订阅源最新内容流，服务三个任务：

- 查看订阅源、未读和更新时间概览。
- 按内容状态和订阅源分组过滤。
- 扫描最新订阅条目。

当前阶段只冻结 RSS 根页面，不展开 RSS 搜索、订阅管理、添加订阅源、条目详情或行内菜单。

## 3. 页面信息架构

页面从上到下为：

1. 系统状态栏安全区。
2. 顶部栏：标题 `RSS`，右侧搜索图标和更多图标。
3. 订阅概览卡：`订阅源 / 未读 / 最近更新 / 刷新`。
4. 内容筛选 chip：`全部 / 未读 / 收藏 / 稍后读 / 书单`。
5. 来源筛选 chip：`全部来源 / 小说更新 / 技术文章 / 书单推送`。
6. 列表标题：`最新订阅`。
7. 订阅条目列表。
8. 底部主导航：当前高亮 `RSS`。

## 5. 交互定义

| 入口 | 行为 |
| --- | --- |
| 搜索图标 | 保留 RSS 搜索入口，P1 |
| 更多图标 | 保留订阅管理/添加订阅源入口，P1 |
| 刷新 | 刷新当前订阅流 |
| 内容筛选 chip | 当前页内过滤列表 |
| 来源筛选 chip | 当前页内切换订阅源分组 |
| 订阅条目行 | 后续进入条目详情，当前只保留语义 |
| 行内三点菜单 | 后续用于标记已读/收藏/稍后读，当前不展开 |
| 底部主导航 | 切换主 Tab 根页面 |

当前阶段不生成订阅管理闭环。

## 6. 状态设计

| 状态 | 设计要求 |
| --- | --- |
| 默认态 | 使用 `图片/01-RSS首页-订阅流.png` 的结构，以 token 校准细节 |
| 刷新中 | 概览和筛选保持可见，列表局部 loading |
| 无订阅 | 使用 AppShell 空态，主操作 `添加订阅源` 后续定义 |
| 无未读 | 保留筛选上下文，提示当前筛选无内容 |
| 网络失败 | 保留上次内容或框架，提供 `重试` |

当前不补 loading/empty/error 图片；后续补图必须复用 `AppShell.State.*`。

## 7. 自适应规则

- 两条 chip rail 横向滚动，不换成多行标签墙。
- 订阅条目保持单列，大屏只增加内容宽度上限内的呼吸感。
- 小屏优先显示顶部栏、概览、首条筛选和第一个条目。
- 不通过缩小字号、隐藏摘要上下文或移除底部导航适配。

## 9. 明确排除

- 不把 RSS 做成发现页的内容模块。
- 不生成添加订阅、订阅管理、条目详情完整页面。
- 不出现账户、登录、我的入口。
- 不把书源管理作为 RSS 主页面功能。
- 不改变底部主导航顺序。

## 10. 验收标准

- 与书架首页共享 App Shell、chip、卡片和底部导航 token。
- 页面能清楚表达订阅概览、两级筛选和订阅流。
- Stitch 输出只包含 RSS 根页面。
- 未读状态可识别，不只依赖颜色。
