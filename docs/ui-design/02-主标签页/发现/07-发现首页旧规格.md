## 概念

发现页是 App 主入口页面，用于从书源和订阅源浏览可发现内容。它不是一个固定分类的全局推荐池，而是由当前选择的信息源决定内容结构：不同书源、订阅源可以拥有不同分类、榜单、栏目或更新流。

发现页不属于阅读控制层，不展示阅读页顶部栏、底部章节进度条、右侧亮度条或阅读快捷区。

当前设计状态：`暂定采用 41-1 主页面`。本阶段发现页只规划一个主页面，不继续生成信息源选择、更多分类、检测、更多菜单等次级页面或弹层；这些交互只作为后续功能设计和开发备注。

## 页面结构

- 顶部状态栏。
- 页面标题 `发现`。
- 顶部右侧 `搜索` 和 `更多`。
- 搜索入口。
- 信息源选择区：`全部来源 / 书源 / 订阅源`，以及当前来源或来源组。
- 来源内分类切换区：展示当前信息源返回的分类，不使用固定全局分类。
- 当前来源内容区：根据来源类型展示推荐、分类、榜单、更新、订阅条目或栏目内容。
- 来源状态区：展示当前来源数量、更新时间、可用状态或检测入口。
- 底部主导航：`书架 / 发现 / RSS / 设置`，当前高亮 `发现`。

## 信息源与分类规则

| 当前信息源 | 分类来源 | 内容结构 |
| --- | --- | --- |
| `全部来源` | 聚合系统分类和来源标签 | 综合推荐、跨源榜单、最近更新、来源分组 |
| `书源组` | 该书源组共同支持或合并后的分类 | 分类书籍、榜单、新书、完结、最近更新 |
| `单个书源` | 该书源真实返回的分类 | 以来源原始分类为准，例如玄幻、都市、历史，也可能是男频、女频、排行 |
| `订阅源组` | 订阅源栏目或标签 | 最新文章、专题、作者、栏目、热榜 |
| `单个订阅源` | 订阅源真实栏目 | 以订阅源栏目为准，例如资讯、推文、更新、专题 |

分类切换必须跟随信息源变化：

- 切换信息源后，分类栏刷新为该来源自己的分类。
- 如果来源没有分类，显示 `全部` 和来源提供的栏目或标签。
- 如果多个来源聚合，分类可以显示合并分类，但必须标明当前是聚合视图。
- 分类切换只改变当前来源下的内容，不自动切换信息源。

## 后续功能交互备注

以下内容只记录按钮语义和未来页面变化方向，不纳入当前发现页主页面设计范围；暂不生成对应 UI 图。

### 交互层级

| 层级 | 类型 | 使用场景 |
| --- | --- | --- |
| 当前页内刷新 | 不离开发现页 | 切换 `全部 / 书源 / 订阅`、切换来源内分类、刷新当前内容。 |
| 底部弹层 | 保留发现页背景 | 来源选择、更多分类、检测结果摘要。 |
| 右上浮层菜单 | 保留发现页背景 | 顶部更多菜单。 |
| 完整页面 | 离开发现页 | 搜索页、书籍信息页、完整榜单页、分类列表页、订阅详情页、书源管理页。 |
| 轻提示 | 短暂反馈 | 加入书架成功、刷新成功、检测完成、复制或操作失败。 |

### 顶部搜索图标 / 搜索入口栏

点击后进入完整 `书籍搜索页`。

页面变化：

- 离开发现页。
- 顶部显示返回按钮、标题或搜索输入框。
- 搜索页可继承当前发现页的信息源范围：如果发现页当前为 `书源 / 玄幻书源组`，搜索页默认带入该范围；用户仍可在搜索页切换范围。

退出：

- 点击返回回到发现页，保留原信息源、分类和滚动位置。

### 顶部更多图标

点击后打开右上浮层菜单。

菜单项建议：

- `刷新当前来源`
- `管理信息源`
- `发现页设置`
- `隐藏不可用来源`
- `清除发现缓存`

页面变化：

- 发现页背景保持不变，可轻微暗化。
- 菜单从右上角展开，不遮挡底部主导航。

后续跳转：

- `刷新当前来源`：关闭菜单，当前页进入局部刷新状态。
- `管理信息源`：进入完整 `书源/订阅源管理页`。
- `发现页设置`：进入完整 `发现页设置页`。
- `隐藏不可用来源`：菜单内切换开关，当前页内容重新筛选。
- `清除发现缓存`：弹出确认对话框，确认后清理缓存并刷新。

### `全部 / 书源 / 订阅`

点击后在当前页内切换信息源类型，不打开新页面。

页面变化：

- 选中态移动到对应标签。
- 当前来源行刷新：
  - `全部`：显示 `全部来源 · 书源与订阅聚合`。
  - `书源`：显示上次选择的书源组或默认书源组。
  - `订阅`：显示上次选择的订阅源组或默认订阅源组。
- 当前分类栏刷新为该信息源类型下的可用分类或栏目。
- 内容区显示 loading 骨架，加载完成后替换为新内容。

异常：

- 如果该类型没有可用来源，显示空状态卡片，并提供 `去添加来源`。

### 当前来源名称 / `切换来源`

点击后打开 `信息源选择底部弹层`。

弹层内容：

- 标题：`选择信息源`
- 顶部分段：`全部 / 书源组 / 单个书源 / 订阅组 / 单个订阅`
- 搜索输入：`搜索来源名称`
- 来源列表：名称、类型、可用状态、更新时间、来源数量或栏目数。
- 底部操作：`取消`、`应用`
- 入口按钮：`管理信息源`

页面变化：

- 发现页作为背景保留，底部弹层覆盖下半屏到 80% 高度。
- 选择来源后，弹层内显示选中态；点击 `应用` 后关闭弹层。
- 关闭后发现页刷新当前来源名称、来源内分类和内容区。

后续跳转：

- `管理信息源` 进入完整 `书源/订阅源管理页`。

### 来源内分类标签

点击后在当前页内切换分类，不打开新页面。

页面变化：

- 当前分类 chip 高亮。
- 内容区标题更新为 `当前来源 · 当前分类`。
- 推荐、榜单、更新列表进入局部 loading，然后显示该分类内容。
- 来源状态条不重置，只更新当前分类下的更新时间或命中数量。

规则：

- 分类名称来自当前来源，不做全局固定。
- 如果该来源分类过多，只展示前 5 到 6 个，剩余通过 `更多分类` 进入底部弹层。

### `更多分类`

点击后打开 `更多分类底部弹层`。

弹层内容：

- 标题：`选择分类`
- 当前来源名称。
- 分类搜索或快速定位。
- 分类网格或列表，包含来源真实分类名称。
- `全部` 固定在第一项。

页面变化：

- 选择分类后立即关闭弹层并刷新发现页内容。
- 如果分类需要二级目录，可在弹层内展开二级分类，不跳完整页。

### `刷新`

点击后刷新当前信息源和当前分类。

页面变化：

- `刷新` 按钮变为 loading 或旋转状态。
- 当前内容区显示轻量骨架，不清空整页结构。
- 成功后显示轻提示 `已刷新`，更新时间更新。
- 失败后显示行内错误条 `刷新失败，重试`。

### 来源状态条 `检测`

点击后打开 `来源检测底部弹层`。

弹层内容：

- 标题：`检测来源`
- 检测对象：当前来源组或当前单个来源。
- 检测步骤：连接、分类读取、列表读取、详情读取。
- 检测结果：可用数量、异常数量、失败来源列表。
- 操作：`重新检测`、`查看详情`、`关闭`

页面变化：

- 检测中不离开发现页。
- 检测完成后，来源状态条更新为最新可用数量。

后续跳转：

- `查看详情` 进入完整 `书源检测结果页` 或 `信息源管理页`。

### 推荐书籍封面 / 标题 / 榜单书籍行

点击后进入完整 `书籍信息页`。

页面变化：

- 离开发现页。
- 书籍信息页展示聚合书籍信息、来源列表、加入书架、开始阅读等。
- 如果来自订阅源的书单或文章，应先进入 `订阅内容详情页`，再由其中的书籍条目进入书籍信息页。

退出：

- 返回发现页，保留信息源、分类和滚动位置。

### `阅读`

点击后进入阅读页。

页面变化：

- 如果书籍已在书架且有阅读记录，进入上次阅读位置。
- 如果未加入书架但来源可读，直接打开临时阅读或先加入后阅读，规则需在书籍信息页统一。
- 如果来源不可用，显示轻提示或进入书籍信息页让用户选择其他来源。

### `加入书架`

点击后停留在发现页，执行加入操作。

页面变化：

- 按钮进入 loading 状态。
- 成功后按钮变为 `已加入` 或 `阅读`，显示轻提示 `已加入书架`。
- 失败后恢复按钮并显示错误轻提示。

### `来源榜单更新` 的 `更多`

点击后进入完整 `来源榜单页`。

页面变化：

- 完整页顶部显示返回按钮和标题。
- 继承当前来源和分类，可在页面内切换榜单维度。
- 榜单页不保留发现页底部主导航，或按 App 主页面规则保留，需后续统一。

### 底部主导航

| 按钮 | 页面变化 |
| --- | --- |
| `书架` | 切换到书架页，发现页状态保留在后台。 |
| `发现` | 当前已在发现页时，点击回到顶部；再次点击可刷新当前来源。 |
| `RSS` | 切换到 RSS/订阅页。 |
| `设置` | 切换到设置页。 |
