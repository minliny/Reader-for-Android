# 发现

## 页面内容
### 页面定位

发现首页是底部主导航 `发现` 对应的根页面，用于从当前选择的信息源浏览可发现内容。当前阶段只保留首页结构，不补完整发现链路。

### 页面结构

1. 系统状态栏安全区。
2. 顶部栏：标题 `发现`，右侧搜索、更多。
3. 搜索入口栏。
4. 信息源类型：`全部来源 / 书源 / 订阅源`。
5. 当前来源行：来源名称、状态、切换入口。
6. 来源内分类 chip：分类名称来自当前来源。
7. 内容区：推荐、榜单、最近更新或栏目内容。
8. 来源状态条：来源数量、更新时间、刷新入口。
9. 底部主导航：当前高亮 `发现`。

### 范围边界

- 当前只生成发现首页。
- 来源选择底表、更多分类、发现设置、榜单详情、来源检测暂缓。
- 搜索入口跳转到书籍搜索链路，不在发现首页内展开。

| 组件 | 说明 |
| --- | --- |
| `AppTopBar` | 标题、搜索、更多 |
| `SearchEntry` | 进入搜索的只读输入栏 |
| `SourceTypeSegment` | 全部来源/书源/订阅源 |
| `CurrentSourceCard` | 当前来源和切换入口 |
| `SourceCategoryChips` | 来源内分类 |
| `DiscoveryContentCard` | 推荐/榜单/栏目卡 |
| `SourceStatusBar` | 更新时间、刷新 |
| `BottomNav` | 底部主导航 |

发现首页不得使用独立于书架/RSS/设置的顶栏、卡片和 chip 体系。

- 页面标题：`发现`
- 搜索占位：`搜索书名、作者或来源`
- 来源类型：`全部来源`、`书源`、`订阅源`
- 来源操作：`切换来源`
- 分类默认项：`全部`
- 内容模块：`推荐内容`、`榜单更新`、`最近更新`
- 操作：`刷新`、`换一批`、`加入书架`、`阅读`
- 空态标题：`当前来源暂无内容`
- 空态说明：`可以刷新，或切换到其他来源。`
- 错误标题：`来源加载失败`
- 错误操作：`重试`

## 交互逻辑
| 元素 | 当前阶段交互 |
| --- | --- |
| 搜索图标/搜索入口 | 进入书籍搜索页 |
| 更多图标 | 保留入口，菜单暂缓 |
| `全部来源 / 书源 / 订阅源` | 当前页内切换内容类型 |
| 当前来源/切换来源 | 保留入口，来源选择底表暂缓 |
| 分类 chip | 当前页内刷新内容区 |
| 刷新 | 刷新当前来源和分类 |
| 推荐书籍/榜单行 | 进入书籍详情 |
| 加入书架 | 加入后显示轻提示 |
| 底部导航 | 切换主标签页 |

所有未展开的次级交互只保留入口语义，不生成完整次级页面。

| 状态 | 当前要求 |
| --- | --- |
| 默认态 | 使用 |
| 切换来源类型 | 内容区局部 loading，页面框架不变 |
| 来源无内容 | 使用 AppShell 状态组件，提供 `刷新` 或 `去管理来源` |
| 来源不可用 | 行内错误条，不替换整个页面 |
| 网络失败 | 保留来源和分类上下文，提供 `重试` |

当前阶段不补发现 loading/empty/error 图片；后续补图时必须复用 `AppShell.State.*`。

- 不要把发现做成固定全局分类池；分类必须来自当前信息源。
- 不要在当前阶段生成来源选择、更多分类、榜单详情等完整次级页面。
- 不要出现 `我的`、账户头像、登录入口。
- 不要把 RSS 首页内容直接混入发现首页。
- 不要使用与 AppShell 不一致的底部导航或卡片样式。
