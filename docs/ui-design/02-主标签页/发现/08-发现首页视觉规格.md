# 发现首页视觉规格

状态：正式视觉 token
适用范围：底部主导航 `发现` 根页面
正式页面入口：`10-正式UI设计稿.md`

## 1. 继承关系

发现首页是主 Tab 根页面，必须继承 `../../01-全局规范/02-应用外壳组件视觉规格.md`：

- 页面背景、顶部栏、底部导航、chip、卡片和列表行使用 `AppShell.*`。
- 内容区组件只定义发现独有密度和排列，不创建新视觉体系。
- 当前阶段只冻结根页面，不展开来源选择、更多分类、榜单详情或发现设置。

## 8. 底部导航

底部导航继承 `AppShell.BottomNav.*`。当前激活项为 `发现`，顺序固定为 `书架 / 发现 / RSS / 设置`。

## 9. 响应式规则

| 条件 | 规则 |
| --- | --- |
| 宽度 `<360dp` | 搜索入口和 chip 保持一行；内容卡片单列 |
| 宽度 `360-430dp` | 标准单列内容，chip 横滚 |
| 宽度 `>430dp` | 内容最大宽居中，不扩展为多列 |
| 高度 `<760dp` | 可压缩当前来源卡 meta 间距，优先露出首张内容卡 |

## 10. 状态复用

- 切换来源类型：内容区局部 loading，框架不变。
- 来源无内容：使用 `AppShell.State.*`，操作为 `刷新` 或 `去管理来源`。
- 来源不可用：当前来源卡内显示错误状态。
- 网络失败：保留来源和分类上下文，提供 `重试`。
