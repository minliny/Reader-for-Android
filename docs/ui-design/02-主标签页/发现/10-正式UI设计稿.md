# 发现首页正式 UI 设计稿

状态：正式稿
等级：P0 / MAIN_TAB_ROOT
适用页面：底部主导航 `发现` 根页面
视觉参考图片：`图片/01-发现首页-信息源分类.png`

## 1. 设计目标

发现首页用于从当前信息源浏览可发现内容，服务三个任务：

- 快速搜索书名、作者或来源。
- 在来源类型与来源分类之间切换。
- 浏览推荐、榜单或最近更新内容。

当前阶段只冻结发现根页面，不展开来源选择、更多分类、榜单详情、来源检测或发现设置。

## 3. 页面信息架构

页面从上到下为：

1. 系统状态栏安全区。
2. 顶部栏：标题 `发现`，右侧搜索图标和更多图标。
3. 搜索入口栏：`搜索书名、作者或来源`。
4. 来源类型 chip：`全部来源 / 书源 / 订阅源`。
5. 当前来源卡：来源名称、状态、`切换来源`。
6. 来源分类 chip：`全部` 和当前来源分类。
7. 内容区：推荐内容、榜单更新、最近更新。
8. 来源状态条：来源数量、更新时间、刷新入口。
9. 底部主导航：当前高亮 `发现`。

## 5. 交互定义

| 入口 | 行为 |
| --- | --- |
| 搜索图标/搜索入口 | 进入 `../../03-书架链路/书籍搜索/00-说明.md` |
| 更多图标 | 保留入口，菜单 P1 |
| 来源类型 chip | 当前页内切换内容类型 |
| 当前来源/切换来源 | 保留入口，来源选择底表 P1 |
| 分类 chip | 当前页内刷新内容区 |
| 刷新 | 刷新当前来源和分类 |
| 推荐书籍/榜单行 | 进入 `../../03-书架链路/书籍详情/00-说明.md` |
| 加入书架 | 加入后显示轻提示 |
| 底部主导航 | 切换主 Tab 根页面 |

未展开的次级交互只保留入口语义，不生成完整页面。

## 6. 状态设计

| 状态 | 设计要求 |
| --- | --- |
| 默认态 | 使用 `图片/01-发现首页-信息源分类.png` 的结构，以 token 校准细节 |
| 切换来源类型 | 内容区局部 loading，页面框架不变 |
| 来源无内容 | 使用 AppShell 状态组件，提供 `刷新` 或 `去管理来源` |
| 来源不可用 | 当前来源卡内显示状态，不替换整页 |
| 网络失败 | 保留来源和分类上下文，提供 `重试` |

当前不补 loading/empty/error 图片；后续补图必须复用 `AppShell.State.*`。

## 7. 自适应规则

- 来源类型和分类 chip 横向滚动，不换成多行标签墙。
- 内容卡片保持单列，避免大屏无限拉宽。
- 小屏优先显示顶部栏、搜索入口、来源类型和首张内容卡。
- 不通过缩小字号、隐藏底部导航或移除来源上下文适配。

## 9. 明确排除

- 不把发现做成固定全局分类池。
- 不把 RSS 首页内容直接混入发现。
- 不出现账户、登录、我的入口。
- 不改变底部主导航顺序。
- 不生成完整来源选择、更多分类、榜单详情。

## 10. 验收标准

- 与书架首页共享 App Shell、chip、卡片和底部导航 token。
- 页面能清楚表达“当前来源”和“来源内分类”。
- Stitch 输出只包含发现根页面。
- 内容卡片不变成营销 hero 或瀑布流。
