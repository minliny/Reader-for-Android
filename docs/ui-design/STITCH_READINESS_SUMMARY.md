# Stitch 初稿可行性判断

> 综合审计：UI 设计稿 + 图片元素 → Stitch 就绪度
> 日期：2026-06-05

---

## 总体结论

```
STITCH_NEAR_READY_WITH_P0_GAPS
```

### 1. 当前能否用 Stitch 生成「阅读模块初稿」

**YES。**

阅读控制层全部页面（沉浸页、控制层默认态、四主按钮快捷模块 ×4、顶部更多菜单、换源底表、夜间模式、亮度调节、章节进度）均有参考图和文字规范。5 张高度对齐图（16-5/18-3/19-1/22-2/25-4）已生成。内容搜索、自动翻页、内容替换、朗读、界面外观、目录/书签均有完整图片覆盖。Stitch 可直接使用这批图片作为风格和布局参考，生成阅读模块初稿。

### 2. 当前能否用 Stitch 生成「全 App 初稿」

**有条件 YES，但有两个 P0 页面阻塞。**

App Shell（书架/发现/RSS/设置）四张底栏主页面均已生成图片（01/02, 41-1/41, 42, 43），底栏结构已经稳定。但有以下页面无图：

- **书源管理首页**（设置内主要管理入口）
- **阅读内完整设置首页**（阅读控制层→设置快捷模块→完整设置> 的落地页）

这两个页面虽然有详细的 text description 和 layout plan，但没有参考图给 Stitch 做风格对齐。状态模板（loading/empty/error）也缺图。

### 3. 若不能，全 App 还差哪些 P0 页面

| 页面 | 影响 | 当前资料 |
| --- | -- | ---- |
| 书源管理首页 | 设置页最大管理入口，无法跳转 | 仅 text description + layout plan |
| 阅读内完整设置首页 | 阅读控制层「完整设置>」无落点 | 仅 text description + layout plan |
| 全局 loading/empty/error 状态模板 | 所有页面缺过渡态和反馈态 | 仅 text description |

### 4. 若不能，图片元素还差哪些 P0 组件

| 组件 | 影响 |
| --- | -- |
| 空状态插画/图标 | 书架/搜索/替换/书签/内容搜索无书/无结果时缺 UI |
| Loading 指示器 | 所有页面的加载过渡态无视觉方案 |
| Error/Network Error 提示 | 所有页面的网络失败/解析失败缺 UI |
| 确认弹窗（删除/清空/恢复默认） | 破坏性操作缺 UI 参考 |
| Toast/Snackbar | 操作反馈缺轻量 UI |

### 5. 第一批 Stitch 最适合生成哪些页面

| 批次 | 页面/模块 | 任务类型 | 输入资料 | 是否建议执行 |
| -- | ----- | ---- | ---- | ------ |
| 1 | 阅读控制层 5 张高度对齐图终审 | FIX_EXISTING_PAGE_POSITION | 16-5/18-3/19-1/22-2/25-4 + 04 基准 | 是（已有图，仅需终审确认） |
| 2 | 书源管理首页默认态 | GENERATE_MISSING_PAGE | `09_MISSING_PAGE_TEXT_DESCRIPTIONS.md` + `10_MISSING_PAGE_LAYOUT_PLAN.md` + 现有 drafts 风格 | 是 |
| 3 | 阅读内完整设置页首页默认态 | GENERATE_MISSING_PAGE | `09_MISSING_PAGE_TEXT_DESCRIPTIONS.md` + `10_MISSING_PAGE_LAYOUT_PLAN.md` + 现有 drafts 风格 | 是 |
| 4 | 通用状态模板（loading/empty/error/network_error/delete_confirm） | GENERATE_STATE_VARIANTS | `05_GLOBAL_STATE_TEMPLATES.md` + `09_MISSING_PAGE_TEXT_DESCRIPTIONS.md` | 是 |
| 5 | 组件风格指南（提取颜色/字体/间距/圆角 token） | GENERATE_COMPONENT_STYLE_GUIDE | 全量 drafts 图片 | 是 |

### 6. 哪些页面不建议现在交给 Stitch

| 页面 | 原因 | 何时可进行 |
| --- | -- | ------- |
| 书源详情/编辑页 | 无书源首页就无进入路径 | 书源管理首页生成后 |
| 阅读内完整设置子页面（屏幕/手势/信息/高级/预设） | 无完整设置首页就无进入路径 | 阅读内完整设置首页生成后 |
| 发现次级页（信息源选择、分类弹层、更多菜单） | 当前规划中第一轮只保留发现主页面 | 后续功能开发阶段 |
| RSS 次级页（添加订阅、订阅管理、条目详情） | 当前规划中第一轮只保留 RSS 主页面 | 后续功能开发阶段 |
| 设置次级功能页（通用、书架、同步、隐私、关于） | 当前规划中第一轮只保留设置主页面 | 后续功能开发阶段 |
| 书架排序/分组/布局设置底表 | 当前规划为 P1 | 第一批核心页面之后 |

## Stitch 前置最小补齐清单

| 优先级 | 缺口 | 影响 | 最小补齐动作 |

| P0 | 无书源管理首页图 | 设置页书源管理入口无落点 | 用 Stitch 生成书源管理首页默认态（参考 `06_STITCH_PROMPTS.md` #5） |
