# 页面级规划收敛表

审计日期：2026-06-13

## 1. 收敛原则

本文件按应用页面规划收敛，不按已有图片数量收敛。

页面只有同时满足以下条件，才可进入“完成”：

1. 页面属于当前范围，且优先级明确。
2. 页面有唯一当前 source 图片；历史稿、参考图、拼图、备份图、Stitch 输出图不能算完成。
3. 页面有中文文字稿，覆盖入口、返回、布局、组件、交互、状态、动效、失败反馈。
4. 页面没有同级多版本冲突。
5. 页面通过人工审核或明确降级为参考。
6. 页面级 source、token、组件、交互映射闭合。

当前仓库没有页面达到完整完成门槛；`90-审计与索引/03-Stitch页面来源映射.md` 中页面级 `usable=true` 仍为 0。

## 2. 分级定义

| 分级 | 含义 | 当前处理 |
| --- | --- | --- |
| P0 | 当前必须先收敛的核心链路 | 先整理页面规划和 source，缺图优先补 |
| P1 | 下一阶段收敛，当前不阻塞 P0 | 保留入口和文字稿，图片不计入 P0 完成 |
| P2 | 后续增强或低频状态 | 保留规划，不进入当前补图队列 |
| 暂缓 | 本轮明确不做 | 不用旧图反向要求补齐 |
| 废弃 | 已被当前规划替代 | 从当前 source 中排除，只保留历史说明 |
