# 页面分级

状态：当前执行标准

## 1. 分级定义

| 层级 | 名称 | 定义 | 示例 |
| --- | --- | --- | --- |
| L0 | 全局规范 | 不对应具体页面，定义设计系统、token、组件、页面包模板、Stitch 输入标准 | 应用外壳、通用卡片、状态组件、页面输入约束 |
| L1 | 主标签页根页面 | 底部主导航一级页面 | 书架、发现、RSS、设置 |
| L2 | 主流程功能页 | 从 L1 进入的完整页面 | 书籍搜索、书籍详情、设置二级页、书源管理、RSS 订阅管理 |
| L3 | 页面状态 | 页面在数据或错误下的状态 | loading、empty、error、permission、offline |
| L4 | 浮层与局部组件 | 不独立作为完整页面的覆盖层或局部模块 | 底表、弹窗、菜单、toast、阅读控制面板 |
| L5 | 历史与参考 | 不能作为当前实现 source 的材料 | 旧图、失败图、备份、拼图、Stitch 初稿 |

## 2. 页面放置规则

### L0 全局规范

放入：

```text
docs/ui-design/01-全局规范/
```

包括：

- 应用外壳视觉规格。
- 通用组件视觉规格。
- 状态组件视觉规格。
- 页面包模板。
- Stitch 输入规范。

### L1 主标签页根页面

放入：

```text
docs/ui-design/02-主标签页/
```

只包括：

- 书架首页。
- 发现首页。
- RSS 首页。
- 设置首页。

禁止放入：

- 书籍搜索。
- 书籍详情。
- 阅读器。
- 书源管理。
- 设置二级页。
- RSS 条目详情。

### L2 主流程功能页

按链路放入：

```text
docs/ui-design/03-书架链路/
docs/ui-design/04-阅读链路/
docs/ui-design/05-设置链路/
docs/ui-design/06-发现链路/
docs/ui-design/07-RSS链路/
```

### L3 页面状态

状态不单独散落到全局目录，必须归属于页面包：

```text
页面名/
  04-状态规则稿.md
  图片/
    02-loading.png
    03-empty.png
    04-error.png
```

跨页面状态模板放入：

```text
docs/ui-design/01-全局规范/03-状态组件视觉规格.md
```

### L4 浮层与局部组件

若浮层强依附某页面，放入该页面包：

```text
书架首页/
  书籍操作底表/
```

若浮层是独立功能流程，放入对应链路：

```text
04-阅读链路/阅读控制层/
```

### L5 历史与参考

放入：

```text
docs/ui-design/91-历史归档/
```

历史材料必须标明：

- 原始路径。
- 归档原因。
- 是否可作为内容参考。
- 是否禁止作为视觉 source。

## 3. 优先级

| 优先级 | 当前含义 |
| --- | --- |
| P0 | 当前必须先闭合：书架根页面、书架搜索、书籍详情、阅读入口 |
| P1 | 下一阶段：设置首页、书源管理、阅读设置、设置二级页 |
| P2 | 后续增强：全量状态、横屏/平板、低频弹层 |
| 暂缓 | 发现和 RSS 的完整链路 |
| 废弃 | 我的主 Tab、书源独立主 Tab |

## 4. 判定规则

一个页面只有满足以下条件，才可以从候选进入冻结：

1. 有唯一页面包。
2. 页面包内有完整文字稿。
3. 当前图片位于页面包 `图片/` 目录。
4. 图片总册有对应条目。
5. 审计文档不再标记为缺图、冲突或 FAIL。
6. Stitch 输入不需要依赖历史图猜测。
