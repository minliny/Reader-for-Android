# UI 设计目录

状态：目录重构规划稿
日期：2026-06-13

## 1. 目标

`docs/ui-design/` 按页面层级和交付物类型重新组织，解决当前问题：

- 顶层同时混放审计、Stitch、页面规划、图片总册和历史草图。
- `drafts/` 承载了当前图、历史图、失败图和参考图，难以判断归属。
- 主标签页、书架链路、阅读链路、设置链路混在同一层。
- 页面包不统一，图片没有完全跟随页面。

重构后要求：

1. 每个当前页面只有一个页面包入口。
2. 当前图片跟随页面包，不再放在全局 `drafts/`。
3. 全局目录只放规范、索引、审计和历史归档。
4. 页面按层级分组：主标签页、书架链路、阅读链路、设置链路、发现链路、RSS 链路。
5. 历史、废弃、失败、备份图不得作为当前 source。

## 3. 各目录职责

| 目录 | 职责 | 不放什么 |
| --- | --- | --- |
| `00-总览/` | 目录说明、页面分级、优先级、迁移映射 | 具体页面稿、图片 |
| `01-全局规范/` | 唯一约束参考、通用 token、专项组件规范、页面包模板 | 单页面状态细节、历史审计结论 |
| `02-主标签页/` | 书架、发现、RSS、设置四个底部主导航根页面 | 搜索、详情、阅读器、设置二级页 |
| `03-书架链路/` | 书架下的搜索、详情、分组、排序、操作底表等 | 阅读器完整链路 |
| `04-阅读链路/` | 沉浸阅读、控制层、目录、朗读、外观、设置、换源 | App 设置主页面 |
| `05-设置链路/` | 设置首页下的二级和三级设置页、书源管理 | 底部主 Tab 根页面 |
| `06-发现链路/` | 发现首页后的来源选择、分类、榜单等 | 当前暂缓时只保留规划 |
| `07-RSS链路/` | RSS 首页后的订阅管理、添加订阅、条目详情等 | 当前暂缓时只保留规划 |
| `90-审计与索引/` | 图片总册、覆盖审计、Stitch 映射、上传清单、评审队列 | 页面包 source |
| `91-历史归档/` | drafts、rejected、backup、Stitch 初稿、旧审计 | 当前 source |

## 4. 迁移原则

- 先建目标目录和索引，再迁移页面包。
- 先迁移 Markdown，再复制图片，最后更新图片总册。
- 每次迁移一个页面组，迁移后运行图片文字稿审计。
- 不直接删除旧目录；先在旧目录放迁移说明，确认无引用后再归档。
- 旧路径若被外部文档引用，先更新引用或在迁移映射表中登记。

## 5. 首批迁移顺序

1. 主标签页根页面：从 `主标签页规划/` 迁移到 `02-主标签页/`。
2. 书架链路：从 `书架主Tab页面规划/` 拆到 `03-书架链路/`。
3. 全局规范：抽取页面设计输入约束、应用外壳视觉规格、组件视觉规格到 `01-全局规范/`。
4. 审计与索引：迁移图片总册、页面映射、上传清单和人工评审队列到 `90-审计与索引/`。
5. 历史归档：整理 `drafts/`、`_rejected/`、`_progress_restore_backup/`、Stitch 初稿到 `91-历史归档/`。

## 6. 当前注意

当前目录骨架已经建立，但旧文件还未整体迁移。任何新的页面设计应优先按目标目录创建，不再向旧顶层或全局 `drafts/` 增加当前 source。

新增或精修视觉数值前，必须先参考 `01-全局规范/00-唯一约束参考.md`。不得直接逐页补散落数值；应先按唯一约束确定视觉母版、完整组件局部、token 和验收规则，再让同类页面继承。
