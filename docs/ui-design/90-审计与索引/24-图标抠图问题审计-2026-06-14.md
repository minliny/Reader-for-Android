## 1. 结论

当前抠图资产仍未通过，不能作为最终图标资产使用。

2026-06-14 追加：已新增 `完整组件局部/` 与 `03-完整组件局部总览.png`。该版本不再尝试自动分离透明小图标，而是保留完整按钮/槽位/标签，作为当前更可信的视觉基准。

主要问题：

- 透明图标自动抠图仍不可靠；当前应优先参考完整组件局部。
- 部分透明图标被阈值算法吃掉抗锯齿边缘。
- 部分图标与按钮背景、文字或分隔线距离太近，自动抠图混入了非图标元素。
- 透明 PNG 只能作为“从设计图取形”的中间参考，不应直接进入正式页面绑定。

## 4. 修复原则

后续重新抠图时：

1. 原始局部必须完整包含图标槽位，不截断按钮圆角、阴影或图标外缘。
2. 透明图标不能混入文字、分隔线、进度线、滑条或卡片边缘。
3. 如果图标与文字/背景太近，保留 `原始局部` 作为参考，不生成透明图标。
4. 标准化 PNG 必须标注来源状态：`CUT_REFERENCE_ONLY`，不能标成可用资产。
5. 只有人工复核通过的图标，才可升级为 `CUT_READY`。
