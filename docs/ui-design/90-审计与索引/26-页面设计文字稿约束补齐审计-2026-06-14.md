## 后续缺口

- `NEEDS_VISUAL_SOURCE` 页面需要补独立 UI 设计图，不能仅靠文字稿进入 `STITCH_READY`。
- `READY_BASELINE` 只表示文字稿约束链路完整，仍需完整组件局部、资产状态和截图叠放验收才能升级到 `VISUAL_VERIFIED`。
- `VISUAL_REVIEW_REQUIRED` 表示文字稿和视觉规格已补齐，但当前 UI 图还需要完整组件局部和截图叠放复核后才能冻结。
- `P1_READY_BASELINE` 表示文字稿约束链路完整，但不进入 P0 冻结和当前 P0 Stitch 输入。
- `ROUTE_SPEC_READY` 表示入口、返回和异常链路约束完整，不单独定义新视觉母版。
- `REFERENCE_ONLY` 页面不得进入当前 Stitch/HTML 输入包。
