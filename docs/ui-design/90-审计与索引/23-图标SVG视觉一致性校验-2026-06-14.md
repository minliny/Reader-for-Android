## 1. 总结论

当前 SVG 未通过第二轮视觉复核，不能作为 HTML/Stitch 初稿生成的正式图标资产使用。

当前 SVG 的价值仍然只是“工程化占位源资产”：具备路径、语义名、`currentColor`、统一 viewBox 和可解析 SVG 结构，但视觉形态还没有达到基准图要求。

上一次把部分图标升级到 `ICON_DRAFT_READY` 是误判：我只判断了语义形态，没有按比例、对称、箭头形状和线条完整性逐项检查。现已撤回 `ICON_DRAFT_READY` 状态。

2026-06-14 追加处理：已从 UI 设计图直接抠出 PNG 图标参考资产，路径为 `docs/ui-design/01-全局规范/图标资产/从设计图抠图/`。但当前抠图资产仍存在裁剪不全、边缘被阈值吃掉、混入背景元素等问题，只能作为 `CUT_REFERENCE_ONLY`，不能作为页面初稿资产。

当前主要问题：

- 比例不对：图标在 24 viewBox 内的占比、粗细、留白与基准裁剪图不一致。
- 不对称：齿轮、书本、换源、替换、声波等图标左右或上下平衡不符合基准图。
- 箭头形状不对：返回、换源、替换、自动翻页、排序箭头端点和折角不符合基准图。
- 线条丢失或结构遗漏：目录圆点、设置齿轮齿、换源交叉线、替换回转关系等仍需按基准重画。

2026-06-14 修正进展：原 `01-基准图图标裁剪对照.png` 裁剪不完整，尤其快捷操作和箭头类图标缺失关键区域，已废弃。当前以 `03-完整局部裁剪对照.png` 和 `04-箭头类基准与SVG对照.png` 为准。修正后的图标不标为通过，统一进入 `ICON_REVISED_REVIEW`，等待与基准图继续复核。

## 4. 修复方向

下一步不应继续扩展页面绑定，而应先把图标资产分级：

| 等级 | 含义 | 处理 |
| --- | --- | --- |
| `ICON_APPROVED` | 与最终页面截图像素级一致 | 可进入实现验收 |
| `ICON_DRAFT_READY` | 与基准图形态一致，可用于初稿 | 本轮没有图标达到该状态 |
| `ICON_REDRAW` | 语义正确但形状不一致 | 先按基准图重绘 |
| `ICON_NEEDS_REDRAW` | 当前 SVG 有比例/对称/箭头/结构错误 | 重新绘制 SVG |
| `ICON_NEEDS_REMEASURE` | 形态接近但比例和尺寸未精确 | 从基准裁剪图重测后微调 |
| `ICON_REVISED_REVIEW` | 已重绘或重测一轮，但未确认通过 | 继续和基准图人工复核 |
| `ICON_UNUSED_IN_BASELINE` | 当前基准图没有该图标 | 暂不绑定到当前页面 |
| `ICON_PLACEHOLDER` | 仅用于占位或后续页面推演 | 不进入正式页面规格 |

## 6. 重绘方法

后续图标重绘按以下顺序执行：

1. 从基准图裁剪单个图标，保留按钮背景和图标外接框。
2. 记录图标在裁剪图中的外接框：`x / y / width / height`。
3. 记录笔画宽度：横线、竖线、斜线分别测量。
4. 记录关键点：箭头端点、折角、圆心、半径、圆点中心、线条端点。
5. 映射到 `24x24 viewBox`，保持视觉中心而不是几何中心。
6. 重绘 SVG 后渲染成 PNG，叠放到基准裁剪图上复核。
7. 只有通过叠放复核的图标才能进入 `ICON_DRAFT_READY`。

在 SVG 通过上述流程前，可参考 `从设计图抠图/标准化96/` 中的 PNG 资产取形；但 PNG 抠图本身也必须通过人工复核后才能进入页面稿。
