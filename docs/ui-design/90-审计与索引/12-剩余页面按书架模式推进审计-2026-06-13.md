## 1. 结论

当前流程可复制，但不能直接把所有页面一次性送 Stitch。

书架首页已经跑通的模式是：

1. 页面包有 `01-06` 基线稿。
2. 选定标准样例图和母版等级。
3. 写 `10-正式UI设计稿.md`，成为页面唯一正式入口。
4. 写页面级或母版级视觉 token。
5. 明确 Stitch 输入顺序：正式稿、视觉规格、结构/组件/交互/状态/文案/禁止项、图片。
6. 创建项目级 DESIGN.md。
7. 生成 demo。
8. 根据截图做 edit 精修。
9. 下载 HTML/PNG 并写入 `.stitch/metadata.json`。

剩余页面目前大多只完成第 1 步。它们有基线稿和图片，但缺少正式 UI 设计稿、页面级视觉 token、Stitch 输入包顺序和 demo 输出记录。

剩余页面共同缺口：

1. 缺 `10-正式UI设计稿.md`。
2. 缺页面级或母版级视觉 token。
3. 缺明确的 Stitch 输入顺序。
4. 缺 Stitch demo HTML/PNG 输出物。
5. 缺 demo 初评和二次 edit 精修记录。
6. 部分页面仍含 `待产品确认`、`草稿`、`候选`、`历史` 等状态标记，需要转为 `正式稿`、`P1` 或 `REFERENCE_ONLY`。

8. `.stitch` 生成物不应进入 `docs/ui-design/90-审计与索引/01-图片文字稿总册.md`，已经通过脚本排除。
