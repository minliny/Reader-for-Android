# 书源管理首页 Stitch 提示词

## 目标

生成 `书源管理首页`，用于补齐设置页中 `书源管理` 入口的落点。

## Stitch 提示词

```text
请生成一张中文阅读 App 的「书源管理首页」移动端 UI 设计稿。

基础要求：
- 画布尺寸：390 × 884。
- 移动端竖屏。
- 中文阅读 App。
- 页面属于书源管理模块。
- 这是二级管理页，不显示 App 主底栏。
- 严格使用 docs/ui-design/05-设置链路/00-设置页面视觉规范.md。
- 页面背景 #FAF7F2，卡片背景 #FFFCF7，主文字 #11100E，次级文字 #6F6A64，强调色 #087EAD。
- 不要生成小说阅读页。
- 不要改变 App Shell。
- 不要出现账户、头像、登录入口、我的页面结构。
- 输出可编辑图层，方便进入 Figma。

页面定位：
- 这是从「设置首页 > 书源管理」进入的完整管理页。
- 它不是阅读页顶部「换源」底表。
- 它不是底栏主页面，但视觉要和 App 主页面一致。
- 它是高密度工具型管理页面，不是普通设置页。

页面结构：
1. 顶部栏
   - 返回按钮。
   - 标题：书源管理。
   - 右侧搜索图标。
   - 右侧更多图标。

2. 状态概览卡
   - 展示 4 个紧凑指标：
     - 12 个书源
     - 8 个启用
     - 4 个异常
     - 刚刚检测
   - 使用小图标和轻分隔线。

3. 搜索与筛选区
   - 紧凑搜索框：搜索书源名称或域名。
   - 状态筛选 chips：全部 / 已启用 / 异常 / 未检测 / 自定义。
   - 分组筛选 chips：全部分组 / 玄幻书源 / 起点导入 / 测试书源。
   - 提供紧凑导入入口：URL / JSON / 本地文件，不做大卡片。

4. 书源列表
   - 列表行展示：
     - 书源名称。
     - 域名或来源说明。
     - 分组标签。
     - 最近检测状态：可用、异常、未检测。
     - 启用/禁用 switch。
     - 行内检测入口。
     - 行内调试信息入口或三点菜单。
   - 示例书源：
     - 起点中文网，可用，已启用。
     - 笔趣阁，异常，已启用。
     - 本地导入源，未检测，已禁用。
     - 测试书源，可用，已启用。
   - 检测失败时必须有行级错误提示，并可进入重试或查看日志。

5. 新增书源入口
   - 右下浮动按钮或列表上方主按钮：新增书源。
   - 不要做成过大的营销按钮。

视觉规则：
- 页面信息密度偏管理型，安静、可扫描。
- 可借鉴 Carbon / Ant Design 的信息密度和状态标签，但必须适配移动端列表。
- 不允许使用大卡片、hero、宽松留白或大插画把页面做成低密度普通设置页。
- 顶部返回箭头独立放置，不使用 pill、按钮底、外框或阴影。
- 页面标题 22-24sp；设置项标题 16-17sp；说明 12-13sp。
- 卡片圆角固定 8dp，1dp 浅暖灰描边，无明显阴影。
- 普通行高 56-64dp，带说明行高 68-76dp，同行之间使用细分隔线。
- 图标 22-24dp、统一 2dp 圆角线性描边，不混用实心图标。
- switch 固定 48×28dp；右箭头 18-20dp。
- 列表行高度稳定，右侧开关和检测入口对齐。
- 不使用阅读控制层、亮度滑条、章节进度条。
- 不要把检测页、详情页、编辑页画进同一张图，只保留入口。

输出目标：
- 书源管理首页默认态。
- 可编辑图层。
```
