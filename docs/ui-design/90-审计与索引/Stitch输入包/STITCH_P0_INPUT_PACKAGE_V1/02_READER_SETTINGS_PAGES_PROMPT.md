# 阅读内完整设置页 Stitch 提示词

## 目标

生成阅读控制层 `设置` 入口后续的完整阅读设置页组，补齐 `完整设置 >` 的落点。

## Stitch 提示词

```text
请生成中文阅读 App 的「阅读内完整设置页」UI 设计稿组。

基础要求：
- 画布尺寸：390 × 884。
- 移动端竖屏。
- 中文阅读 App。
- 严格使用 docs/ui-design/05-设置链路/00-设置页面视觉规范.md。
- 页面背景 #FAF7F2，卡片背景 #FFFCF7，主文字 #11100E，次级文字 #6F6A64，强调色 #087EAD。
- 从阅读控制层的「设置」入口进入，具体入口为：阅读控制层 > 设置快捷模块 > 完整设置 >。
- 完整设置页必须是独立全页，并明确可以返回阅读页。
- 不要保留阅读控制层底部四主按钮。
- 不要保留右侧亮度条。
- 不要生成字体、主题、版式、翻页动画页面；这些属于「界面」模块。
- 页面需要包含开关、选择项、滑块、预设恢复。
- 输出可编辑图层，方便进入 Figma。

需要生成或整理的页面：
1. 阅读设置首页
2. 阅读显示设置
3. 阅读行为设置
4. 阅读辅助设置
5. 阅读预设/恢复默认

页面 1：阅读设置首页
- 顶部栏：返回、标题「阅读设置」、右上更多。
- 说明卡：管理阅读行为，不含字体、主题、版式。
- 分组入口列表：
  - 屏幕与显示行为
  - 翻页与手势
  - 进度与信息显示
  - 阅读辅助
  - 缓存与章节加载
  - 预设与恢复默认
- 每行包含图标、标题、副说明、右箭头。
- 明确返回阅读页或阅读控制层。

页面 2：阅读显示设置
- 屏幕方向：跟随系统 / 竖屏 / 横屏。
- 屏幕超时：跟随系统 / 5 分钟 / 常亮。
- 状态栏显示开关。
- 导航栏显示开关。
- 阅读页底部信息显示开关。
- 使用分段控件、开关和设置项 row。

页面 3：阅读行为设置
- 音量键翻页开关。
- 点击区域设置入口。
- 左右滑动翻页开关。
- 单手翻页开关。
- 长按动作选择项。
- 误触保护开关。
- 可包含一个轻量滑块或强度选择，用于误触保护灵敏度。

页面 4：阅读辅助设置
- 章节加载失败自动重试。
- 缓存优先读取。
- 断点恢复。
- 阅读统计记录。
- 异常章节提示。
- 涉及权限的项显示状态标签。

页面 5：阅读预设/恢复默认
- 当前预设卡。
- 保存当前设置为预设。
- 预设列表。
- 应用预设。
- 删除预设。
- 恢复默认。
- 恢复默认必须有确认态入口，不要直接执行。

视觉规则：
- 全部页面保持同一视觉系统。
- 顶部返回箭头独立放置，不使用 pill、按钮底、外框或阴影。
- 页面标题 22-24sp；分组标题 15-16sp；设置项标题 16-17sp；说明 12-13sp。
- 卡片圆角固定 8dp，1dp 浅暖灰描边，无明显阴影。
- 普通行高 56-64dp，带说明行高 68-76dp，同行之间使用细分隔线。
- 图标 22-24dp、统一 2dp 圆角线性描边，不混用实心图标。
- switch 固定 48×28dp；右箭头 18-20dp；右侧控件统一对齐。
- 预览区使用次级浅暖色不透明容器，8dp 圆角和细边框。
- 不要出现 App 底栏。
- 不要出现阅读控制层亮度条和章节进度条。
- 不要做大 hero。
- 页面应偏管理型，列表紧凑可扫描。
- 危险操作放在底部或独立危险区。

输出目标：
- 至少输出阅读设置首页默认态。
- 如 Stitch 支持多画面，请同时输出显示、行为、辅助、预设四个子页面。
- 所有画面输出可编辑图层。
```
