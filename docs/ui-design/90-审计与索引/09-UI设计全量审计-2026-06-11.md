## 4. 关键发现（按严重程度排列）

### 🔴 严重 (CRITICAL)

#### C-1: Preflight 基线冲突 — `04` vs `04-2`

**位置：** `91-历史归档/freezes/.../01_FINAL_IMAGE_CANDIDATES.md` L14、`09_REDRAW_TASK_LIST.md` L32、`00_PREFLIGHT_SUMMARY.md` L42

**问题：** 权威 spec (`READER_CONTROL_LAYER_SPEC.md`) 明确规定 `04-2-reader-controls-opaque-icon-aligned.png` 为唯一视觉和几何基线。但 preflight 的以下位置仍引用 `04-reader-controls.png`：
- `01_FINAL_IMAGE_CANDIDATES.md` L14：`reader_overlay_default` 首选候选为 `04`（应为 `04-2`）
- `09_REDRAW_TASK_LIST.md` L32：说 `03 / 04` 已确认不变（应为 `03 / 04-2`）
- `00_PREFLIGHT_SUMMARY.md` L42：重画高度对齐目标引用为 `04`（应为 `04-2`）
- 第 5.2 节的"已解决"判断不准确：spec 层面已解决，但 preflight 中仍有 3 处引用错误

**影响：** 如果以此 preflight 执行 freeze，可能使用错误的基线图，导致整个 control layer redraw 对齐目标错误。

**修复：** 将 01、09、00 中所有对 `04-reader-controls.png` 的基线引用改为 `04-2-reader-controls-opaque-icon-aligned.png`。

---

#### C-2: Preflight `CANDIDATE_OK` 与 Spec `USE_CONTENT_REFERENCE_ONLY` 冲突

**位置：** `91-历史归档/freezes/.../01_FINAL_IMAGE_CANDIDATES.md` vs `04-阅读链路/阅读控制层/07-派生页面.md`

**问题：** Preflight 将 4 张 quick-panel 图（16-5、18-3、21-1、22-2）标记为 `CANDIDATE_OK`，但 `reader-control-derived-pages.md` L473-484 明确指出这些图应归类为 `USE_CONTENT_REFERENCE_ONLY`，需要生成新的 04-2 标准版本（16-6、18-4、21-2、22-3）。

**影响：** Freeze 包可能接受 spec 认为不够标准的图片。

**修复：** 需要人工决策 — 接受当前图为 freeze 级，还是承诺产出 04-2 标准版本。
