## 1. 总体结论

```
UI_COVERAGE_REAUDITED_WITH_P1_VISUAL_GAPS
```

**当前原因：** P0 图片输入已经补齐，normalized HTML 与 Compose handoff 也覆盖主要页面。
剩余缺口集中在 51 系列暖白 token 复核，以及 43/44/46/47/48/51 的人工视觉验收。
下方 2026-06-05 详细表中的 `MISSING` 是历史快照，不代表当前仓库状态。

核心缺口：

1. **主导航页面已覆盖**：书架、发现、RSS、设置均有设计图与规范。
2. **通用状态模板已覆盖**：51～51-4 提供 loading、empty、error、network、permission、确认和操作反馈。
3. **“我的”已取消**：能力合并到设置，不再作为待补页面。
4. **书源管理图片组已覆盖**：44、47～47-6 已存在。
5. **App 设置图片已覆盖**：43、48～48-5 已存在，仍待人工视觉终审。
6. **App 启动页 / Splash、权限弹窗 完全缺失**。
7. **多版本冲突有部分残留（28-34 vs 35-39），但已在冻结前整理中做出决策**。
8. **无统一基准画布和视觉 token 规范**。
