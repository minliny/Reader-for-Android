# 视觉母版术语误用审计

日期：2026-06-14

## 1. 结论

已将页面包内的“当前 UI 图 = 母版”误称拆分为四类：

| 类型 | 使用场景 | 是否可被其他页面继承 |
| --- | --- | --- |
| `VISUAL_MASTER` | 标准母版，定义同类页面可继承的视觉 token、组件结构和比例 | 可以 |
| `PAGE_VISUAL_BASELINE` | 单个页面自己的当前视觉基准图 | 不可以 |
| `PAGE_VISUAL_CANDIDATE` | 单个页面的候选视觉图，尚未确认 | 不可以 |
| `REFERENCE_BASELINE` | 引用其他页面或状态的参考图 | 不可以 |

本次重点修复两类明确错误：

1. `03-书架链路/本地书导入`：image2 生成图只属于本页候选图，不是母版，状态统一为 `PAGE_VISUAL_CANDIDATE`。
2. `04-阅读链路/阅读入口`：沉浸页和控制层图片是引用副本，不是阅读入口自己的视觉母版，角色统一为 `REFERENCE_BASELINE` / `STATE_OR_VARIANT_REFERENCE`。

## 2. 全局规则修订

`01-全局规范/00-唯一约束参考.md` 已修订：

- 输入包基准输出改为“当前页面视觉基准”。
- 图片角色新增 `PAGE_VISUAL_BASELINE`、`PAGE_VISUAL_CANDIDATE`、`MISSING_PAGE_VISUAL_BASELINE`。
- `STITCH_READY` 最低条件改为 `PAGE_VISUAL_BASELINE + FORMAL_DRAFT + VISUAL_SPEC + COMPONENT_REFERENCE + TOKEN_READY + ASSET_READY`。
- `VISUAL_MASTER` 只保留给“标准母版”清单中的可继承页面。

## 3. 页面包批量修复范围

已批量修复以下页面包 Markdown 中的术语：

| 范围 | 修复内容 |
| --- | --- |
| `02-主标签页/*` | `主视觉母版`、`VISUAL_MASTER` 等页面级误称改为 `当前页面视觉基准` / `PAGE_VISUAL_BASELINE` |
| `03-书架链路/*` | 页面级默认图改为 `PAGE_VISUAL_BASELINE`；本地书导入保留 `PAGE_VISUAL_CANDIDATE` |
| `04-阅读链路/*` | 派生页和覆盖层页面改为页面视觉基准；阅读入口改为引用参考 |
| `05-设置链路/*` | 设置二级页面自己的图改为页面视觉基准 |

本次共更新页面包 Markdown `90` 个。

## 4. 当前校验

页面包范围内已无以下残留：

- `主视觉母版`
- `当前整页视觉母版`
- `VISUAL_MASTER_CANDIDATE`
- `MISSING_VISUAL_MASTER`
- `候选视觉母版`
- `正式视觉母版`

页面包范围内也已无直接使用的状态：

- `` `VISUAL_MASTER` ``
- `VISUAL_MASTER +`
- `/ VISUAL_MASTER`

## 5. 后续约束

以后新增或补充页面时按以下规则命名：

1. 页面自己的当前截图：写 `PAGE_VISUAL_BASELINE`，中文写“当前页面视觉基准”。
2. 未确认的生成图：写 `PAGE_VISUAL_CANDIDATE`，中文写“页面视觉候选”。
3. 引用其他页面截图：写 `REFERENCE_BASELINE`，中文写“引用参考”。
4. 只有全局标准母版清单中的可继承页面才能写 `VISUAL_MASTER`。
5. 历史审计文件可以保留旧词作为历史记录，但不得作为当前生成输入。
