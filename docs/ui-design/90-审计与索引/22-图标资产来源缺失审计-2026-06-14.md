# 图标资产来源缺失审计

日期：2026-06-14
范围：`docs/ui-design`

## 结论

当前主要缺口不是“页面文字稿没有说明图标语义”，而是“没有工程化图标资产来源”。多数页面已经写了图标语义、尺寸、线宽或触控区，但没有绑定可复用 SVG path、图标文件路径或图标注册表。

这会导致生成 HTML 或 Stitch 复原时退化为：

- 用中文字符代替图标。
- 用默认字体符号代替图标。
- 用第三方默认图标风格代替当前设计图中的粗线圆角图标。
- 同一页面内图标尺寸、描边、槽位不一致。

## 当前文字稿是否足够

足够的部分：

- 多数页面已经说明“图标语义是什么”。
- 一些页面已经说明图标尺寸、线宽、触控区。
- 阅读控制层已经说明 `03-当前视觉基线.png` 是视觉母版，并约束图标尺寸/描边。

不足的部分：

- 没有统一图标资产目录。
- 没有图标注册表。
- 没有把页面语义绑定到具体 SVG path。
- 没有区分 `AppShell` 图标集和 `ReaderControl` 图标集的资产来源。
- 没有为 Stitch/HTML 生成提供可直接复制的内联 SVG。

## 建议补齐方式

新增一个全局图标资产包，而不是在每个页面文字稿里重复描述图标：

```text
docs/ui-design/01-全局规范/图标资产/
  00-图标资产注册表.md
  app-shell/
    search.svg
    more.svg
    back.svg
    sort.svg
    settings.svg
  reader-control/
    back.svg
    source-switch.svg
    more.svg
    search.svg
    auto-page.svg
    replace.svg
    catalog.svg
    read-aloud.svg
    appearance.svg
    reading-settings.svg
    brightness.svg
    font-size.svg
    system-brightness.svg
```

注册表需要写清：

- 图标语义名。
- SVG 文件路径。
- 适用页面包。
- 视觉尺寸。
- 线宽。
- 图标槽尺寸。
- 颜色 token。
- 是否允许复用到 App Shell 或只限阅读控制层。

页面文字稿只需要引用注册表，例如：

```text
图标资产：`01-全局规范/图标资产/00-图标资产注册表.md#reader-control`
```

这样页面稿保持正向描述：“使用哪个图标资产、放在哪个槽位、尺寸是多少”，不需要靠禁止项枚举所有错误做法。
