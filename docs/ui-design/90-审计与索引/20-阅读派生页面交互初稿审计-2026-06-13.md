# 阅读派生页面交互初稿审计

日期：2026-06-13
对象：`.stitch/designs/阅读派生页面交互初稿.html`
审计截图：`.stitch/designs/阅读派生交互审计接触表-2026-06-13.png`

## 结论

交互 HTML 可以作为“状态机验证稿”保留，但不能作为正式交互设计稿。当前可证明 26 个页面状态可以合并到一个 HTML 中，并且核心状态可达；但真实入口、二级跳转、页面包覆盖和信息层级仍需修复。

## 验证通过项

- 四个快捷模块可切换：目录、朗读、界面、设置。
- 目录快捷模块可进入目录覆盖层和完整目录页。
- 朗读快捷模块可启动运行胶囊。
- 内容搜索覆盖层可进入完整搜索页。
- 内容替换覆盖层可进入规则管理页，规则管理页可进入规则编辑页。
- 自动翻页覆盖层可启动运行胶囊，也可进入完整设置页。

## 主要问题

### P1-001：搜索、替换、自动翻页入口是测试入口，不是真实产品入口

位置：`scripts/generate_reading_interactive_prototype.py:158`、`scripts/generate_reading_interactive_prototype.py:167`

当前在正文和控制层之间直接放了 `搜正文 / 替换 / 自动翻页` 三个按钮。它们用于串联测试，但不符合阅读控制层正式入口设计。正式稿中这些入口应从顶部更多菜单、长按正文菜单或控制层派生入口进入。

影响：如果继续基于此稿精修，Stitch 或实现侧会误以为阅读页常驻三枚快捷按钮，破坏沉浸阅读母版。

### P1-002：阅读设置首页的二级入口没有真实跳转

位置：`scripts/generate_reading_interactive_prototype.py:107`

`阅读设置` 完整页列出了 `屏幕与显示 / 翻页与手势 / 阅读辅助 / 进度与信息`，但这些行只是普通设置行，没有 `data-page` 跳转。当前测试是从快捷模块直接进入子页，绕过了完整设置首页的二级导航。

影响：页面层级表达不完整，无法验证完整设置页的真实返回和二级页面跳转。

### P1-003：已规划页面状态覆盖不完整

位置：`scripts/generate_reading_interactive_prototype.py:98`

交互稿缺少或未正确串联以下已规划状态：

- `阅读设置-阅读辅助`
- `阅读外观-版式高级` 没有从阅读外观快捷模块进入，当前被 `阅读设置` 快捷项中的 `段落缩进` 间接触发。
- `阅读设置-首页` 到各子页的二级跳转缺失。

影响：单页初稿存在，但合并交互稿不能完整证明这些状态的真实交互关系。

### P2-001：运行胶囊状态与完整页状态没有明确互斥策略

位置：`scripts/generate_reading_interactive_prototype.py:242`、`scripts/generate_reading_interactive_prototype.py:254`

当前启动朗读或自动翻页后，再进入完整页时，运行胶囊不会被显式关闭或挂起，只是被完整页层级遮住。返回阅读页后运行胶囊仍在。

影响：这可能是合理产品策略，也可能是状态泄露。需要明确：进入完整设置页时运行态是否继续、暂停、隐藏或转为顶部状态。

### P2-002：章节行点击只关闭覆盖层，没有表达跳章反馈

位置：`scripts/generate_reading_interactive_prototype.py:10`

目录覆盖层中的章节行点击后只执行 `data-close`，没有更新当前章节、阅读进度或目录选中态。

影响：目录功能的核心交互没有被验证，只验证了覆盖层关闭。

## 建议修复顺序

1. 移除正文上方常驻测试入口，改为从顶部更多菜单或长按菜单打开搜索、替换、自动翻页。
2. 为完整页增加真实二级路由：阅读设置首页 -> 子设置页，阅读外观快捷模块 -> 版式高级。
3. 补齐 `阅读设置-阅读辅助` 状态。
4. 定义运行胶囊与完整页的互斥策略。
5. 为目录章节点击增加当前章节更新和选中反馈。
