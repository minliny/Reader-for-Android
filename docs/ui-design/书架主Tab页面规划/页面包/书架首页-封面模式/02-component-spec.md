# 书架首页封面模式 组件规格稿

## Design Token 使用原则

封面模式和列表模式必须复用同一套书架组件 token。当前图片存在视觉漂移，Stitch 不得逐图照抄。

权威视觉规格：`../../18-书架首页组件视觉规格.md`

本文件只保留封面模式页面包中的组件使用说明；具体 token 数值、响应式断点和跨模式一致性约束以权威视觉规格为准。

## 页面背景

- 使用统一暖白背景 token。
- 不允许不同状态或不同模式使用相近但不同的背景色。

## 顶部栏

- 高度随安全区自适应。
- 标题左对齐。
- 搜索和更多为图标按钮，触控区不小于 44dp。
- 图标线宽、尺寸、颜色在所有书架状态中一致。

## 分组 Chip

- 单行横向滚动。
- 高度固定为同一 token。
- 左右 padding 使用同一 token。
- 激活态和未激活态必须有明确颜色、边框、文字样式。
- chip 宽度由内容决定，不强行等宽。

## 顶部信息卡片

`继续阅读` 和 `最近更新` 使用同一 `PanelCard` token：

- 背景一致。
- 圆角一致。
- 边框粗细一致。
- 阴影层级一致。
- 内边距一致。

在窄屏下，两个卡片可以纵向堆叠；在常规手机宽度下可以并排展示。

## 书籍封面卡片

- 卡片背景、圆角、边框、阴影使用统一 `BookCard` token。
- 封面保持固定宽高比，建议书籍封面比例接近 2:3。
- 封面圆角使用小于卡片圆角的 token。
- 书名最多两行；作者最多一行；章节最多一行。
- 进度条固定高度、圆角、轨道色、填充色。

## 网格列数

网格列数按容器宽度计算：

- 窄屏：2 列。
- 常规手机：2 或 3 列，取决于最小卡片宽度。
- 大屏/折叠屏：最多 4 列；超过 4 列时增加左右留白，不继续压缩卡片。

不得把候选图的三列视为固定规则。

## 底部主导航

- App Shell 组件复用。
- 高度、圆角、阴影、图标尺寸、文字字号固定。
- 封面模式和列表模式不得分别生成两套底栏。
