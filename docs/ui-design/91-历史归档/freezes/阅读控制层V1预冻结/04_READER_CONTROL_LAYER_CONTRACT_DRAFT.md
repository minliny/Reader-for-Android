# 阅读控制层合同草案

## 1. 基准画布

基准画布确认为 `390 x 884`。

- 高分辨率 PNG 按相对比例映射到该画布。
- 完整参数见 `docs/ui-design/91-历史归档/demos/阅读控制层演示/04-2-complete-relative-parameter-spec.md`。
- 状态：`DEFINED`

## 2. 安全区说明

设计图按完整屏幕处理。

- 顶部弱状态为 App 自绘阅读状态信息。
- 底部控制层为悬浮面板，底部留白取 `max(1.13% 画布高, bottomInset)`。
- 状态：`DEFINED`

## 3. 阅读正文层规则

- 阅读正文层是所有阅读覆盖层的不可变基础层。
- 控制层、快捷区、浮层出现时不得重排、挤压或移动正文。
- 沉浸态不显示应用主底部导航。
- 沉浸态不显示按钮。
- 沉浸态不显示贴屏幕底部的阅读细进度条。

若现有图仍包含贴屏幕底部的阅读细进度条，标记为：`IMAGE_CONFLICT_NEEDS_REDRAW`。

## 4. 控制层 overlay 规则

- 控制层叠加在阅读基础页之上。
- 控制层包含顶部控制栏、底部控制面板、右侧亮度条、章节进度区和底部四主按钮。
- 控制层是固定模块容器，不是多个互相替换的独立底表。
- 点击四主按钮时，只替换左侧主内容区。
- 底部控制面板四角完整，悬浮于正文页上，不与屏幕底边相连。
- 唯一视觉和几何基底为 `04-2-reader-controls-opaque-icon-aligned.png`。

## 5. 顶部控制栏规则

- 顶部控制栏位于阅读页上方区域。
- 内容包括返回、书名/来源章节、换源、三点菜单。
- 顶部控制栏位置在所有控制层状态中应保持一致。

状态：`DEFINED_BY_04_2`

## 6. 底部章节进度条规则

- 控制层内部章节进度条应位于底部工具栏上方。
- 章节进度条用于当前章节内定位。
- 它不同于沉浸页贴屏幕底部的阅读细进度条。
- 沉浸页底部阅读细进度条不应出现。

冲突标记：`IMAGE_CONFLICT_NEEDS_REDRAW`

## 7. 底部工具栏规则

- 底部工具栏固定在控制层底部区域。
- 四主按钮顺序固定为：`目录 / 朗读 / 界面 / 设置`。
- 四主按钮不得被快捷模块移动、隐藏、复制或重画到新位置。

## 8. 快捷区面板规则

- 快捷区只替换阅读控制层左侧主内容区。
- 快捷区不挤压正文。
- 快捷区之间互斥。
- 右侧亮度条保持原位置。
- 底部四主按钮保持原位置。

## 9. 亮度浮层规则

- 亮度条属于控制层右侧常驻或临时浮层。
- 亮度调整不参与正文布局。
- 系统亮度状态应弱化或禁用滑条，并高亮 `系统`。

状态：`DEFINED_BY_04_2`

## 10. 章节进度浮层规则

- 拖动章节进度条时显示临时位置提示。
- 松开后跳转正文位置。
- 提示层不参与正文布局。

状态：`DEFINED_BY_DERIVED_PAGE_SPEC`

## 11. 全页弹层规则

- 目录覆盖层、内容搜索覆盖层、内容替换覆盖层、自动翻页覆盖层打开时，应覆盖或收起阅读控制层。
- 不允许控制层底部工具栏与全页弹层混乱叠加。
- 完整管理页不保留阅读控制层底部四主按钮和右侧亮度条。

## 12. 不允许改动项

P0 不允许改动：

- 阅读正文层不得因控制层出现而重排。
- 顶部控制栏位置不得在控制层状态间漂移。
- 底部工具栏位置不得下移。
- 四主按钮数量和顺序不得改变。
- 控制层内部章节进度条必须位于底部工具栏上方。
- 沉浸页不显示贴屏幕底部阅读细进度条。
- 快捷区不挤压正文。
- 亮度条和章节进度提示不参与正文布局。
- 底部控制层四角和底部悬浮留白不得取消。
- 所有直接相关页面必须继承 `reader-control-derived-pages.md`。
