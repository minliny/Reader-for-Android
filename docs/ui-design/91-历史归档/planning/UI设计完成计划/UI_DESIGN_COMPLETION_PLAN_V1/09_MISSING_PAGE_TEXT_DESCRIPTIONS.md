## 第一批：App 主入口缺页

- `书架`

- `RSS`
- `设置`

当前已暂定主页面：

- `书架首页`：已有 `01-书架首页-封面模式.png`、`02-书架首页-列表模式.png`。
- `发现页`：暂定采用 `41-1-discovery-source-category.png`。
- `RSS/订阅首页`：暂定采用 `42-rss-home.png`。
- `设置首页`：暂定采用 `43-settings-home.png`。

当前还缺的主页面：

- 无。

不属于主页面但后续仍需规划的功能页：

- `书籍搜索页`
- `书籍信息页`
- `阅读页`
- `书源管理首页`
- `阅读内完整设置首页`

本清单只规划底栏主页面，不展开次级页面、弹层、loading/empty/error 状态。

### 发现页

用途：承载站内发现、分类、榜单、推荐和可探索内容，是书架之外的内容入口。

入口：底部主导航 `发现`。

默认态内容：顶部标题 `发现`，搜索入口，信息源选择，来源内分类切换，当前来源推荐分区，来源状态条，来源榜单/更新列表。底栏选中 `发现`。

状态：后续功能设计阶段再补加载态、空状态、网络错误态。空状态用于暂无推荐或发现源未配置；错误态需要重试按钮。

设计重点：不使用阅读控制层结构；视觉密度和书架页一致，避免营销页式大 hero。推荐内容来自书源和订阅源，不同来源的分类不同，因此必须先选择信息源，再展示该来源自己的分类。

当前阶段保存状态：

- 暂定只规划一个发现页主页面：`docs/ui-design/06-发现链路/发现首页候选图/图片/01-发现首页信息源分类.png`。
- 信息源选择、更多分类、顶部更多菜单、来源检测、分类切换状态、订阅栏目切换状态、loading/empty/error 均暂不补图。
- 次级页面和实际功能后续再独立设计开发。

### RSS/订阅首页

用途：承载订阅源内容流，是底部主导航 `RSS` 对应的主页面。它展示用户已添加订阅源的最新内容，不等同于发现页里的 `订阅` 信息源筛选。

入口：底部主导航 `RSS`。

默认态内容：顶部标题 `RSS` 或 `订阅`，顶部右侧搜索/更多，订阅源筛选区，最新条目列表，订阅源分组入口，未读/收藏/稍后读快捷筛选。底栏选中 `RSS`。

设计重点：内容以文章、更新条目、书单或来源推送为主；列表项应显示来源、标题、摘要、更新时间、未读状态。不要做成书籍推荐页，也不要和发现页的信息源选择重复。

当前阶段保存状态：

- RSS/订阅页只规划一个主页面。
- 订阅源管理、分组管理、详情页、添加订阅、loading/empty/error 等暂不补图。
- 次级页面和实际功能后续再独立设计开发。

### 设置首页

用途：作为底部主导航 `设置` 对应的主页面，承载全局设置入口和常用管理入口。本项目不规划账户功能，也不保留 `我的` 底栏入口；原本属于 `我的` 的阅读统计、缓存、备份、关于、反馈等能力统一放入设置页。

入口：底部主导航 `设置`。

默认态内容：顶部标题 `设置`，常用入口卡片，阅读偏好入口，书源管理入口，RSS/订阅管理入口，同步/备份入口，隐私与权限入口，关于与反馈入口。底栏选中 `设置`。

设计重点：这是 App 级设置入口，不是阅读控制层里的 `设置快捷模块`，也不是阅读内完整设置页。页面应偏管理型，信息密度紧凑，不放阅读页亮度条或控制层四按钮。

布局规划：

- 顶部区域：状态栏、标题 `设置`、右上搜索图标、右上更多图标。
- 概览卡片：展示本地阅读数据和维护状态，例如 `本地书籍`、`订阅源`、`书源可用`，仅展示本地统计，不出现账户头像或登录入口。
- 常用管理：`书源管理`、`RSS/订阅管理`、`阅读偏好`、`缓存管理`，使用 2 x 2 紧凑入口卡。
- 设置分组列表：
  - `通用`：语言、启动页、主题跟随系统。
  - `阅读`：默认阅读行为、阅读内完整设置入口。
  - `书架与搜索`：书架展示、搜索范围、历史记录。
  - `同步与备份`：本地备份、WebDAV、导入导出。
  - `隐私与权限`：存储权限、通知权限、隐私清理。
  - `关于与反馈`：版本信息、更新日志、反馈入口。
- 底部主导航：`书架 / 发现 / RSS / 设置`，当前高亮 `设置`。

当前阶段保存状态：

- 设置页只规划一个主页面。
- 暂定采用 `docs/ui-design/05-设置链路/设置首页历史图/图片/01-设置首页管理入口.png`。
- 书源管理、RSS/订阅管理、阅读内完整设置、同步备份、关于反馈等只作为入口，不在本阶段展开图。
- loading/empty/error 状态暂不补图。

### App 全局设置首页

用途：管理 App 级别设置，不属于阅读页内设置。当前与底栏 `设置首页` 合并为同一主页面。

入口：底部 `设置` 主页面。底部主导航不设置 `我的`，因此 `设置首页` 与 `App 全局设置首页` 合并为同一个主页面。

默认态内容：标题 `设置`，分组入口：通用、书架与搜索、书源、同步/备份、隐私、安全、通知、关于。每行包含标题、简短当前值或说明、右侧进入箭头。

状态：需要默认态、加载态、错误态。权限类设置需要 permission_required 状态。

设计重点：必须和阅读内 `完整阅读设置页` 区分；不展示阅读控制层底部四主按钮和右侧亮度条。

### 书源管理首页

用途：管理可用书源，支持搜索、筛选、启用/禁用、检测、新增。

入口：`设置页 > 书源管理`、`App 全局设置 > 书源`。

默认态内容：标题 `书源管理`，搜索栏，分组/状态筛选，书源列表，行内启用开关，行内检测入口，新增书源按钮。列表项显示名称、域名/地址、分组、启用状态、最近检测结果。

状态：需要默认态、加载态、空状态、错误态、检测中、检测结果、检测失败。

设计重点：这是完整管理页，不是阅读页顶部 `换源` 底表；不保留阅读控制层结构。

## 第二批：阅读内完整设置缺页

### 阅读设置首页

用途：从阅读控制层 `设置快捷模块 > 完整设置` 进入，聚合阅读行为设置。

内容：屏幕与显示行为、翻页与手势、进度与信息显示、阅读辅助、预设/恢复默认。顶部返回上一层。

状态：默认态、保存成功、保存失败、恢复默认确认。

### 阅读显示设置

用途：管理阅读中状态栏、安全区、屏幕方向、屏幕超时等显示行为。

内容：屏幕方向、阅读时保持屏幕常亮、状态栏显示、底部信息显示、横竖屏策略。

状态：默认态、权限不足态、保存失败态。

### 阅读行为设置

用途：管理翻页和手势行为。

内容：音量键翻页、点击热区、左右滑动、单手翻页、长按动作、误触保护。

状态：默认态、冲突提示态、恢复默认确认。

### 阅读辅助设置

用途：管理辅助阅读功能，不包含朗读参数和外观主题。

内容：章节进度提示、阅读统计显示、自动缓存行为、断点恢复、章节异常提示。

状态：默认态、保存成功、保存失败。

### 阅读预设/恢复默认

用途：管理阅读行为预设，并提供恢复默认。

内容：预设列表、当前应用标记、保存当前设置为预设、删除预设、恢复默认确认。

状态：默认态、空状态、删除确认、恢复默认确认、操作成功/失败。

## 第三批：书源管理完整链路缺页

### 书源列表

用途：展示所有书源及启用状态。

内容：搜索、筛选、分组、列表、启用开关、检测入口、新增按钮。

状态：默认态、加载态、空状态、错误态。

### 书源详情

用途：查看单个书源的配置摘要和健康状态。

内容：名称、地址、分组、启用状态、规则摘要、最近检测、编辑、删除、检测、调试信息入口。

状态：默认态、检测中、检测结果、检测失败、删除确认。

### 新增书源

用途：创建新书源。

内容：名称、地址、分组、规则输入或导入、保存、检测。

状态：默认态、表单校验错误、保存中、保存成功、保存失败。

### 编辑书源

用途：修改已有书源。

内容：与新增页一致，但显示当前值，并提供保存、检测、删除。

状态：默认态、校验错误、保存中、保存成功、保存失败、删除确认。

### 书源检测

用途：检查目录、章节定位、正文读取和内容匹配。

内容：检测进度、检测步骤、当前书源信息、取消按钮。

状态：检测中、检测通过、检测失败。

### 检测结果

用途：展示书源检测的可用性结论。

内容：通过/失败摘要，目录匹配、章节读取、正文质量、耗时，应用/返回/查看日志按钮。

状态：通过、部分通过、失败。

### 调试信息与错误日志

用途：帮助定位书源解析和网络问题。

内容：请求地址、响应摘要、解析步骤、错误堆栈摘要、复制按钮、清空按钮。

状态：默认态、空日志、复制成功、清空确认。

## 第四批：通用状态模板缺页

### Loading

用途：用于列表、详情、搜索、设置保存、书源检测等等待状态。

内容：轻量加载指示、当前动作文案，必要时保留页面骨架。

### Empty

用途：用于书架无书、发现无内容、书源为空、搜索无结果。

内容：空状态标题、简短原因、主操作按钮。

### Error / Network Error

用途：用于网络失败、接口失败、解析失败。

内容：错误标题、原因摘要、重试按钮，必要时提供查看日志入口。

### Permission Required

用途：用于存储、通知、系统 TTS 或外部页面权限。

内容：权限说明、授权按钮、稍后处理按钮。

### Delete Confirm

用途：用于删除书源、删除预设、清空日志、移除字体/主题。

内容：对象名称、风险说明、取消、确认删除。

### Operation Success / Operation Failed

用途：用于保存、导入、检测、复制、恢复默认等操作反馈。

内容：轻提示或小面板，成功/失败状态、下一步操作。
