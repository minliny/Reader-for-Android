逐图文字稿入口为
[`../90-审计与索引/01-图片文字稿总册.md`](../90-审计与索引/01-图片文字稿总册.md)。页面级规格负责产品规则，

的当前权威规格和逐图总册中的资产状态为准。

## 全局决策

- 主视觉方向：柔和卡片 + 分区式信息块。
- 设置与完整管理页的视觉参数以 `SETTINGS_PAGE_DESIGN_SPEC.md` 为准，不继承阅读控制层
  的 pill 顶栏、面板圆角和粗图标参数。
- 主页面仅指底部主导航一级入口页面，当前固定为 `书架 / 发现 / RSS / 设置`。
- 阅读基础页：正文 + 四角信息层。透明点击热区属于交互规格。
- 手机竖屏统一逻辑基准为 `390×884`；高分辨率导出图按相对比例映射。
- 阅读覆盖层必须叠加在阅读页上，不重排、不重设正文底层。
- 书架卡片不展示书源。
- 书籍搜索与书内内容搜索是两条独立流程。
- 目录章节行只展示缓存与书签状态，不展示未读标记。
- 每个页面规格都应包含交互矩阵，说明按钮行为与目标页面。
- 跨页面关系统一记录在 `page-flows.md`，避免静态 UI 示意图丢失导航上下文。
- 新增 UI 示意图不得覆盖既有主图。补充状态图统一使用 `原编号-序号-页面说明.png`，并在 `91-历史归档/drafts/旧全局草稿目录/README.md` 与对应规格文档中登记。
- 阅读控制层以 `READER_CONTROL_LAYER_SPEC.md`、`READER_CONTROL_ACTION_FLOWS.md`、
  `READER_CONTROL_RESPONSIVE_RULES.md`、`READER_CONTROL_GEOMETRY_SPEC.md`、
  `READER_CONTROL_IMAGE_USAGE.md` 五份规格为权威。
- 响应式模式选择以 `READER_CONTROL_RESPONSIVE_RULES.md` 为权威；侧置悬浮的专项设计
  以 `READER_CONTROL_TABLET_SPEC.md` 为准，不得反向覆盖手机控制层结构。
- 旧控制层规格仅作历史专项参考；当前结构、动作流、响应式规则、统一几何和图片使用关系
  均以五份 `READER_CONTROL_*` 权威规格为准。
- `04-1` 定义模块关系，`04` 定义默认功能，`04-2` 定义当前视觉和固定位置。
- 点击 `目录 / 朗读 / 界面 / 设置` 时只切换左侧主内容区，模块派生图不得覆盖 `04-2`。
- `34-reader-appearance-main-panel-standard.png` 已废弃，不得作为任何新规格依据。
