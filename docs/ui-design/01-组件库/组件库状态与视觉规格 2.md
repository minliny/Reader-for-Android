# 组件库状态与视觉规格

## 文档定位

本文先定义“组件库规格文档应该包含什么”，再进入具体组件规则。

当前阶段不直接把单次审计问题写成答案，也不直接生成 Figma Make Prompt。所有具体视觉规则、状态规则和组件转化规则，必须先归入本文的结构，再按组件逐项补齐。

## 填写顺序

本文必须按以下顺序维护：

1. 定义文档边界和信息来源。
2. 定义组件库总目录和组件分类。
3. 定义所有组件都必须遵守的通用规则。
4. 定义单个组件规格模板。
5. 按组件逐个补齐规格。
6. 把审计发现的问题归入对应组件的待决策项。
7. 决策完成后，再把待决策项转成明确规格。
8. 最后才生成 Figma Make Prompt 或执行 Figma Design 组件转化。

如果发现问题时直接写解决方案，视为顺序错误。

## 文档边界

本文负责定义：

- 组件库的组件范围。
- 每个组件需要哪些 variant properties。
- 每个状态必须如何被识别。
- 每个组件的结构、槽位、嵌套关系和可 override 内容。
- 组件之间的依赖关系。
- 视觉 token、圆角、线宽、分割线、阴影、loading 等跨组件规则。
- Figma Make 生成稿的验收标准。
- Figma Design 真实组件转化的验收标准。

本文不负责：

- 重新设计完整页面。
- 直接描述营销式展示图。
- 直接替代页面文字稿。
- 直接规定 Compose 实现细节。
- 直接修改 Figma 画布。
- 在未完成组件规则前生成 Prompt。

## 信息来源与优先级

后续填写规格时，必须按以下优先级取证：

1. 已确认的本地页面文字稿。
2. 已确认的本地 UI 设计图。
3. 当前 Figma Design 中已经验收通过的真实组件。
4. Figma Make 生成的 `UI 组件库` 视觉稿。
5. 本轮人工审计问题。

当信息冲突时，不直接合并，应记录到“待决策项”。

## 组件库总目录

组件库规格至少应包含以下分区。

### 1. Foundations

必须定义：

- Color tokens。
- Typography tokens。
- Spacing tokens。
- Radius tokens。
- Shadow tokens。
- Stroke / outline rules。
- Separator rules。
- Loading / empty / error / unknown state language。

### 2. App Shell

必须定义：

- `AppFrame`
- `AppTopBar`
- `BottomNav`
- `BottomNavItem`
- `IconButton`

本轮如果不处理这些组件，也要明确它们是已有组件，不在当前转化范围。

### 3. Basic Controls

必须定义：

- `Chip`
- `SearchBar`
- `SegmentedControl`
- `ProgressBar`
- `ProgressSlider`

重点要定义 value、selected、focused、typing、loading、disabled 等状态差异。

### 4. Cards & Rows

必须定义：

- `Card`
- `SettingIcon`
- `SettingRow`
- `ListRow`
- `BookCover`
- `BookCard`
- `SearchResultItem`
- `SourceListItem`
- `RSSSourceItem`
- `NoteCard`

重点要定义嵌套关系、slot、progress 显示策略、selected/separator 几何规则。

### 5. Sheets & Panels

必须定义：

- `BottomSheet`
- `PanelHeader`
- `ReaderPanel`
- `TocPanel`
- `AppearancePanel`
- `TTSPanel`
- `AnnotationPopover`

重点要定义外层容器、内部 panel、内部控件的圆角层级和覆盖关系。

### 6. States

必须定义：

- `EmptyState`
- `ErrorState`
- `LoadingState`
- `OfflineState`
- `PermissionState`

重点要定义不同状态之间的语义边界，避免 loading、empty、error、unknown 混淆。

## 单组件规格模板

每个组件必须按同一模板填写。

### 组件名称

记录真实组件名，必须与 Figma Component Set 命名一致。

### 组件职责

说明该组件解决什么问题，不解决什么问题。

### 使用场景

列出该组件会在哪些页面或面板出现。

### 不使用场景

列出容易误用但不应使用该组件的场景。

### 结构 Anatomy

必须列出：

- 外层容器。
- leading 区域。
- content 区域。
- trailing 区域。
- 状态标识区域。
- 可选区域。
- 不允许存在的区域。

### Nested Components

必须说明：

- 组件内部是否必须嵌套其他 component instance。
- 哪些内容必须通过 instance swap 表达。
- 哪些内容不得通过巨大 variant matrix 表达。

### Variant Properties

必须列出：

- property 名称。
- property 可选值。
- 每个 property 的用途。
- 哪些组合必须存在。
- 哪些组合禁止存在。

### States

必须列出：

- default。
- selected / active。
- pressed。
- disabled。
- loading。
- empty。
- error。
- unknown。

不是每个组件都必须支持所有状态，但必须明确“不支持”的原因。

### Content Slots

必须定义：

- 标题是否必填。
- 副标题是否必填。
- metadata 是否必填。
- value 是否可 override。
- 百分比是否必显。
- badge 是否可选。
- 图标是否可 swap。

### Geometry

必须定义：

- 宽度策略。
- 高度策略。
- padding。
- gap。
- 对齐方式。
- 分割线起止位置。
- 阴影边界。
- 圆角规则。

### Visual Tokens

必须定义：

- 背景色。
- 文字颜色。
- 边框色。
- 进度色。
- 图标颜色。
- 阴影。
- opacity。

### Typography

必须定义：

- title 字号/行高。
- body 字号/行高。
- metadata 字号/行高。
- caption 字号/行高。
- 最大行数和截断规则。

### Interaction

必须定义：

- 点击行为。
- 长按行为。
- 拖动行为。
- 禁用行为。
- loading 时是否允许点击。
- 失败时是否保留上下文。

### Acceptance Criteria

必须定义：

- Figma Make 输出验收项。
- Figma Design 组件转化验收项。
- 页面实例替换验收项。
- 视觉差异回滚条件。

### Open Decisions

必须记录尚未决定的问题，不能把未确认内容写成规则。

## 已定义的跨组件规则

以下规则是当前组件库的基础系统定义。后续组件规格不得绕过这些规则。

### Icon System

#### 尺寸

| 项 | 规则 |
| --- | --- |
| 图标容器 | 默认 `36 x 36` |
| 图形绘制区 | `20 x 20`，复杂图标可用 `24 x 24` |
| 行内 leading icon | 容器居中，content 起点不随图标宽度变化 |
| toolbar icon | 图形绘制区 `24 x 24`，触控区不小于 `48 x 48` |

同一 Component Set 内不得混用 `20` 和 `24` 两套绘制区；确需两套时必须拆成不同 size variant。

#### 线宽

| 项 | 规则 |
| --- | --- |
| 标准线宽 | 视觉重量等同 `2px` |
| 可接受范围 | `1.75px - 2px` |
| 禁止范围 | `1px` 极细线、`2.5px+` 重线 |
| cap / join | 优先 round cap / round join |

生成稿中如果同一组图标出现明显粗细跳变，视为组件源失败，不进入转化。

#### 颜色状态

| state | icon | container |
| --- | --- | --- |
| `default` | `text.secondary` | `bg.surfaceAlt` |
| `active` | `brand.primary` | `brand.primarySoft` |
| `disabled` | `text.tertiary`，60% opacity | `bg.surfaceAlt`，60% opacity |
| `danger` | `danger` | 透明或 8% danger tint |

#### 语义规则

- 明确动作可以使用图标：搜索、返回、更多、设置、书签、缓存、同步。
- 图标语义不明确时，优先使用文字按钮，不使用自造抽象图形。
- Setting 类图标必须通过 `SettingIcon` instance swap 表达，不为每个设置项创建独立 row variant。
- 占位图标必须标记为 `needs-icon-review`，不得伪装为已确认图标。

### Progress System

#### 进度来源

| 场景 | 数据源 | 显示口径 |
| --- | --- | --- |
| 书籍阅读进度 | book progress | 整本书百分比 |
| 当前章节进度 | chapter progress | 本章百分比 |
| 目录当前章 | current chapter progress | `当前 · xx%` |
| loading | 无稳定进度 | 不显示假百分比 |
| unknown | 数据不可判断 | 显示中性未知，不显示假进度 |

#### Variant 属性

`ProgressBar` 必须支持：

- `value=0 / 25 / 50 / 75 / 100 / unknown`
- `size=compact / full`
- `state=default / loading / disabled`

`ProgressSlider` 必须支持：

- `state=default / dragging / disabled`

#### 几何

| size | 宽度 | 高度 | 用途 |
| --- | --- | --- | --- |
| `compact` | `90-120` | `3` | BookCard 封面模式 |
| `full` | `216-326` | `5` | 列表、面板、阅读进度 |

轨道和填充圆角都等于高度的一半。

#### Value 差异

| value | fill 比例 | 文案策略 |
| --- | --- | --- |
| `0` | `0%`，最多保留 2px 起点 | 可显示 `0%` |
| `25` | `25%` | 必须能看出四分之一 |
| `50` | `50%` | 必须能看出一半 |
| `75` | `75%` | 必须能看出四分之三 |
| `100` | `100%` | 显示 `100%` 或 `已读完` |
| `unknown` | 无 fill 或中性虚线 | 显示 `进度未知` 或隐藏百分比 |

如果 `25 / 50 / 75` 肉眼看起来几乎一样，视为失败。

#### 百分比文案绑定

- `progress=partial` 的 BookCard 必须同时显示百分比文案和 ProgressBar。
- 封面模式、列表模式、推荐模式都遵守同一 progress 口径。
- `progress=none` 才允许隐藏百分比和进度条。
- `progress=loading` 不得显示旧百分比。

### Loading System

#### 状态边界

| 状态 | 含义 | 视觉规则 |
| --- | --- | --- |
| `loading` | 数据正在获取 | skeleton 或明确 `加载中`，保留结构 |
| `empty` | 获取完成但无数据 | 空原因 + 下一步 |
| `error` | 获取失败 | 失败原因 + 重试 |
| `unknown` | 当前没有可判断值 | 中性弱标识，不伪装加载 |

#### Skeleton 规则

- skeleton 使用 `bg.surfaceAlt` 或更浅同色，不使用彩色渐变。
- skeleton 必须对应真实内容结构，不能只放一块无意义空白。
- BookCover loading 必须在封面区域内显示 skeleton 或 `加载中`。
- Progress loading 不显示 fill 百分比。
- 列表局部 loading 不清空顶部栏、底部导航或稳定父结构。

#### 点击规则

- loading 中的组件默认不可点击。
- 如果保留旧内容刷新，旧内容可继续点击，但 loading 标识必须局部显示。
- error 后可点击重试；unknown 不提供重试，除非业务定义可重新获取。

### Radius System

#### 圆角层级

| 层级 | 圆角 | 适用 |
| --- | --- | --- |
| `BookCover` | `8` | 书封 |
| `InnerControl` | `10-14` | chip、slider、badge、局部按钮 |
| `BookCard` | `16` | 书籍卡片 |
| `ReaderPanel` | `16-20` | 底表内部内容模块 |
| `Card` | `22` | 页面内容卡 |
| `FloatingControl` | `18-24` | 顶部浮动控制卡 |
| `BottomSheet.stuck` | top `28`，bottom `0` | 贴住屏幕底边 |
| `BottomSheet.floating` | `28` all corners | 四周露出背景 |
| `BrightnessRail` | `20-24` | 自包含竖向 rail |

#### 父子关系

- 子容器圆角不得大于父容器圆角。
- 贴底 BottomSheet 如果底部露出背景，必须改为 floating 模式并使用四角圆角。
- Reader 控制层中的亮度 rail 必须是独立圆角容器，不能只用竖线表达。
- 内部 row group 不得和外层 sheet 使用同等强边界，避免卡片套卡片。

### Separator / Shadow System

#### 分割线

| 场景 | 起点 | 终点 |
| --- | --- | --- |
| SettingRow | content x 起点 | 右侧 padding 内 |
| ListRow | content x 起点 | 右侧 padding 内 |
| ChapterRow | row 左 padding 或 title 起点，按面板规格统一 | 右侧 padding 内 |

分割线必须和内容起点对齐，不得跟随图标宽度漂移。

#### selected 状态

- selected 背景覆盖完整可点击行区域。
- selected 行内部不得显示穿过背景的分割线。
- selected 上下相邻分割线要么隐藏，要么与普通行分割线同 x 对齐。
- selected 阴影如果存在，必须贴合 selected 背景圆角边界。
- 普通 row 不使用明显投影；投影优先属于 card、sheet、popover。

### Chapter Status System

#### 状态优先级

当一个章节同时满足多个状态时，优先级为：

1. `current`
2. `error`
3. `loading`
4. `bookmark`
5. `new`
6. `cached`
7. `read`
8. `unread`

#### 状态规则

| status | 背景 | 标题 | 右侧标识 | 备注 |
| --- | --- | --- | --- | --- |
| `current` | `brand.primarySoft` | `brand.primary` | `当前 · 38%` | 必须一眼可见 |
| `read` | 透明 | `text.secondary` | `已读` | 弱状态 |
| `unread` | 透明 | `text.primary` | `未读` 可选 | 默认状态 |
| `cached` | 透明 | `text.primary` | `缓存` badge | 不使用橙色 |
| `bookmark` | 极淡黄或透明 | `text.primary` | 书签图标或 `书签` | 必须和缓存区分 |
| `new` | 透明 | `text.primary` | `新` 或橙色点 | 使用 `accent.update` |
| `loading` | skeleton 行 | 不显示真实状态 | `加载中` 可选 | 不可点击 |
| `error` | 弱错误背景 | `text.primary` | `重试` 或错误标识 | 可局部重试 |

#### 当前章节

- 当前章节不使用独立左侧图标。
- 当前章节必须通过浅蓝背景、蓝色标题、右侧 `当前 · 百分比` 三者共同表达。
- 当前章节仍是紧凑列表行，不变成大卡片。

## 已定义的关键组件规格

以下规格是当前 P0/P1 组件库转化的执行依据。仍未确认的内容必须进入各组件的 Open Decisions，不得在生成稿中自行发挥。

### SettingIcon

#### 组件职责

为设置类、管理类入口提供统一线性图标来源。它只负责图标语义，不负责 row 布局。

#### Variant Properties

| property | values |
| --- | --- |
| `type` | `webdav / backup / reading / bookshelf / storage / privacy / about / sync / appearance / source / default` |

#### 结构

- 外层：`36 x 36` icon container。
- 内层：`20 x 20` icon drawing area。
- 图标必须使用 Icon System 的线宽和颜色规则。

#### Open Decisions

- `appearance` 和 `source` 图标需要人工确认语义是否足够清晰。
- 如果后续新增设置入口，必须先补 `SettingIcon type`，再替换页面 row。

### SettingRow

#### 组件职责

表达设置入口行。它不是卡片，不负责承载整组设置。

#### Nested Components

- 必须嵌套 `SettingIcon` instance。
- 图标通过 `SettingIcon` instance swap 表达。
- 禁止创建 `icon x trailing x state` variant matrix。

#### Variant Properties

| property | values |
| --- | --- |
| `trailing` | `arrow / switch / value / none` |
| `state` | `default / pressed / disabled` |

#### Content Slots

| slot | 规则 |
| --- | --- |
| `title` | 必填，真实 Text，可 override |
| `description` | 可选，真实 Text，可 override |
| `value` | 仅 `trailing=value` 必填，真实 Text，可 override |
| `icon` | 必填，来自 `SettingIcon` |

#### Geometry

| 项 | 规则 |
| --- | --- |
| row height | `64` |
| leading x | `16` |
| content x | `64` |
| trailing right padding | `16` |
| divider | 起点与 content x 对齐 |

#### Open Decisions

- 剩余 `我的` 页设置行应分批替换，每批替换后截图验收。
- 如果某个设置项没有可靠 SettingIcon，必须先补图标，不直接替换。

### ProgressBar

#### 组件职责

表达不可拖动的阅读进度或加载进度摘要。可拖动进度使用 `ProgressSlider`。

#### Variant Properties

| property | values |
| --- | --- |
| `value` | `0 / 25 / 50 / 75 / 100 / unknown` |
| `size` | `compact / full` |
| `state` | `default / loading / disabled` |

#### Content Slots

- `percentLabel` 可选，但 BookCard 的 `progress=partial` 必须显示百分比文案。
- `unknown` 不显示假百分比。
- `loading` 不显示真实百分比。

#### Acceptance Criteria

- `25 / 50 / 75` 的 fill 长度必须明显不同。
- `unknown` 与 `loading` 必须能区分。
- `100` 必须满格。
- `disabled` 必须弱化但保留布局。

### BookCover

#### 组件职责

表达书籍封面图像及封面缺失、加载、失败状态。

#### Variant Properties

| property | values |
| --- | --- |
| `state` | `normal / noCover / loading / error` |
| `size` | `small / medium / large` |

#### Geometry

- 比例固定 `2:3`。
- 圆角固定 `8`。
- small / medium / large 只改变尺寸，不改变语义结构。

#### 状态规则

| state | 必须表达 |
| --- | --- |
| `normal` | 真实或拟真封面 |
| `noCover` | 极简占位，并显示书名或 `无封面` |
| `loading` | skeleton 或 `加载中` |
| `error` | 弱错误标识或 `封面加载失败` |

loading 不得是空白封面；error 不得和 noCover 相同。

### BookCard

#### 组件职责

表达书架、搜索、推荐等场景中的书籍入口。书封是视觉中心。

#### Nested Components

- 必须嵌套 `BookCover`。
- 只要存在进度，必须嵌套 `ProgressBar`。

#### Variant Properties

| property | values |
| --- | --- |
| `mode` | `cover / list / recommend` |
| `progress` | `none / partial / complete / unknown / loading` |
| `state` | `default / pressed / disabled` |

#### Progress 显示规则

| progress | 必须显示 | 禁止 |
| --- | --- | --- |
| `none` | 不显示百分比，不显示 ProgressBar | 不得显示假进度 |
| `partial` | 百分比文案 + ProgressBar | 不得只显示条 |
| `complete` | `已读完` 或 `100%`，ProgressBar 满格 | 不得显示 partial 样式 |
| `unknown` | `进度未知` 或隐藏进度区 | 不得显示随机百分比 |
| `loading` | skeleton 或 `进度加载中` | 不得显示旧百分比 |

#### Typography

- title 最多两行，尾部省略。
- author / status / chapter 属于 metadata。
- 百分比属于 progress metadata，不能被 metadata 挤掉。

#### Open Decisions

- recommend 模式是否允许比 cover 模式多一行简介，需要单独确认。

### ListRow

#### 组件职责

表达安静的工具型列表项，不用于社交动态或营销卡片。

#### Variant Properties

| property | values |
| --- | --- |
| `type` | `book / source / rss / setting / note` |
| `state` | `default / selected / pressed / disabled` |

#### Anatomy

- leading：icon、thumbnail 或空。
- content：title、subtitle、metadata。
- trailing：arrow、value、switch、badge 或空。
- divider：属于 row group，不属于浮动装饰。

#### Geometry

- leading 区域宽度稳定。
- content x 不随 leading 图标语义变化。
- divider 起点与 content x 对齐。
- selected 背景覆盖整行可点击区域。
- selected 行内部隐藏穿过背景的 divider。

#### Open Decisions

- selected 是否允许阴影：默认不建议。若使用，必须贴合 selected 背景边界。

### ReaderPanel / BottomSheet

#### 组件职责

`BottomSheet` 提供外层覆盖容器；`ReaderPanel` 提供阅读控制层内部模块。二者不能混为一个自由排版 frame。

#### Variant Properties

`BottomSheet`：

| property | values |
| --- | --- |
| `mode` | `stuck / floating` |
| `type` | `default / action / settings / reader` |

`ReaderPanel`：

| property | values |
| --- | --- |
| `type` | `toc / tts / appearance / settings / brightness / autoPage` |
| `state` | `default / loading / empty / error` |

#### Radius Rules

- `BottomSheet mode=stuck`：顶部圆角 `28`，底部圆角 `0`。
- `BottomSheet mode=floating`：四角圆角 `28`。
- `ReaderPanel`：圆角 `16-20`，不得大于外层 sheet。
- 内部控件：圆角 `10-14`。
- `BrightnessRail`：圆角 `20-24`，但必须自包含，不得只用竖线分隔。

#### Reader Context

- 阅读控制层必须保留阅读上下文。
- ReaderPanel 不得变成全屏设置页。
- 切换目录、朗读、界面、设置时，只替换左侧或主体 panel 内容；外壳和固定控制区不应漂移。

### TocPanel / ChapterRow

#### 组件职责

表达阅读内目录/书签快捷面板和章节跳转行。

#### Variant Properties

`TocPanel`：

| property | values |
| --- | --- |
| `tab` | `目录 / 书签` |
| `state` | `default / loading / empty / error` |

`ChapterRow`：

| property | values |
| --- | --- |
| `status` | `current / read / unread / cached / bookmark / new / loading / error` |

#### ChapterRow 规则

- current 必须使用 Chapter Status System 的三重标识：浅蓝背景、蓝色标题、右侧 `当前 · xx%`。
- read / unread / cached / bookmark / new 必须能互相区分。
- cached 使用 badge，不使用橙色。
- new 可使用 `accent.update` 点或 `新` badge。
- bookmark 使用书签标识或 `书签`，不得和 cached 共用同一种 badge。
- loading 使用 skeleton 行，不显示真实章节状态。
- error 使用局部错误标识，可提供重试。

#### TocPanel 结构

- Header：`目录` + `完整目录 >`。
- Subtitle：阅读百分比和总章节数。
- Segment：`目录 / 书签`，selected 必须明确。
- 当前卷信息和 `回到进度` 可作为弱信息行。
- 章节行必须紧凑，不得卡片化。
