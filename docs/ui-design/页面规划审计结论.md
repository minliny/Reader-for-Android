# 页面规划审计结论

审计日期：2026-06-13

## 1. 当前主 TAB

仓库现有文档中存在两套互相冲突的主 TAB 口径：

| 来源 | 主 TAB 口径 | 当前判断 |
| --- | --- | --- |
| `docs/ui-handoff/ROUTE_MAP.md`、`docs/ui-design/STITCH_DESIGN_SOURCE_OF_TRUTH.md` | 书架 / 发现 / RSS / 设置 | 当前较新的治理口径 |
| `docs/cross-platform-ui/CROSS_PLATFORM_ROUTE_MATRIX.md`、`docs/cross-platform-ui/CROSS_PLATFORM_UI_BASELINE.md`、`docs/ui-handoff/SCREEN_MATRIX.md` | 书架 / 发现 / 书源 / 我的 | 旧口径，和当前规划冲突 |

按本次修正后的范围口径：

- P0 当前主入口：书架。
- 发现：暂缓，不计入当前必须完成的主 TAB 验收。
- RSS：暂缓，不计入当前必须完成的主 TAB 验收。
- 设置：属于 P1，不计入当前 P0 主 TAB 完成标准。
- 不规划“我的”主 TAB。
- 书源管理不是主 TAB，属于设置或管理链路中的功能页。

因此，当前页面规划必须先统一为：

```text
P0：书架主入口
P1：设置入口与设置链路
暂缓：发现、RSS
废弃：我的、书源独立主 TAB
```

## 2. 页面规划总体结论

当前页面规划不是空白，但不能判定为“全应用页面规划完成”。

主要原因：

1. 主 TAB 口径在多个文件中不一致。
2. 发现和 RSS 已被确认暂缓，但部分 route、矩阵和旧 handoff 仍把它们当当前主模块。
3. 设置被确认属于 P1，但部分文档仍把设置首页、App 设置二级页和全局状态模板纳入 P0。
4. 书架、搜索、详情的状态页和底表仍未完成。
5. 阅读外观 `35` 至 `39` 虽有图片，但不能作为最终 source。
6. 页面级 `usable=true` 仍为 0，说明 source、文字稿、token、组件、交互、人工审核未形成闭环。

## 3. 49/50 系列还差什么

49/50 系列是当前书架、搜索、详情链路的高优先级缺口。它们不是“已有图片缺文字稿”的问题，而是规划页面/状态尚未形成可用图片 source。

### 49 系列：书架状态与操作

还差 4 张：

| 编号 | 页面或状态 | 当前缺口 |
| --- | --- | --- |
| 49 | 书架空状态 | 需要保留书架标题栏、分组栏和主入口结构，提供导入本地书、搜索书籍等首要动作 |
| 49-1 | 书籍操作底表 | 需要覆盖继续阅读、查看信息、刷新目录、缓存、移动分组、导出、移出书架 |
| 49-2 | 书架排序/筛选 | 需要覆盖排序方式、升降序、筛选条件和应用/取消行为 |
| 49-3 | 书架分组管理 | 需要覆盖分组列表、新增、重命名、排序、删除、移动书籍后的状态 |

49 系列完成标准：

- 每张图必须是单一页面或单一状态，不使用拼图。
- 必须使用当前暖白、紧凑、管理型视觉语言。
- 必须有中文文字稿，说明入口、返回、组件、交互、状态、失败反馈。
- 必须明确是否属于 P0、P1 或暂缓。

### 50 系列：搜索与详情状态

还差 5 张：

| 编号 | 页面或状态 | 当前缺口 |
| --- | --- | --- |
| 50-0 | 搜索 loading | 搜索结果页骨架、搜索框保留、取消或返回行为 |
| 50-1 | 搜索 empty | 无结果说明、修改关键词、切换来源、返回搜索首页 |
| 50-2 | 搜索 error | 请求失败、重试、切换来源、保留查询词 |
| 50-3 | 书籍详情 loading | 顶栏保留、封面/标题/目录预览骨架 |
| 50-4 | 书籍详情 error / 来源失效 | 区分来源失效、网络失败、书籍不存在，不能直接用全屏空状态替代 |

50 系列完成标准：

- 搜索状态不能混同书架状态。
- 详情失败不能直接显示“书籍不存在”，必须区分来源失效和网络异常。
- loading、empty、error 必须保留页面上下文，不得跳到通用孤立状态页。
- 每张图必须配套中文文字稿。

## 4. 35-39 还差什么

`35` 至 `39` 对应阅读外观链路。当前问题不是“完全没有图”，而是现有图不能作为最终 source。

| 图片 | 规划页面 | 当前问题 | 还差什么 |
| --- | --- | --- | --- |
| `35-reader-appearance-font-selection-standard.png` | 字体选择 | 缺少规格要求的字体管理内容，视口不符合目标 | 需要补字体分组、导入、管理、不可用字体、示例预览，并重出标准视口图 |
| `36-reader-appearance-theme-selection-standard.png` | 主题选择 | 主题卡缺正文预览，视口不符合目标 | 需要每个主题卡展示正文预览、当前选中态、自定义主题入口，并重出标准视口图 |
| `37-reader-appearance-theme-edit-standard.png` | 主题编辑 | 内容基本完整，但 final token 未冻结，视口门禁未过 | 需要按最终 token 重审颜色、对比度、保存/取消/重置状态 |
| `38-reader-appearance-layout-advanced-standard.png` | 版式高级 | 内容基本完整，但 final token 和目标视口未闭合 | 需要按目标视口重审页边距、行高、段距、缩进、预览区 |
| `39-reader-appearance-page-turn-animation-standard.png` | 翻页动画 | 存在 iOS 状态栏、视口不符、token 未冻结 | 需要 Android 标准状态栏、安全区、减少动态选项和真实行为说明 |

35-39 完成标准：

- 必须重申它们属于“界面/外观”，不属于“阅读设置”。
- 必须使用 Android 手机 `390×884` 逻辑基准。
- 必须去除 iOS 状态栏或非 Android 系统元素。
- 必须通过 final token、页面级人工审核和 source mapping。
- 必须配套中文文字稿。

## 5. 发现、RSS、设置的范围修正

### 发现

当前处理：暂缓。

不应计入当前 P0 完成标准。已有发现首页或旧图只能作为后续参考，不作为当前必须完成项。

### RSS

当前处理：暂缓。

RSS 首页、订阅管理、订阅详情、搜索等均不计入当前 P0 完成标准。旧 route 可保留为后续规划，但不能反向要求当前补齐。

### 设置

当前处理：P1。

设置首页、App 设置二级页、同步备份、缓存、隐私权限、关于反馈等进入 P1。现有 `48` 组已经人工 FAIL/EXCLUDE，不能作为 P1 完成依据；如果 P1 要继续推进，需要重新出图、重新写中文文字稿、重新人工审核。

## 6. 中文文稿和中文文件名要求

后续新增审计、规划、文字稿文件应使用中文文件名和中文正文。

建议命名规则：

| 类型 | 中文文件名示例 |
| --- | --- |
| 页面规划审计 | `页面规划审计结论.md` |
| 图片文字稿总册 | `图片逐图文字稿总册.md` |
| 页面 source 映射 | `页面来源映射表.md` |
| 人工视觉审核队列 | `人工视觉审核队列.md` |
| 缺失页面文字稿 | `缺失页面中文文字稿.md` |
| 上传前审计 | `上传前审计报告.md` |

已有英文文件名不应直接批量改名，除非同步更新所有引用、脚本、清单和交叉链接。否则会制造新的断链。
