同步与备份 页面包以本目录 `01-06` 文字稿为产品结构和交互基础。生成 UI、HTML 或 Stitch 初稿时，必须先读取唯一全局约束，再读取本页面包全部 Markdown，最后读取当前 UI 设计图。

## 3. 页面树约束

```text
Page
  StatusBar
  TopBar
  ContentScroll
    PrimarySection
    SecondarySection
    StateLayer
  OverlayLayer
```

页面树必须与 `01-页面结构稿.md` 保持一致。不得为了贴近某张旧图而新增未声明入口、卡片、底部导航或营销模块。

## 4. 组件约束

组件清单以 `02-组件规格稿.md` 为准；视觉实现必须继承：

- `01-全局规范/00-唯一约束参考.md`
- `01-全局规范/02-应用外壳组件视觉规格.md` 中适用的顶部栏、卡片、列表行、状态组件规则
- 设置链路页面继续继承 `05-设置链路/00-设置页面视觉规范.md`
- 阅读链路页面继续继承 `04-阅读链路/阅读控制层/09-阅读控制层视觉规格.md` 或 `04-阅读链路/08-阅读派生模块继承视觉规格.md` 中适用规则

## 5. 状态约束

必须覆盖 `default/loading/empty/error/offline/permission/partial success` 中与本页相关的状态。缺少当前 UI 图的状态只能标为 `NEEDS_VISUAL_SOURCE`，不能补成最终稿。

## 6. 响应式约束

- 手机竖屏以 `390x884dp` 为逻辑基准。
- 宽度 `<360dp` 时压缩外边距和模块间距，不压缩 `44dp` 触控区。
- 宽度 `360-430dp` 使用默认布局。
- 宽度 `>430dp` 内容居中并限制最大宽度，不让列表行和卡片无限拉宽。
- 高度不足时优先保留标题、返回、主操作和首个核心内容模块。

## 7. Stitch/HTML 门禁

进入 `STITCH_READY` 前必须具备：

```text
PAGE_VISUAL_BASELINE + FORMAL_DRAFT + VISUAL_SPEC + COMPONENT_REFERENCE + TOKEN_READY + ASSET_READY
```

当前未具备完整组件局部和截图叠放验收时，只能生成 `HTML_DRAFT`，不得标为最终复刻。
