# 书源管理首页 Stitch 提示词

## 目标

生成 `书源管理首页`，用于补齐设置页中 `书源管理` 入口的落点。

## Stitch 提示词

```text
请生成一张中文阅读 App 的「书源管理首页」移动端 UI 设计稿。

基础要求：
- 画布尺寸：390 × 884。
- 移动端竖屏。
- 中文阅读 App。
- 页面属于书源管理模块。
- 底栏结构固定为：书架 / 发现 / RSS / 设置。
- 遵循现有 docs/ui-design/drafts 风格：暖色纸张背景、柔和卡片、轻阴影、紧凑列表、蓝色强调色、低刺激阅读 App 视觉密度。
- 不要生成小说阅读页。
- 不要改变 App Shell。
- 不要出现账户、头像、登录入口、我的页面结构。
- 输出可编辑图层，方便进入 Figma。

页面定位：
- 这是从「设置首页 > 书源管理」进入的完整管理页。
- 它不是阅读页顶部「换源」底表。
- 它不是底栏主页面，但视觉要和 App 主页面一致。

页面结构：
1. 顶部栏
   - 返回按钮。
   - 标题：书源管理。
   - 右侧搜索图标。
   - 右侧更多图标。

2. 状态概览卡
   - 展示 4 个紧凑指标：
     - 12 个书源
     - 8 个启用
     - 4 个异常
     - 刚刚检测
   - 使用小图标和轻分隔线。

3. 搜索与筛选区
   - 紧凑搜索框：搜索书源名称或域名。
   - 状态筛选 chips：全部 / 已启用 / 异常 / 未检测 / 自定义。
   - 分组筛选 chips：全部分组 / 玄幻书源 / 起点导入 / 测试书源。

4. 书源列表
   - 列表行展示：
     - 书源名称。
     - 域名或来源说明。
     - 分组标签。
     - 最近检测状态：可用、异常、未检测。
     - 启用/禁用 switch。
     - 行内检测入口。
     - 行内调试信息入口或三点菜单。
   - 示例书源：
     - 起点中文网，可用，已启用。
     - 笔趣阁，异常，已启用。
     - 本地导入源，未检测，已禁用。
     - 测试书源，可用，已启用。

5. 新增书源入口
   - 右下浮动按钮或列表上方主按钮：新增书源。
   - 不要做成过大的营销按钮。

视觉规则：
- 页面信息密度偏管理型，安静、可扫描。
- 卡片圆角 8-16dp。
- 列表行高度稳定，右侧开关和检测入口对齐。
- 不使用阅读控制层、亮度滑条、章节进度条。
- 不要把检测页、详情页、编辑页画进同一张图，只保留入口。

输出目标：
- 书源管理首页默认态。
- 可编辑图层。
```
