# UI 设计唯一约束参考

状态：唯一权威约束
适用范围：`docs/ui-design` 下所有当前实现页面包
目标：约束从文字稿到 UI 设计图，再到 HTML/Stitch/客户端实现的完整复刻流程。

## 0. 权威性

本文件是 UI 设计目录下唯一的全局约束入口。其他全局规范、页面包文字稿、审计文件只允许补充专项细节，不允许另立一套优先级、输入格式或验收标准。

当文档冲突时，按以下顺序处理：

1. 当前页面的 UI 设计图和完整组件局部图。
2. 本文件。
3. 页面包 `10-正式UI设计稿.md` 与页面视觉规格。
4. 全局专项规范，例如 App Shell、阅读控制层、设置链路规范。
5. 审计文件中的结论。
6. 历史图、候选图、旧草稿。

产品语义、入口、交互结果以正式文字稿为准；视觉比例、密度、颜色、圆角、阴影、边距、字号以当前 UI 设计图为优先来源。

## 1. 复刻闭环

每个页面从文字稿到 HTML 完全复刻，必须形成以下闭环：

```text
当前 UI 设计图
  -> 完整组件局部
  -> 页面结构稿
  -> 视觉 token
  -> 资产清单
  -> HTML/Stitch 初稿
  -> 截图叠放验收
  -> 修正后冻结
```

缺任一环节，只能标记为候选稿，不得标记为 `STITCH_READY` 或 `IMPLEMENTATION_READY`。

## 2. 来源约束

每个页面包必须有唯一当前来源：

| 来源 | 要求 |
| --- | --- |
| 当前视觉母版 | `图片/*.png` 中只允许一个当前采用母版；多状态图必须标明用途 |
| 正式文字稿 | `10-正式UI设计稿.md`，定义页面结构、内容、状态、交互 |
| 视觉规格 | `08-*视觉规格.md` 或 `09-*视觉规格.md`，定义可执行 token |
| 完整组件局部 | `组件局部/*.png` 或等价目录，裁完整组件上下文 |
| 资产清单 | 图标、封面、背景、插图、纹理必须给路径和状态 |
| 验收规则 | 定义截图尺寸、允许误差、人工复核项 |

历史图、候选图、Stitch 草稿、旧 HTML、文件名包含 `旧规格` 的文档不得反向定义当前页面视觉。

## 3. 页面结构约束

文字稿必须给出页面树。页面树用于约束 HTML/Stitch 不能自由重组布局。

示例：

```text
Page
  StatusBar
  TopBar
  ContentScroll
    Section
    Card
    List
  BottomNav
  OverlayLayer
```

页面树必须明确：

- 固定区和滚动区。
- 顶部、底部、浮层、底表的层级。
- 首屏必须可见的模块。
- 可折叠、可滚动、可替换的区域。
- 哪些组件继承全局 token，哪些组件是页面专属。

## 4. 视觉分层

每个页面按四层定义视觉：

| 层级 | 内容 | 必填产物 |
| --- | --- | --- |
| `Page Token` | 页面背景、安全区、边距、最大宽度、滚动规则 | 页面 token 表 |
| `Component Token` | 顶栏、卡片、列表项、按钮、chip、导航、底表、控制面板 | 组件 token 表 |
| `Content Token` | 标题、正文、章节、作者、封面、来源、时间、进度 | 内容 token 表 |
| `State Token` | 默认、激活、未激活、选中、禁用、loading、empty、error | 状态 token 表 |

只写“简洁、现代、舒适、高级”不算视觉规格。

## 5. 几何约束

完全复刻必须有可执行几何数据：

| 类别 | 必填 |
| --- | --- |
| 页面 | 基准宽高、安全区、左右边距、最大内容宽度 |
| 模块 | x/y/width/height 或响应式布局规则 |
| 容器 | padding、gap、radius、border、shadow |
| 文本 | 字号、字重、行高、颜色、最大行数、截断规则 |
| 图片 | 比例、圆角、裁切、占位、失败态 |
| 控件 | 高度、宽度、触控热区、激活/未激活状态 |
| 浮层 | 遮罩、锚点、层级、外壳尺寸、拖拽柄 |

如果某页面暂时没有精确 x/y，必须至少有完整组件局部图和可量化 token，不能让生成器凭感觉排版。

## 6. 完整组件局部

完整组件局部是复刻的核心输入，不等于小图标抠图。

| 组件 | 裁剪范围 |
| --- | --- |
| 顶部栏 | 标题、右侧操作、触控槽位、状态栏关系 |
| 底部导航 | 单项图标、文字、激活态颜色、导航背景、顶部边界 |
| 卡片 | 背景、边框、圆角、阴影、内边距、全部内容 |
| 列表项 | 图标/封面、标题、辅助文本、尾部控件、分隔线 |
| Chip | 激活/未激活背景、文字、边框、间距 |
| 进度条/滑条 | 轨道、填充、滑块、标签、长度规则 |
| 底表/面板 | 外壳、拖拽柄、区域分割、按钮组 |
| 阅读正文 | 背景、正文行高、段间距、信息层关系 |

图标透明 PNG、SVG、单独 crop 只能作为 `ICON_REFERENCE_ONLY`，不能替代完整组件局部。

## 7. 资产约束

资产必须标注状态：

| 状态 | 含义 |
| --- | --- |
| `ASSET_READY` | 可直接用于 HTML/实现 |
| `REFERENCE_BASELINE` | 可作为视觉基准，但不一定可直接使用 |
| `CUT_REFERENCE_ONLY` | 从设计图截取，仅用于取形参考 |
| `ICON_REFERENCE_ONLY` | 图标参考，不能替代完整组件 |
| `NEEDS_REDRAW` | 需要重新绘制 |
| `REJECTED` | 不参与当前实现 |

当前图标策略：

- 视觉判断优先看完整组件局部。
- 小图标抠图和手画 SVG 都不是当前最终资产。
- 只有通过叠放复核的图标才能进入 `ASSET_READY`。

## 8. 最小 token 集

每个页面视觉规格至少包含：

```text

Page.SafeAreaTop
Page.SafeAreaBottom
Page.MaxContentWidth

TopBar.Height
TopBar.PaddingX
TopBar.TitleSize
TopBar.IconSlot

Card.Background
Card.BorderColor
Card.BorderWidth
Card.Radius
Card.Shadow
Card.Padding
Card.Gap

Text.TitleSize
Text.TitleWeight
Text.BodySize
Text.MetaSize
Text.LineHeight
Text.PrimaryColor
Text.SecondaryColor

Control.Height
Control.Radius
Control.Padding
Control.ActiveColor
Control.InactiveColor

Image.AspectRatio
Image.Radius
Image.Fit

State.Default
State.Active
State.Disabled
State.Loading
State.Empty

```

页面可以增加专项 token，但不能少于本最小集。

## 9. 响应式约束

页面必须写清：

| 条件 | 必填 |
| --- | --- |
| 小屏 | 宽度 `<360dp` 时保留哪些模块、压缩哪些间距 |
| 常规屏 | `360-430dp` 的默认布局 |
| 大屏 | 最大内容宽度、居中规则、列数变化 |
| 短屏 | 高度不足时首屏优先级 |
| 横屏/平板 | 面板锚点、侧置/底置切换条件 |

响应式不能靠 HTML 自适应默认行为推断。

## 10. 验收约束

HTML/Stitch 输出必须截图验收。

| 项目 | 要求 |
| --- | --- |
| 基准图 | 当前视觉母版和完整组件局部 |
| 输出尺寸 | 必须与基准截图尺寸一致，或使用约定设备尺寸 |
| 页面背景 | 不允许色相偏移 |
| 主模块位置 | 默认允许误差 `±2px`，未量化前必须人工复核 |
| 圆角/边框 | 默认允许误差 `±1px` |
| 字号 | 默认允许误差 `±1px` |
| 文案 | 必须完全一致 |
| 资产 | 必须使用资产清单路径，不得替换为默认图标/占位图 |
| 状态 | default/loading/empty/error/selected 必须覆盖 |

未截图验收的 HTML 只能算草稿。

## 10.1 HTML/Stitch 输入包约束

每个页面进入 HTML/Stitch 生成前，必须有页面级 `11-HTML-Stitch输入包.md`。输入包只定义生成输入和验收门禁，不等于页面已经通过复刻验收。

输入包必须包含：

| 项目 | 要求 |
| --- | --- |
| 状态 | 使用 `STITCH_INPUT_READY_NEEDS_GENERATION` 或明确说明阻塞原因 |
| 输出门禁 | 初次生成统一标记 `HTML_DRAFT` |
| 基准输出 | 当前页面视觉基准、基准宽高、截图验收尺寸 |
| 输入读取顺序 | 全局唯一约束、正式稿、视觉规格、资产清单、组件局部、页面分稿、图片资产 |
| 图片角色 | 区分 `PAGE_VISUAL_BASELINE`、`VISUAL_MASTER`、`STATE_OR_VARIANT_REFERENCE`、`REFERENCE_BASELINE`、`REFERENCE_ONLY`、`COMPONENT_REFERENCE` |
| 页面生成约束 | 用正向语言说明页面应该生成的结构、模块、状态和交互结果 |
| 响应式约束 | 明确小屏、常规屏、短屏、横屏/平板的适配规则 |
| 验收清单 | 同尺寸截图叠放、主模块误差、token 误差、人工复核项 |

输入包生成规则：

1. 页面没有当前页面视觉基准时，不得创建 ready 输入包，只能登记为 `MISSING_PAGE_VISUAL_BASELINE`。
2. 页面没有完整组件局部时，不得标记为 `STITCH_READY`，只能登记为 `NEEDS_COMPONENT_REFERENCE`。
3. 多张图片页面必须先指定当前页面视觉基准，历史图、候选图、流程图不得作为默认生成输入。
4. HTML 生成必须优先读取当前 UI 设计图和完整组件局部；文字稿负责语义、交互和状态。
5. 输出通过同尺寸截图叠放前，状态必须停留在 `HTML_DRAFT`。

## 11. 页面状态

页面统一使用以下状态：

| 状态 | 含义 |
| --- | --- |
| `PAGE_VISUAL_BASELINE` | 有唯一当前页面视觉基准图，只约束本页面输出 |
| `PAGE_VISUAL_CANDIDATE` | 有页面视觉候选图，尚未确认成当前页面视觉基准 |
| `MISSING_PAGE_VISUAL_BASELINE` | 缺少当前页面视觉基准图 |
| `VISUAL_MASTER` | 标准母版图，可被同类页面继承视觉 token 与组件结构 |
| `FORMAL_DRAFT` | 有正式 UI 设计稿 |
| `VISUAL_SPEC` | 有页面视觉规格 |
| `COMPONENT_REFERENCE` | 有完整组件局部图 |
| `TOKEN_READY` | 页面/组件/内容/状态 token 已量化 |
| `ASSET_READY` | 页面依赖资产可用 |
| `HTML_DRAFT` | 已生成 HTML 初稿 |
| `VISUAL_VERIFIED` | 截图叠放验收通过 |
| `STITCH_READY` | 可交给 Stitch 复原 |
| `IMPLEMENTATION_READY` | 可进入客户端实现 |

`STITCH_READY` 的最低条件：

```text
PAGE_VISUAL_BASELINE + FORMAL_DRAFT + VISUAL_SPEC + COMPONENT_REFERENCE + TOKEN_READY + ASSET_READY
```

## 12. 页面包标准结构

```text
页面包/
  00-说明.md
  01-页面结构稿.md
  02-组件规格稿.md
  03-交互规则稿.md
  04-状态规则稿.md
  05-文案稿.md
  06-禁止项.md
  08-视觉规格.md
  10-正式UI设计稿.md
  图片/
  组件局部/
  资产/
  验收/
```

现有页面可以逐步迁移，但新页面必须按此结构创建。

母版只能用于可复用、可继承的同类页面标准。单个页面自己的当前 UI 图必须写为 `PAGE_VISUAL_BASELINE`；未确认的生成图必须写为 `PAGE_VISUAL_CANDIDATE`；引用其他页面图片只能写为 `REFERENCE_BASELINE` 或 `STATE_OR_VARIANT_REFERENCE`。

退出当前实现的历史/候选页面不能定义 token。文件名包含 `旧规格` 的页面文档只能作为 `REFERENCE_ONLY`，不能与 `10-正式UI设计稿.md` 或页面视觉规格并列作为当前约束。

## 14. 执行顺序

1. `02-主标签页/书架` 达到 `TOKEN_READY`。
2. `04-阅读链路/阅读控制层` 达到 `TOKEN_READY`。
3. `04-阅读链路/沉浸阅读` 达到 `TOKEN_READY`。
4. `03-书架链路/书籍操作底表` 达到 `TOKEN_READY`。
5. 四个主标签页继承 App Shell token。
6. 书架链路和阅读链路 P1 页面补组件局部和 token。
7. 设置链路补正式稿、视觉规格和组件局部。

每次只允许一个页面包升级状态；状态升级必须有审计记录。
