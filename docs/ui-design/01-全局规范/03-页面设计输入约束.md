# 页面设计输入约束示例

状态：草案，后续所有页面设计默认按本稿准备  
适用范围：书架主 Tab 页面，以及后续发现、RSS、设置等主流程页面  
目标：让人工评审、Stitch 复原和客户端实现使用同一套页面输入，不再依赖图片自由推断

## 1. 总原则

每个页面必须同时具备文字稿和图片示例。文字稿是权威约束，图片示例是视觉参考。

当文字稿与图片冲突时，按以下优先级处理：

1. 页面级视觉规格和全局组件 token。
2. 页面文字稿中的结构、状态、交互、文案、禁止项。
3. 当前 source 图片。
4. 历史图、参考图、探索稿。

Stitch 输入不能只给图片，也不能只给一句页面描述。图片无法稳定表达组件复用、响应式、状态边界和禁止项；文字稿无法完全替代版面密度、视觉层级和真实比例。

## 2. 每个页面必须提供的文字稿

每个页面至少拆成 6 类文字稿。可以放在一个 Markdown 文件，也可以放在页面包目录中分文件维护。

| 类型 | 必填内容 | 用途 |
| --- | --- | --- |
| `页面结构稿` 页面结构稿 | 页面目标、入口、返回、页面分区、固定区、滚动区、首屏优先级 | 约束 Stitch 不要重组页面 |
| `组件规格稿` 组件稿 | 页面使用哪些组件、组件顺序、组件间距、组件复用关系 | 约束不同页面不要产生新样式 |
| `visual-spec` 视觉规格稿 | 背景、卡片、边框、圆角、阴影、字号、颜色、尺寸、响应式 token | 约束 UI 还原和客户端实现 |
| `交互规则稿` 交互稿 | 点击、长按、滑动、弹窗、底表、模式切换、加载触发 | 约束操作结果 |
| `状态规则稿` 状态稿 | default、loading、empty、error、permission、offline、partial data | 约束异常态和边界态 |
| `文案稿` 文案稿 | 页面可见中文、按钮文案、提示语、错误文案、空态文案 | 避免 Stitch 自造文案 |
| `禁止项` 禁止项 | 禁止出现的入口、按钮、组件、样式、交互 | 避免生成“看起来合理但产品不允许”的内容 |

复杂页面还需要补充：

| 类型 | 触发条件 | 必填内容 |
| --- | --- | --- |
| `responsive-spec` 自适应稿 | 页面受屏幕宽高影响明显 | 小屏、常规屏、大屏、横屏、短屏下的布局规则 |
| `data-spec` 数据稿 | 页面有列表、搜索、排序、筛选、详情字段 | 字段来源、字段为空时的显示、最多行数、截断规则 |
| `navigation-spec` 导航稿 | 页面有跨页面流转 | 来源页面、目标页面、返回栈、是否保留筛选和滚动位置 |
| `accessibility-spec` 可访问性稿 | 有图标按钮、表单、开关、长列表 | 触控热区、语义标签、动态字号、对比度底线 |

## 3. 每个页面必须提供的图片示例

每个页面至少有 1 张当前 source 图片。页面复杂度越高，图片示例越要覆盖状态和布局分支。

| 图片类型 | 是否必需 | 命名示例 | 要表达的信息 |
| --- | --- | --- | --- |
| `default` 默认态 | 必需 | `01-书架首页-封面模式.png` | 页面真实结构、视觉密度、首屏比例 |
| `alternate-layout` 替代布局 | 有模式切换时必需 | `02-书架首页-列表模式.png` | 同一页面在不同模式下的重排方式 |
| `loading` 加载态 | 有远程数据时必需 | `10-搜索状态-loading.png` | 骨架屏、加载位置、是否保留外框 |
| `empty` 空态 | 有列表/搜索结果时必需 | `11-搜索状态-empty.png` | 空态插画/图标、文案、主操作 |
| `error` 错误态 | 有网络/来源/权限错误时必需 | `12-搜索状态-error.png` | 错误层级、重试入口、辅助说明 |
| `sheet-dialog` 底表/弹窗 | 有浮层时必需 | `07-书籍操作底表.png` | 容器高度、圆角、按钮排列、遮罩 |
| `edge-case` 边界态 | 数据长度变化大时必需 | `xx-长书名-多分组.png` | 长文案、无图片、极少/极多数据 |
| `responsive` 响应式示例 | 页面布局会跨断点变化时建议必需 | `xx-书架首页-小屏.png` | 小屏、短屏、横屏或大屏的布局验证 |

图片只能放在页面所属目录的专属图片目录中，不再混放到全局 `drafts/`。书架主 Tab 当前使用：

```text
docs/ui-design/03-书架链路/
```

图片命名使用中文页面名，格式为：

```text
序号-页面名-状态或模式.png
```

例如：

```text
01-书架首页-封面模式.png
07-书籍操作底表.png
14-书籍详情-error来源失效.png
```

## 4. 页面包标准结构

复杂页面或需要交给 Stitch 复原的页面，必须建立页面包。

```text
页面包/页面名/
  00-说明.md
  01-页面结构稿.md
  02-组件规格稿.md
  03-交互规则稿.md
  04-状态规则稿.md
  05-文案稿.md
  06-禁止项.md
  images/
    default.png
    loading.png
    empty.png
    error.png
```

页面包内图片可以是专属图片目录的副本，也可以是引用说明。用于 Stitch 时，页面包必须能独立说明当前页面，不依赖读者猜测外部上下文。

## 5. 约束示例：书架首页封面模式

### 必需文字稿

| 文件 | 约束重点 |
| --- | --- |
| `01-页面结构稿.md` | 顶栏、分组 chip、继续阅读、最近更新、书架网格、底部导航的顺序和滚动关系 |
| `02-组件规格稿.md` | `ShelfTopBar`、`ShelfChipGroup`、`ContinueReadingCard`、`RecentUpdateCard`、`BookCard`、`BottomNav` |
| `18-书架首页组件视觉规格.md` | 页面背景、卡片、chip、封面、BookCard、BookListItem、进度条、底部导航 token |
| `03-交互规则稿.md` | 搜索、更多、分组切换、封面/列表切换、点击书籍、书籍更多按钮 |
| `04-状态规则稿.md` | 有书、空书架、加载、错误、无封面、长书名 |
| `05-文案稿.md` | 所有中文标题、按钮、空态、错误提示 |
| `06-禁止项.md` | 禁止新增发现/RSS/我的入口，禁止在书籍操作底表放配置项 |

### 必需图片

| 图片 | 用途 |
| --- | --- |
| `01-书架首页-封面模式.png` | 默认态和视觉密度参考 |
| `02-书架首页-列表模式.png` | 模式切换后的同 token 对照 |
| `06-书架空状态.png` | 无书状态 |
| `07-书籍操作底表.png` | 书籍更多按钮后的底表 |

### Stitch 生成约束

```text
请复原书架首页封面模式。
必须读取页面包全部 Markdown 和 default 图片。
图片只作为结构、比例、密度参考。
视觉 token、响应式和跨模式一致性以 18-书架首页组件视觉规格.md 为准。
不要根据图片自行新增按钮、Tab、渐变背景、营销卡片或不在文字稿中的入口。
```

| 列表页 | `页面结构稿`、`组件规格稿`、`data-spec`、`交互规则稿`、`状态规则稿`、`文案稿`、`禁止项` | 默认列表、loading、empty、error、长文本/无图边界态 |
| 搜索页 | `页面结构稿`、`组件规格稿`、`交互规则稿`、`状态规则稿`、`文案稿`、`禁止项` | 搜索首页、结果页、loading、empty、error |
| 详情页 | `页面结构稿`、`组件规格稿`、`data-spec`、`交互规则稿`、`状态规则稿`、`文案稿`、`禁止项` | 默认态、加载态、来源失效/错误态、字段缺失态 |
| 底表/弹窗 | `组件规格稿`、`visual-spec`、`交互规则稿`、`文案稿`、`禁止项` | 默认态、危险操作态、内容溢出态 |
| 筛选/排序页 | `页面结构稿`、`组件规格稿`、`交互规则稿`、`状态规则稿`、`文案稿`、`禁止项` | 默认态、已选择态、无结果态 |

## 7. 视觉规格必须覆盖的 token

每个页面的视觉规格至少包含：

- 页面背景色。
- 页面安全区、左右边距、内容最大宽度。
- 固定区和滚动区的边界。
- 卡片背景色、边框颜色、边框粗细、圆角、阴影层级。
- 主要组件高度、内边距、间距、触控热区。
- 标题、正文、辅助信息、按钮、错误信息的字号、字重、颜色、行数。
- 图标尺寸、线宽、颜色、激活态。
- 图片比例、圆角、裁切方式、占位、加载失败。
- loading、empty、error 是否复用默认态组件外框。
- 小屏、常规屏、大屏、横屏、短屏的布局规则。
- 与同族页面或同一模式切换页面必须共用的 token。

## 8. 跨页面一致性要求

同一功能域内的页面必须共用组件 token，不能每张图独立成套。

书架主 Tab 至少保持以下一致：

| 组件 | 必须一致的内容 |
| --- | --- |
| 顶栏 | 高度、标题字号、图标尺寸、触控区 |
| 分组 chip | 高度、圆角、激活/未激活颜色、文字样式 |
| 通用卡片 | 背景、边框、圆角、阴影 |
| 书籍封面 | 比例、圆角、占位规则 |
| 书籍信息 | 标题/作者/章节字号层级、截断规则 |
| 进度条 | 高度、圆角、轨道色、填充色 |
| 底部导航 | 高度、圆角、图标、文字、激活态 |
| 状态页 | 空态/错误态的容器、文案层级、主按钮样式 |

允许变化的是布局位置、列数、列表/网格方向、模块显隐和状态内容，不允许变化的是同一组件的基础视觉语言。

## 9. 审核标准

页面进入设计或 Stitch 前必须通过以下检查：

1. 当前页面是否有唯一 source 图片。
2. 图片是否位于页面所属目录，而不是全局混放目录。
3. 文字稿是否说明入口、返回、状态、交互和禁止项。
4. 视觉规格是否给出可执行 token，而不是只写“简洁”“高级”“现代”。
5. 图片中的组件是否与视觉规格一致。
6. 封面/列表、默认/状态、页面/底表之间是否复用同一组件 token。
7. Stitch 是否能在不猜测产品逻辑的情况下复原页面。
8. 客户端是否能按文字稿直接拆组件实现。

不满足以上任一项时，页面只能标记为候选稿，不能标记为冻结稿。
