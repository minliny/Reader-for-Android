# 页面包模板

状态：新目录执行标准

## 1. 标准结构

每个当前页面必须使用以下结构：

```text
页面名/
  00-页面包说明.md
  01-页面结构稿.md
  02-组件规格稿.md
  03-交互规则稿.md
  04-状态规则稿.md
  05-文案稿.md
  06-禁止项.md
  07-Stitch输入提示.md
  图片/
    01-默认态.png
    02-loading.png
    03-empty.png
    04-error.png
```

`07-Stitch输入提示.md` 只在需要 Stitch 复原时必填。

## 2. 文件职责

| 文件 | 职责 |
| --- | --- |
| `00-页面包说明.md` | 页面定位、输入文件、图片索引、权威约束、当前状态 |
| `01-页面结构稿.md` | 页面目标、入口、返回、固定区、滚动区、首屏优先级 |
| `02-组件规格稿.md` | 组件清单、组件顺序、组件复用、组件 token 来源 |
| `03-交互规则稿.md` | 点击、长按、滑动、弹窗、跳转、返回栈 |
| `04-状态规则稿.md` | default、loading、empty、error、permission、offline |
| `05-文案稿.md` | 页面所有中文可见文案 |
| `06-禁止项.md` | 禁止出现的入口、按钮、组件、样式、交互 |
| `07-Stitch输入提示.md` | 给 Stitch 的输入顺序、图片使用规则、禁止推断项 |

## 3. 图片命名

图片必须放在页面包自己的 `图片/` 目录中。

```text
01-默认态.png
02-列表模式.png
03-loading.png
04-empty.png
05-error.png
06-底表.png
```

禁止：

- 当前 source 图片继续放在全局 `drafts/`。
- 一个页面引用多个目录里的同名当前图。
- 使用英文状态名作为最终文件名。

## 4. 页面包说明模板

```markdown
# 页面名页面包

状态：
层级：
优先级：

## 页面定位

## 输入文件

## 图片示例

## 权威约束

## 当前不包含
```

## 5. Stitch 输入规则

给 Stitch 时必须输入：

1. 全局规范。
2. 页面包全部 Markdown。
3. 页面包 `图片/` 中的当前示例图。
4. 若图片与文字稿冲突，以文字稿和 token 为准。

禁止让 Stitch 从历史图、失败图、备份图中推断当前视觉。
