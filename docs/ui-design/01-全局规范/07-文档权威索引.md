## 4. 审计文档

`90-审计与索引/` 下的审计文件是某个时间点的检查记录，不是全局约束来源。

使用规则：

1. 审计文件可以说明缺口、风险和历史结论。
2. 审计文件不得覆盖 `00-唯一约束参考.md`。
3. 如果审计结论与唯一约束冲突，以唯一约束为准。
4. 新审计必须写明采用的约束版本和日期。

## 6. 旧规格文档

文件名包含 `旧规格` 的文档统一降级为 `REFERENCE_ONLY`：

1. 只能用于追溯历史决策或迁移来源。
2. 不定义当前视觉 token、页面结构、交互验收或 Stitch/HTML 生成标准。
3. 与 `10-正式UI设计稿.md`、页面视觉规格或 `00-唯一约束参考.md` 冲突时，旧规格无效。
