# 目录命名规范

状态：当前执行标准

## 1. 命名原则

- 目录和 Markdown 文件名使用中文。
- 产品固有名词可以保留：`RSS`、`Stitch`、`Android`。
- 技术 token 可以保留英文，例如 `AppShell.Page.Background`。
- 当前页面包文件名不得使用 `page-spec`、`component-spec`、`copy`、`do-not` 等英文模板名。

## 2. 编号规则

同一目录下使用两位数字排序：

```text
00-说明.md
01-页面结构稿.md
02-组件规格稿.md
03-交互规则稿.md
04-状态规则稿.md
05-文案稿.md
06-禁止项.md
07-Stitch输入提示.md
```

## 3. 图片规则

当前 source 图片使用中文名称：

```text
01-默认态.png
02-列表模式.png
03-loading.png
04-empty.png
05-error.png
```

若状态词已有中文表达，应使用中文：

| 不推荐 | 推荐 |
| --- | --- |
| `default.png` | `01-默认态.png` |
| `loading.png` | `03-加载态.png` |
| `empty.png` | `04-空态.png` |
| `error.png` | `05-错误态.png` |

## 4. 旧目录处理

旧目录或旧文件如果暂时不能迁移，应保留迁移说明，不再继续新增当前 source。

旧英文文件名只允许存在于：

- 历史归档。
- 外部工具生成结果。
- token 或代码上下文。

不得存在于新的页面包结构中。
