# 本地书导入正式 UI 设计稿

状态：`FORMAL_DRAFT_READY`
视觉状态：`PAGE_VISUAL_CANDIDATE`
页面等级：`P1 / BOOKSHELF_FLOW`

## 1. 设计目标

本地书导入用于通过 Android 系统文件选择器导入用户授权选择的本地书籍文件，并把导入结果加入书架。

本页先冻结文字稿，不冻结最终视觉稿。当前已有 image2 视觉 demo 候选并已裁组件局部，但尚未确认为正式页面视觉基准。

## 4. 状态

| 状态 | 要求 |
| --- | --- |
| 默认 | 展示选择说明、支持格式、`选择文件` |
| 解析中 | 展示解析进度和当前文件 |
| 成功 | 展示成功摘要和 `完成` |
| 部分失败 | 展示混合结果、失败原因、`重新选择` 和 `完成` |
| 全部失败 | 展示失败原因、`重新选择` 和 `返回书架` |
| 取消选择 | 返回进入前书架状态，不展示错误 |

## 5. 交互

| 入口 | 行为 |
| --- | --- |
| `选择文件` | 调起 Android 系统文件选择器 |
| `重新选择` | 再次调起系统文件选择器 |
| `完成` | 返回书架首页 |
| `返回书架` | 放弃当前失败结果并返回书架 |
| 失败结果行 | 展开或查看失败原因 |

## 6. 明确排除

- 不扫描整个存储。
- 不默认申请 `MANAGE_EXTERNAL_STORAGE`。
- 不要求登录或同步。
- 不静默丢弃失败文件。
- 不把取消选择当作错误。
