# 本地书导入视觉规格

状态：`PAGE_VISUAL_CANDIDATE`
文字稿状态：`FORMAL_DRAFT_READY`
唯一全局约束：`../../01-全局规范/00-唯一约束参考.md`

## 1. 当前视觉来源

当前已有 image2 生成的视觉 demo 候选：`图片/01-本地书导入-视觉demo候选.png`。该图已裁完整组件局部，可作为页面视觉候选继续评审；确认前不得标记为 `STITCH_READY`。

## 2. 继承 token

| 类别 | 继承来源 | 要求 |
| --- | --- | --- |
| 页面背景 | 书架链路 / App Shell | 不新增大面积渐变或营销背景 |
| 顶部栏 | App Shell | 返回、标题、图标槽位继承全局规格 |
| 卡片 | 书架链路卡片 token | 背景、边框、圆角、阴影统一 |
| 列表行 | App Shell 列表行 token | 文件名、状态、失败原因清晰分层 |
| 按钮 | 全局按钮 token | 主按钮和次按钮层级明确 |
| 状态组件 | 全局状态组件 | loading/error/empty 不重画页面外壳 |

## 3. 必须补图的状态

1. 默认导入说明页。
2. 解析中。
3. 导入完成。
4. 部分失败。
5. 全部失败。

系统文件选择器不需要自绘 UI 图，只需要在流程稿中说明调用系统能力。

## 4. 组件视觉约束

| 组件 | 约束 |
| --- | --- |
| `ImportIntroCard` | 卡片内包含标题、说明、支持格式、主按钮 |
| `ImportProgressCard` | 进度和当前文件名必须可读 |
| `ImportResultSummary` | 成功、跳过、失败三类结果分层展示 |
| `ImportResultRow` | 文件名优先，状态靠右或次级展示，失败原因可展开 |
| `BottomActionBar` | 主按钮靠右或视觉优先，次按钮弱化 |

## 5. 响应式约束

- 小屏优先保留标题、说明、主按钮和首个结果行。
- 长文件名单行省略，失败原因换行。
- 结果列表过长时滚动，不压缩触控区。

## 6. 验收门禁

进入 `STITCH_INPUT_READY_NEEDS_GENERATION` 前必须确认该候选图是否作为本页当前 UI 图，并完成截图叠放验收。候选未确认前保持 `PAGE_VISUAL_CANDIDATE`。
