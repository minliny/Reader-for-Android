# 本地书导入

## 页面内容
### 1. 页面目标

让用户明确本地导入的授权方式，选择文件后查看解析进度和逐项导入结果。

### 3. 默认结构

默认页包含：

1. 顶部栏：返回、标题 `导入本地书`。
2. 导入说明卡：说明系统文件选择器授权方式。
3. 支持格式说明：`TXT / EPUB` 等可支持格式。
4. 主按钮：`选择文件`。
5. 辅助说明：不会扫描整个存储。

### 4. 结果结构

导入结果页包含：

1. 结果摘要：成功、重复跳过、失败数量。
2. 文件结果列表：文件名、状态、失败原因。
3. 操作区：`重新选择`、`完成` 或 `返回书架`。

### 1. 组件清单

| 组件 | 用途 |
| --- | --- |
| `TopBar` | 返回和页面标题 |
| `ImportIntroCard` | 说明导入方式和隐私边界 |
| `SupportedFormatRow` | 展示支持格式和数量限制 |
| `FilePickerButton` | 调起 Android 系统文件选择器 |
| `ImportProgressCard` | 展示解析中状态 |
| `ImportResultSummary` | 汇总成功、跳过、失败数量 |
| `ImportResultRow` | 展示单个文件结果 |
| `ErrorGuidance` | 展示失败原因和下一步 |
| `BottomActionBar` | 承载完成、重新选择、返回书架 |

### 2. 组件约束

- `FilePickerButton` 是默认页唯一主操作。
- `ImportResultRow` 必须包含文件名、状态和失败原因；成功项可省略失败原因。
- `ImportProgressCard` 不允许重复触发选择文件。
- `BottomActionBar` 中主按钮只能有一个。

### 3. 状态组件

| 状态 | 组件 |
| --- | --- |
| default | `ImportIntroCard` + `FilePickerButton` |
| importing | `ImportProgressCard` |
| success | `ImportResultSummary` + 成功列表 + `完成` |
| partial | `ImportResultSummary` + 混合结果列表 + `重新选择` / `完成` |
| failed | `ErrorGuidance` + 失败列表 + `重新选择` / `返回书架` |

### 1. 默认页文案

| 位置 | 文案 |
| --- | --- |
| 页面标题 | `导入本地书` |
| 主标题 | `选择本地书籍` |
| 说明 | `通过系统文件选择器导入文件，不会扫描整个存储。` |
| 支持格式 | `支持 TXT、EPUB 等本地书籍文件` |
| 主按钮 | `选择文件` |

### 2. 解析中文案

| 位置 | 文案 |
| --- | --- |
| 标题 | `正在导入` |
| 说明 | `正在解析文件，请稍候` |
| 当前文件 | `正在解析：{fileName}` |

### 3. 结果文案

| 场景 | 标题 | 说明 | 主按钮 | 次按钮 |
| --- | --- | --- | --- | --- |
| 成功 | `导入完成` | `已将选中的书籍加入书架` | `完成` | - |
| 部分失败 | `部分文件导入失败` | `可以查看失败原因后重新选择` | `重新选择` | `完成` |
| 全部失败 | `导入失败` | `未能导入选中的文件` | `重新选择` | `返回书架` |

### 4. 文件结果文案

- `已导入`
- `已存在，已跳过`
- `格式不支持`
- `解析失败`

### 5. 文案规则

- 不使用技术字段名。
- 按钮文案必须对应真实结果，不用含糊的 `确定` 替代。
- 错误文案必须说明原因和下一步。

## 交互逻辑
### 1. 操作流

进入本地书导入
 -> 点击选择文件
 -> Android 系统文件选择器
 -> 取消：返回进入前书架状态
 -> 选择文件：进入解析中
 -> 全部成功：导入完成
 -> 部分失败：展示部分失败结果
 -> 全部失败：展示失败结果

### 2. 交互定义

| 操作 | 结果 |
| --- | --- |
| 返回 | 返回进入前页面并保留上下文 |
| 选择文件 | 调起 Android 系统文件选择器 |
| 取消系统选择器 | 返回进入前书架状态，不显示错误 |
| 重新选择 | 再次调起系统文件选择器 |
| 完成 | 返回书架首页 |
| 返回书架 | 放弃当前失败结果并返回书架首页 |
| 点击失败行 | 展开或查看失败原因 |

### 3. 上下文保留

从书架进入时，返回后必须保留：

- 当前分组。
- 当前排序和筛选。
- 书架滚动位置。
- 已有书籍列表状态。

### 4. 提交中规则

- 解析中禁止重复点击 `选择文件` 或 `重新选择`。
- 解析中允许系统返回，但需要保留已经完成的解析结果或给出中断确认。
- 失败后重新选择不得清空上一轮结果，除非用户完成新一轮选择。

### 5. 权限边界

本页不请求全盘扫描权限。文件访问只来自 Android 系统文件选择器返回的用户授权 URI。

### 1. 状态总表

| 状态 | 触发 | 展示 |
| --- | --- | --- |
| `default` | 进入页面 | 导入说明、支持格式、`选择文件` |
| `picker_cancelled` | 用户取消系统文件选择器 | 返回进入前书架状态，不展示错误 |
| `importing` | 已选择文件并开始解析 | 解析进度、当前文件、禁用重复导入 |
| `success` | 全部导入成功或重复跳过无错误 | 成功摘要、结果列表、`完成` |
| `partial_failed` | 部分成功、部分失败 | 混合结果列表、失败原因、`重新选择`、`完成` |
| `failed` | 全部失败 | 失败摘要、失败原因、`重新选择`、`返回书架` |
| `unsupported` | 文件格式不支持 | 文件行展示 `格式不支持` |
| `parse_error` | 文件解析失败 | 文件行展示 `解析失败` 和原因 |

### 2. 文件结果状态

| 状态 | 文案 |
| --- | --- |
| 成功 | `已导入` |
| 重复跳过 | `已存在，已跳过` |
| 格式不支持 | `格式不支持` |
| 解析失败 | `解析失败` |

### 3. 错误规则

- 不得静默丢弃失败文件。
- 部分失败时不得只显示成功状态。
- 全部失败时不得自动返回书架。
- 取消选择不是错误状态。

### 1. 功能边界

- 不扫描整个存储。
- 不默认申请 `MANAGE_EXTERNAL_STORAGE`。
- 不要求登录、会员、同步或云端账号。
- 不把本地导入放入书架 P0 闭环。
- 不在取消文件选择后展示错误页。

### 2. 数据边界

- 不静默丢弃失败文件。
- 不把重复文件当作失败。
- 不在全部失败时自动返回书架。
- 不隐藏失败原因。

### 4. 交互边界

- 解析中不得允许重复提交。
- 重新选择不得丢失上一轮失败原因，除非新一轮选择完成。
- 关键操作不得只依赖隐藏手势。
