## 审计标准

每张 UI 图片必须同时满足：

- 对应唯一页面。
- 页面文字稿已补齐。
- 图片内容与文字稿口径一致。
- 所有用户可见中文与文字稿一致。
- 按钮、返回、状态、清单在图片中有对应表达。
- 不包含已暂缓或已排除范围的内容。
- 不存在历史稿、多版本稿、示例稿混入最终稿。

## 01 书架首页封面模式

图片：

- `docs/ui-design/02-主标签页/书架/图片/01-书架首页-封面模式.png`

审计结论：候选可用，但不能冻结。

匹配点：

- 有 `书架` 标题。
- 有搜索和更多入口。
- 有继续阅读。
- 有最近更新。
- 有我的书架。
- 有封面模式。
- 有底部主 Tab。

不匹配：

- 图片展示 `默认`、`本地书`、`追更`，与当前“用户自定义分组为主”的文字口径不一致。
- 图片呈现三列封面布局，只能作为当前屏幕示例，不能作为固定布局规则。
- 图片没有表达“分组来自用户自定义”的规则。
- 图片没有表达空、加载、错误、部分刷新失败等状态。
- 与列表模式在模块大小、边框线条、圆角、阴影和局部颜色上不一致，不能作为同一组件系统的冻结状态。

处理建议：

- 保留为首页候选图。
- 修正分组示例，避免让 `默认`、`本地书`、`追更` 看起来像固定分组。
- 标注自适应布局，不把三列作为冻结规则。

## 03 书籍搜索首页
