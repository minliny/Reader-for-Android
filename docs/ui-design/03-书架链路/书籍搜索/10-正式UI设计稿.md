# 书籍搜索正式 UI 设计稿

状态：正式稿
等级：P0 / PAGE_VISUAL_BASELINE
适用页面：搜索首页、搜索结果、loading、empty、error
视觉基准：`图片/02-书籍搜索-结果页.png`

## 1. 设计目标

书籍搜索用于从书架或发现入口搜索书籍，展示结果，并进入详情、阅读或加入书架。

## 3. 页面结构

```text
BookSearchPage
  TopBar
    BackButton
    SearchBox
    SearchSubmit(optional)
  FilterRail
  SearchHomeContent / SearchResultList / StateContent
```

搜索页不是主 Tab，不显示底部主导航。

## 4. 搜索首页

搜索首页展示搜索框、搜索历史、推荐搜索或最近搜索、清空历史入口。不展示大面积营销内容。

## 5. 搜索结果

搜索结果展示查询词、筛选 chip、结果数量或来源摘要、结果卡片列表。每个结果卡片包含封面、书名、作者、来源摘要、最新章节或匹配说明，以及 `详情`、`阅读` 或 `加入书架` 操作。

## 6. 状态

| 状态 | 要求 |
| --- | --- |
| 首页 | 历史/推荐内容 |
| Loading | 搜索框保留，结果区骨架 |
| Empty | 显示查询词和建议 |
| Error | 显示错误原因和重试 |
| Offline | 保留历史搜索，提示网络不可用 |
| 长查询 | 搜索框单行处理，不压缩字号 |

## 7. 跳转

| 操作 | 目标 |
| --- | --- |
| 返回 | 回到来源页 |
| 点击结果卡 | `../书籍详情/00-说明.md` |
| 点击阅读 | 进入阅读入口 |
| 点击加入书架 | 加入成功后更新按钮状态 |
| 点击重试 | 使用当前查询重新搜索 |

## 8. 明确排除

- 不在搜索页展示复杂书源规则。
- 不展示账号、广告、会员。
- 不把搜索结果直接混入书架首页。
- 不在结果卡里放调试信息。

## 10. 验收标准

- 用户能清晰输入、修改和提交查询。
- 结果卡片密度适合移动端扫描。
- loading、empty、error 只替换结果区，不重画顶部搜索结构。
- 结果进入详情，配置在详情页处理。
