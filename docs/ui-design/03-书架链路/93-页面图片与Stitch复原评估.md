## Stitch 复原可行性

### 可以复原的部分

Stitch 可以基于“文字稿 + 本目录图片示例”复原以下内容：

- 书架首页的顶部栏、分组 chip、封面/列表两种布局、底部主导航。
- 书架空态、排序筛选底表、操作底表、分组管理页的页面结构。
- 搜索首页、搜索结果和 loading / empty / error 状态的上下文保留。
- 书籍详情默认态、loading 骨架、来源失效状态的结构关系。

### 不建议直接复原为冻结稿的部分

- 旧候选图 `01/02/03/04/05` 的文字和字段还没完全对齐当前页面稿。
- `01-书架首页-封面模式.png` 和 `02-书架首页-列表模式.png` 的模块尺寸、边框、圆角、阴影和局部颜色不一致，不能直接作为同一组件系统学习样本。
- `09-书架分组管理.png` 把多个状态合在单图里，Stitch 可能会误解为同屏常驻元素。
- 阅读入口两张是引用副本，不应让 Stitch 反向改写阅读模块视觉规则。
- 如果没有项目级 design system，Stitch 生成时容易漂移颜色、字体、圆角和密度。

### 推荐输入方式

每次只让 Stitch 复原一个页面或一组同源状态，不要一次喂完整 16 张图。

输入包应包含：

- 页面文字稿：对应 `01` 到 `12` 的单页 Markdown。
- 图片示例：本目录中对应 PNG。
- 禁止项：直接摘自每个页面稿的“不允许”段落。
- 状态边界：loading / empty / error / partial success / invalid source 的差异。
- 设计系统：使用项目级 design system；生成 prompt 中不要重复写颜色、字体和 token。

## Stitch 生成顺序建议

1. `01-书架首页-封面模式.png` + `02-书架首页-列表模式.png`：先复原书架首页组件和布局。
2. `06-书架空状态.png`、`08-书架排序与筛选.png`、`07-书籍操作底表.png`、`09-书架分组管理.png`：补书架周边状态/弹层。
3. `03-书籍搜索-首页.png`、`04-书籍搜索-结果页.png`：复原搜索链路。
4. `10/11/12`：复原搜索状态。
5. `05-书籍详情-默认页.png`、`13/14`：复原详情链路。
6. `15/16`：仅用于校验书架到阅读的入口，不作为阅读模块重绘输入。
