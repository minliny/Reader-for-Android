# 书籍详情正式 UI 设计稿

状态：正式稿
等级：P0 / PAGE_VISUAL_BASELINE
适用页面：书籍详情默认页、loading、来源失效 error
视觉基准：`图片/01-书籍详情-默认页.png`

## 1. 设计目标

书籍详情承接所有单本书配置修改，同时提供阅读入口、目录预览和简介。书架页底表中的 `修改` 必须进入本页。

## 3. 页面结构

```text
BookDetailPage
  TopBar
  BookHeader
    Cover
    Title
    Author
    SourceStatus
    Meta
  PrimaryActions
  SourceStatusCard(optional)
  ChapterPreviewCard
  IntroCard
  ConfigEntrances
```

## 4. 内容规则

- 默认头部展示封面、书名、作者、来源、字数/章节、更新时间。
- 主操作固定优先阅读，不把配置操作放在主视觉中心。
- 目录预览展示最近章节或前几章，点击进入目录或直接定位章节。
- 简介默认展示摘要，长内容折叠。
- 来源失效时，在来源状态卡中说明，提供修复或查看来源入口。

## 5. 交互

| 入口 | 行为 |
| --- | --- |
| 返回 | 回到来源页并保留上下文 |
| 开始/继续阅读 | 进入沉浸阅读 |
| 加入书架 | 加入成功后变为已在书架 |
| 修改配置 | 在本页配置区完成，不回到底表扩展 |
| 目录行 | 进入对应章节 |
| 查看全部目录 | 进入目录页 |
| 来源异常修复 | 进入 P1 换源或来源修复链路 |

## 6. 状态

| 状态 | 要求 |
| --- | --- |
| 默认 | 完整展示头部、阅读操作、目录预览、简介 |
| Loading | 顶部栏保持，头部和卡片骨架 |
| 来源失效 | 保留详情内容，来源状态卡显示错误和修复入口 |
| 未加入书架 | 展示加入书架次级操作 |
| 已加入书架 | 展示已在书架和配置入口 |

## 7. 明确排除

- 不做营销落地页。
- 不展示账号、会员、广告。
- 不把所有书源配置塞进书架底表。
- 不把阅读控制层放进详情页。

## 9. 验收标准

- 用户能一眼找到阅读主操作。
- 用户能理解书籍是否已在书架。
- 配置修改入口在详情页内完成。
- 来源失效不破坏页面结构。
- 页面视觉继承书架首页，不自创一套颜色和卡片样式。
