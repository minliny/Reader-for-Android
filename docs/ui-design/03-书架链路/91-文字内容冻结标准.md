# 文字内容冻结标准

## 页面定位

本文件用于评估书架主 Tab 所有页面的中文文字稿是否可以冻结。

评估对象包括：

- 页面标题。
- 页面说明。
- 分区标题。
- 列表字段。
- 按钮。
- 菜单项。
- 空状态。
- 加载状态。
- 错误状态。
- 确认弹窗。
- 返回逻辑。
- 结果清单。
- 操作反馈。

## 冻结标注

每个页面文字稿必须在文件顶部或完成条件中标注一个状态：

- `文字稿状态：未开始`
- `文字稿状态：草稿`
- `文字稿状态：待补清单`
- `文字稿状态：待产品确认`
- `文字稿状态：已冻结`

只有 `文字稿状态：已冻结` 可以进入最终图片验收。

## 标准格式

每个页面文字稿必须包含以下章节：

```text
页面定位
当前 source
入口
返回
页面结构
主要文案
按钮与操作
状态文案
错误与空状态
不允许
完成条件
文字稿状态
```

如果某页面没有某类内容，需要写明 `无`，不能省略章节。

## 主要文案清单

每个页面必须列清楚所有对用户可见的中文：

- 顶部栏标题。
- 分区标题。
- 卡片标题。
- 字段名称。
- 标签名称。
- 占位提示。
- 说明文字。
- 错误提示。
- 成功提示。
- 二次确认文案。
- 菜单项名称。
- 底表标题。
- Toast 或 Snackbar 文案。

## 按钮标准

每个按钮必须明确：

- 按钮文案。
- 按钮类型：主按钮、次按钮、危险按钮、文本按钮、图标按钮。
- 默认态。
- 禁用态。
- 加载态。
- 点击后结果。
- 点击失败反馈。

图标按钮也必须有中文语义名称，例如 `搜索`、`更多`、`返回`、`排序`。

## 返回逻辑标准

每个页面必须明确：

- 从哪里进入。
- 点击系统返回时去哪里。
- 点击顶部返回时去哪里。
- 操作完成后去哪里。
- 操作取消后去哪里。
- 是否保留上一次筛选、搜索词、分组、排序和滚动位置。

## 结果清单标准

涉及批量结果或检测结果的页面必须列出：

- 成功项。
- 失败项。
- 跳过项。
- 部分成功。
- 失败原因。
- 可重试操作。
- 返回或完成操作。

不得只写“操作完成”。

## 状态文案标准

每个状态必须独立写清：

- loading。
- empty。
- error。
- permission。
- offline。
- partial success。
- invalid source。

如果页面不需要某状态，需要标注 `不适用`。

## 书源信息展示标准

书源信息默认不在书架首页展示。

允许展示书源信息的场景：

- 搜索书籍并准备添加到书架。
- 添加至书架时配置默认书源。
- 书籍信息编辑过程。
- 阅读出现乱码、章节不全、章节错乱或内容加载失败。
- 来源失效后重新选择来源。

书源信息需要包含：

- 书源名称。
- 章节数量。
- 目录状态。
- 最新章节。
- 更新时间或检测时间。
- 当前默认书源标记。

## 冻结流程

1. 先按页面补齐 `标准格式` 中的所有章节。
2. 再逐页列出所有用户可见中文。
3. 对按钮、返回、状态、结果清单逐项验收。
4. 产品确认后将页面标为 `文字稿状态：已冻结`。
5. 冻结后才能进入最终图片验收。
6. 图片验收通过后再标注 `图片状态：已冻结`。

## 冻结完成定义

页面同时满足以下条件才算文字稿冻结：

- 所有用户可见中文已列出。
- 所有按钮有文案、状态和点击结果。
- 所有入口和返回路径已明确。
- 所有空、加载、错误、权限、失效状态已明确。
- 所有清单类结果有成功、失败、跳过和重试说明。
- 不存在待定口径。
- 产品确认通过。
