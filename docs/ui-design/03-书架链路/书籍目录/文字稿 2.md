# 书籍目录

## 页面内容
### 页面结构

 ChapterRow*

### 内容规则

- 顶部标题为 `目录`。
- 摘要栏展示书名、当前来源和章节数量。
- 当前阅读章节置顶或在列表中高亮，显示阅读进度。
- 章节行展示章节序号、标题、状态：当前阅读、未读、已读、新。
- P0 目录页不设置顶部筛选区；`未读`、`已读`、`新` 只作为章节行状态出现。
- 当前阅读章节通过浅蓝背景、文字状态和右侧进度表达，不使用独立左侧图标。
- 本页不展示换源入口；换源仍回到书籍详情头部或换源页处理。

### 层级关系

- `查看目录` 只进入本页。
- `展开更多章节` 不再作为书籍详情页入口。
- 章节行进入阅读沉浸页。

### 组件

| 组件 | 用途 |
| --- | --- |
| `BookDirectory.TopBar` | 返回和标题 |
| `BookDirectory.SummaryBar` | 书名、来源、总章节弱信息 |
| `BookDirectory.ChapterRow` | 单章节入口 |
| `BookDirectory.CurrentRow` | 当前阅读章节高亮 |

### 组件规则

- 章节列表是主要内容，不使用卡片套卡片。
- 当前阅读行可以使用浅蓝底，但不得变成大面积营销卡。
- 当前阅读行不配置左侧图标槽位，依靠浅蓝背景、章节标题、`当前阅读` 状态和右侧进度建立识别。

- 页面标题：`目录`
- 摘要来源：`当前来源：起点中文网`
- 摘要章节：`共 1862 章`
- 当前阅读：`当前阅读`
- 状态：`未读`、`已读`、`新`
- loading：`正在加载目录`
- empty：`暂无目录`
- error：`目录加载失败`
- 操作：`重试`、`返回详情`

## 交互逻辑
### 交互

| 操作 | 结果 |
| --- | --- |
| 返回 | 回到书籍详情页，保留详情页滚动位置 |
| 点击当前阅读章节 | 进入阅读沉浸页并定位当前章节 |
| 点击普通章节 | 进入阅读沉浸页并定位选中章节 |

### 返回栈

从书籍详情进入本页后，阅读返回应优先回到阅读来源上下文；若用户从阅读返回详情，再回到目录时保留目录滚动位置。

### 状态

| 状态 | 规则 |
| --- | --- |
| default | 展示摘要栏和章节列表 |
| loading | 保留顶部栏和摘要栏，章节列表显示骨架 |
| empty | 展示 `暂无目录`，保留返回和重试入口 |
| error | 展示目录加载失败、重试和返回详情 |

### 边界

- 来源失效不在本页做换源，返回详情或换源页处理。
- 章节数量很大时使用列表虚拟化，保持章节定位和滚动体验稳定。

- 不在本页重复展示书籍详情页的主操作按钮。
- 不在本页展示底部主 Tab 导航。
- 不把章节列表做成多张卡片堆叠。
- 不保留 `展开更多章节` 作为第二个完整目录入口。
- 不在 P0 顶栏生成搜索、更多、筛选、排序等未定义入口。
- 不在章节列表上方生成筛选 chip 行。
- 不在当前阅读章节左侧生成独立图标。
