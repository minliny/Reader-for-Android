# 书籍操作组件

## 组件内容
### 组件目标

对书架中的单本书提供上下文操作入口。它不是独立页面，不允许作为 `book-action-sheet` 路由打开；只能集成在书架管理态、书籍详情页或书籍长按浮层中。

### 入口

书架管理态、书籍详情页操作列表、书籍封面/列表项长按浮层。

### 组件区域

1. 来源页内容区：保留来源页滚动位置、筛选条件和当前书籍上下文。
2. 操作列表区：提供修改信息、移动分组、移除书架等单书操作。
3. 确认弹窗区：危险操作必须进入 `DialogHost` 二次确认。
4. 反馈区：loading、error、offline 使用来源页的状态容器或轻量 toast。

### 专项约束补充

#### 结构执行规则

- 组件不得生成独立页面、独立返回顶栏或独立路由。
- 操作列表、底表/浮层和确认弹窗必须分层实现，不能合并成单个自由排版容器。

### 组件清单

操作列表、书籍摘要、修改操作、移动分组操作、删除操作、删除确认弹窗。

### 组件约束

- 图标只表达明确动作；无法确认语义时使用文字按钮。
- 危险操作使用危险色和二次确认，不与普通主按钮同层级混淆。

### 专项约束补充

#### 页面专属组件

- `BookSummary`
- `ActionRow.Delete`
- `ActionRow.Edit`
- `ActionRow.Group`
- `ConfirmDialog`

#### 组件执行规则

- 未在本清单出现的组件只能作为状态反馈或系统控件出现，不能成为新的主功能入口。

### 核心文案

- `修改`
- `删除`
- `删除书架记录？`
- `从书架移除，不会删除本地文件或网络来源`
- `取消`

### 通用反馈文案

- `正在加载`
- `加载失败，请重试`
- `网络不可用，请稍后重试`
- `保存成功`
- `操作失败，请重试`
- `取消`
- `确认`

### 文案规则

- 中文文案短句优先，不使用技术字段名。
- 危险操作必须说明影响范围。
- 不显示账号、社区、推荐算法等当前范围外文案。

### 专项约束补充

#### 必须覆盖的页面文案

- `删除`
- `修改`
- `移除书架`
- `确认移除`

#### 文案执行规则

- 生成稿不得替换这些核心中文文案。
- 状态文案必须说明原因和下一步，不能只写通用失败。
- 按钮文案必须与交互结果一致，不能使用含糊的 `确定` 替代关键动作。

## 交互逻辑
### 主要操作

修改在书籍详情页内编辑信息；移动分组进入分组选择或分组管理；删除打开二次确认并执行移除书架。

### 返回规则

- 组件关闭时回到来源页并保留来源页滚动位置、筛选条件和当前书籍上下文。
- 弹层、底表、确认框优先关闭当前浮层，再返回上级页面。
- 操作提交中禁止重复点击主按钮。

### 反馈规则

- 成功反馈使用轻量 toast 或行内状态，不改变页面结构。
- 失败反馈保留用户输入和当前上下文，提供重试入口。
- 网络不可用时只阻断依赖网络的动作，不阻断本地查看和关闭页面。

### 手势规则

- 如使用底表，底表支持点击遮罩、下滑、系统返回关闭。
- 列表页支持纵向滚动，不额外定义横向手势。
- 长按只用于已有明确业务入口的书籍或列表项，不新增隐藏功能。

### 专项约束补充

#### 页面关键操作

- 删除=移除书架，必须二次确认或危险反馈
- 修改=在书籍详情页修改配置
- 移动分组=在当前书架上下文中改变书籍归属
- 点击遮罩/下滑/返回关闭底表

#### 交互执行规则

- 所有操作必须保留来源页上下文，包括滚动位置、筛选条件、当前章节或当前设置值。
- 生成稿不得把书籍操作做成单独页面；不得出现 `book-action-sheet` 业务路由。
- 提交中禁用重复点击；失败时保留用户已输入或已选择内容。
- 浮层关闭顺序为：先关闭当前浮层，再返回上级页面。

### default

展示当前页面的完整默认内容，所有主操作可见且可达。

### loading

- 首次进入可使用骨架屏或居中加载。
- 局部刷新只替换对应局部内容，不清空整页。

### empty

- 空状态必须说明原因，并给出一个主行动作。
- 空状态不得显示错误语气。

### error

- 错误状态保留返回入口。
- 可恢复错误提供 `重试`；不可恢复错误给出原因和下一步。

### offline

- 网络不可用时保留本地数据展示。
- 需要联网的按钮进入禁用或点击后提示 `网络不可用，请稍后重试`。

### permission

- 需要系统权限时先说明用途，再提供去授权入口。
- 未授权不得进入空白页。

### 专项约束补充

#### 页面状态定义

- `default`：只展示删除和修改两项
- `danger`：删除行为使用危险色并确认
- `loading`：移除书架提交中禁用重复点击
- `error`：删除失败保留底表并提示重试

#### 状态执行规则

- 状态层不得改变页面主结构，只替换对应内容区或显示可恢复反馈。

### 功能禁止

- 不新增当前页面包未声明的入口、按钮或跳转。
- 不引入账号、社区、会员、推荐流、广告或云端商业化入口。

### 交互禁止

- 不在提交中允许重复点击。
- 不在关闭弹层后丢失来源页筛选、分组、排序和滚动位置。
- 不用隐藏手势承载唯一关键操作。

### 专项约束补充

#### 页面专属禁止项

- 不得出现换源、目录、分组、阅读设置等配置项
- 不得生成三个以上操作按钮
- 不得把修改做成底表内表单
