## 4. 页面生成约束

底表是书架页面上的浮层状态，不是独立全屏页面。背景保留书架页面视觉并叠加遮罩；底表从底部锚定，包含拖拽柄、书籍摘要、操作行、安全区。

操作只包含两项：`修改` 和 `删除`。`修改` 进入书籍详情页，在书籍详情页修改全部配置信息；`删除` 对应移除书架。底表本身只承载入口，不承载书籍配置表单。

书籍摘要区域必须保留封面、书名、作者和当前章节/进度摘要的层级。操作区使用两条列表行，不使用独立卡片组；行高、分隔线、左右内边距以组件局部为准。

响应式以 `833 x 1888 px` 为基准。常规手机底表宽度贴合视口；高度由内容决定并保留底部安全区；短屏时底表内容区可滚动，但操作项仍保持同一列表容器结构。
