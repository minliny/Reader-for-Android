# 书籍操作底表正式 UI 设计稿

状态：正式稿
等级：P0 / COMPONENT_MASTER
适用场景：书架首页书籍更多按钮、长按书籍条目
视觉参考：`图片/01-书籍操作底表.png`

## 1. 设计目标

书籍操作底表只承担书架级最小操作，减少误操作和配置分散。

本底表只允许：

- `修改`：进入书籍详情页。
- `删除`：从书架移除该书。

所有配置、来源、缓存、分组、导出、目录刷新等操作都在书籍详情页或对应管理页处理。

## 3. 结构

```text
BookActionSheet
  Scrim
  SheetContainer
    DragHandle
    BookSummary
    ActionRow("修改")
    ActionRow("删除")
  DeleteConfirmDialog(optional)
```

## 4. 交互

| 操作 | 结果 |
| --- | --- |
| 点击遮罩、下滑、系统返回 | 关闭底表，回到原书架状态 |
| 点击 `修改` | 关闭底表并进入 `../书籍详情/00-说明.md` |
| 点击 `删除` | 打开删除确认弹窗 |
| 确认删除 | 移除书架记录，关闭弹窗和底表 |
| 取消删除 | 关闭确认弹窗，回到底表 |

删除失败时不关闭底表，保留用户上下文并显示错误反馈。

## 5. 状态

| 状态 | 要求 |
| --- | --- |
| 默认 | 摘要 + `修改` + `删除` |
| 摘要 loading | 摘要区显示骨架，操作行可禁用 |
| 修改不可用 | `修改` 禁用并显示简短原因 |
| 删除确认 | 居中弹窗，危险主按钮 |
| 删除提交中 | 删除按钮 loading，禁止重复提交 |
| 删除失败 | 保留底表，提示 `删除失败，请重试` |

## 6. 明确排除

- 不出现 `换源`、`缓存`、`移动分组`、`导出`、`刷新目录`、`继续阅读`。
- 不出现账号、分享、推荐或社区入口。
- 不把删除解释为删除本地文件或网络来源。

## 8. 验收标准

- 默认态只出现 `修改`、`删除` 两项。
- 摘要区能识别书名、作者和当前章节/进度。
- `删除` 有二次确认。
- Stitch 不生成多余配置操作。
