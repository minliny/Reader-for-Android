# 书架分组管理

状态：正式 UI 设计稿已建立
主入口：`10-正式UI设计稿.md`

## 页面定位

书架分组管理用于管理用户自定义分组，不删除书籍本身。

`全部` 是系统聚合入口，不参与重命名、删除或排序。未加入任何自定义分组的书籍归入系统兜底状态 `未分组`。

## 当前 source

候选图：`docs/ui-design/03-书架链路/分组管理/图片/01-书架分组管理.png`

图片状态：页面参考；如与正式稿冲突，以 `10-正式UI设计稿.md` 和 `08-分组管理视觉规格.md` 为准。

## 入口

- 书架右上 `更多` 菜单。
- 书架分组筛选区域的管理入口。
- 当前分组为空状态中的 `管理分组`。

## 返回

- 顶部返回：返回书架首页。
- 系统返回：返回书架首页。
- 新建、重命名、删除或排序完成后，返回分组管理页并刷新列表。
- 离开分组管理页后，书架首页保留进入前的展示模式、排序和滚动位置。

## 页面结构

顶部栏：

- 图标按钮：`返回`
- 标题：`分组管理`
- 文本按钮：`新建`

主体：

- 分组列表。
- 每行显示分组名称、书籍数量、拖动排序手柄、更多操作。
- `全部` 不显示在可管理列表中，或以只读说明形式展示。
- `未分组` 可显示数量，但不可重命名和删除。

弹窗和底表：

- 新建分组输入弹窗。
- 重命名分组输入弹窗。
- 删除分组确认弹窗。
- 删除失败反馈。

## 主要文案

- 页面标题：`分组管理`
- 新建按钮：`新建`
- 输入标题：`新建分组`
- 输入占位：`输入分组名称`
- 重命名标题：`重命名分组`
- 删除确认标题：`删除分组？`
- 删除确认说明：`分组内的书籍不会被删除，将移动到未分组。`
- 删除确认按钮：`删除`
- 取消按钮：`取消`
- 保存按钮：`保存`

- 删除失败提示：`删除失败，请重试`
- 排序保存提示：`分组顺序已更新`

## 按钮与操作

- `返回`：图标按钮，返回书架首页。
- `新建`：文本按钮，打开新建分组输入弹窗。
- `保存`：主按钮，保存新建或重命名结果。
- `取消`：次按钮，关闭弹窗或底表。
- `删除`：危险按钮，确认删除当前分组。
- 行级 `更多`：图标按钮，打开重命名和删除操作。
- 拖动手柄：调整自定义分组顺序，松手后保存。

按钮状态：

- 分组名为空时 `保存` 禁用。
- 分组名重复时 `保存` 禁用，并提示 `分组名称已存在`。
- 保存中显示加载态，避免重复提交。

## 状态文案

loading：

- 标题保留，列表区域展示骨架。

empty：

- 标题：`还没有自定义分组`
- 说明：`可以新建分组来整理书架`
- 主按钮：`新建分组`

error：

- 标题：`分组加载失败`
- 说明：`请稍后重试`
- 主按钮：`重试`

permission：

- 不适用。

offline：

- 可展示本地分组。
- 同步失败时提示：`分组同步失败，已保留本地修改`

partial success：

- 排序部分保存失败时提示：`部分分组顺序保存失败`

invalid source：

- 不适用。

## 错误与空状态

- 删除分组失败时，分组列表保持原样。
- 新建失败时，保留输入内容。
- 重命名失败时，保留原分组名称。
- 排序失败时，恢复上一次成功顺序。

## 不允许

- 不删除书籍本身。
- 不允许删除 `全部`。
- 不允许把 `全部` 当作普通自定义分组。
- 不在分组管理页展示书源规则。

## 完成条件

- `未分组` 作为系统兜底命名。
- 默认主图覆盖分组列表；新建、重命名、删除确认和空列表作为后续状态图。
