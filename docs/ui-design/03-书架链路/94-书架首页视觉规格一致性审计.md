# 书架首页视觉规格一致性审计

审计日期：2026-06-13

## 结论

`01-书架首页-封面模式.png` 和 `02-书架首页-列表模式.png` 不能作为同一套组件的两个冻结状态直接使用。两图整体风格接近，但多个 UI 组件的尺寸、边框、圆角、间距、阴影和局部颜色不一致；如果直接交给实现或 Stitch，会导致封面模式和列表模式被复原成两套相似但不同的组件系统。

## 当前图片

- `docs/ui-design/02-主标签页/书架/图片/01-书架首页-封面模式.png`
- `docs/ui-design/02-主标签页/书架/图片/02-书架首页-列表模式.png`

两张图画布均为 `853×1844 px`，比例一致，但内部组件并非同一母版派生。

## 主要差异

| 区域 | 封面模式 | 列表模式 | 风险 |
| --- | --- | --- | --- |
| 分组 chip | 宽度、圆角、激活色与列表图近似但非完全一致 | 激活色略偏亮，非激活 chip 的文字/背景对比与封面图不同 | Stitch/实现会生成两套 chip 样式 |
| 顶部继续阅读卡片 | 左侧卡片和右侧最近更新卡片高度、边距较紧 | 同区域高度和内部封面比例不同 | 同一模块在模式切换时会跳动 |
| 我的书架标题行 | 封面模式右侧显示 `封面`、排序、设置 | 列表模式右侧显示 `列表`、排序、设置，但图标位置和文字锚点不同 | 模式切换控件难以建立统一组件 |
| 书籍卡片 | 三列卡片，封面较大，卡片内文字纵向堆叠 | 单列横向行，封面较小，右侧进度和更多按钮 | 这是正常布局差异，但卡片圆角/阴影/边框应共用 token |
| 卡片边框 | 边框极弱，部分卡片更像无边框阴影卡 | 行卡片边框/阴影更明显 | 边框线条粗细和阴影层级不统一 |
| 进度条 | 卡片内进度条短，蓝色和轨道色明确 | 列表行进度条位置靠右，长度和线宽不同 | 进度组件需要统一高度、圆角和颜色 |
| 底部主导航 | 高度、圆角、图标大小与列表图接近 | 与封面图近似但阴影和垂直留白略不同 | 主导航必须是同一组件，不能按页面重画 |

## 文字稿中的规格缺口

现有书架首页文字稿已经定义页面结构、入口、返回、状态和不允许项，但缺少可冻结的视觉规格：

- 未定义 `PageBackground` 的最终颜色。
- 未定义 `ShelfChip` 的高度、水平 padding、圆角、边框粗细、激活/未激活颜色。
- 未定义 `BookCard` 和 `BookListItem` 的统一圆角、边框、阴影、内边距和最小高度。
- 未定义封面图比例、最小/最大尺寸、封面圆角。
- 未定义进度条高度、轨道色、填充色、圆角和长度规则。
- 未定义顶部继续阅读/最近更新模块是否使用同一 `PanelCard` token。
- 未定义底部主导航栏高度、圆角、阴影、图标尺寸和文字字号。
- 未定义模式切换时哪些组件保持固定尺寸，哪些组件允许重排。

## 初步统一建议

这些数值先作为重绘约束，不能直接视为已冻结 token：

| Token | 建议 |
| --- | --- |
| 页面背景 | 使用一个暖白背景 token，不允许逐图漂移 |
| 卡片背景 | 使用一个卡片 surface token |
| 卡片边框 | 统一 1px 低对比暖灰描边 |
| 卡片圆角 | 首页模块卡片和书籍卡片统一一个半径层级；封面缩略图使用较小半径 |
| 阴影 | 首页所有卡片使用同一低阴影层级，列表行和封面卡片不得分裂 |
| 进度条 | 统一高度、圆角、轨道色、填充色；长度由容器决定 |
| 分组 chip | 统一高度、padding、圆角、激活色、未激活背景和边框 |
| 底部导航 | 作为 App Shell 组件复用，不允许在封面/列表两图中分别生成 |

## 对 Stitch 的影响

当前两张图可以作为结构参考，但不适合直接让 Stitch 学习最终组件 token。用于 Stitch 时必须在 prompt 中明确：

- 封面模式和列表模式是同一书架首页的两种状态。
- 顶栏、分组 chip、顶部信息模块、底部导航必须复用同一组件样式。
- 书籍卡片和书籍列表项允许布局不同，但颜色、边框、圆角、阴影、封面圆角和进度条 token 必须统一。
- 以文字稿和统一 token 约束修正图片中的漂移，不得逐图照抄。

- `02-书架首页-列表模式.png`：结构可用，视觉规格不可冻结。
