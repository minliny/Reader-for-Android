# UI 设计图转前端设计稿优先级

当前共有 30 张 `UI设计图.png`。转换顺序应先扩大公共组件覆盖，再处理差异最大的页面。

## P0：公共骨架闭环

1. `02-主标签页/书架`：已完成，作为书籍卡片、Chip、MainNav 种子。
2. `04-阅读链路/阅读控制层`：已完成，作为底表、模块导航、滑杆种子。
3. `02-主标签页/设置`：已完成，补 `SettingRow`、状态 badge、设置分组和主标签页设置骨架。
4. `03-书架链路/书架空状态`：已完成，补空状态卡、主/次操作按钮、导入入口规则。
5. `03-书架链路/书籍搜索`：补 `SearchBar`、`SearchResultItem`。

## P1：书架链路完整闭环

6. `03-书架链路/书籍详情`
7. `03-书架链路/书籍目录`
8. `03-书架链路/排序与筛选`
9. `03-书架链路/书籍操作底表`
10. `03-书架链路/分组管理`
11. `03-书架链路/本地书导入`

理由：这些页面能最大化复用 `BookCard`、`BookRow`、`BottomSheet`、`Chip`、`SearchBar`，同时补齐书架域缺口。

## P2：阅读链路面板体系

12. `04-阅读链路/目录与书签`
13. `04-阅读链路/阅读外观`
14. `04-阅读链路/朗读`
15. `04-阅读链路/阅读设置`
16. `04-阅读链路/自动翻页`
17. `04-阅读链路/内容搜索`
18. `04-阅读链路/内容替换`
19. `04-阅读链路/阅读入口`
20. `04-阅读链路/沉浸阅读`
21. `04-阅读链路/换源`

理由：阅读控制层已完成，但这些页面需要把简化展开态升级成完整 panel 和 overlay 组件。`换源` 是横向画布，应放在阅读基础面板之后处理。

## P3：主标签页补齐

22. `02-主标签页/发现`
23. `02-主标签页/RSS`

理由：需要 `ListRow`、`RSSSourceItem`、订阅状态、频道卡片等新增组件。

## P4：设置链路补齐

24. `05-设置链路/App通用设置`
25. `05-设置链路/书架与搜索设置`
26. `05-设置链路/书源管理`
27. `05-设置链路/同步与备份`
28. `05-设置链路/缓存管理`
29. `05-设置链路/隐私与权限`
30. `05-设置链路/关于与反馈`

理由：设置页族复用度高，适合在 `SettingRow`、`Switch`、`PermissionState` 稳定后批量转换。

## 每页完成定义

- 有独立 `frontend-input/` 包。
- 有数据 fixture 和 renderer。
- 有可打开的 `preview.html`。
- 有状态矩阵或明确说明无需状态矩阵。
- 加入总 `manifest.json`。
- 自动验证通过。
