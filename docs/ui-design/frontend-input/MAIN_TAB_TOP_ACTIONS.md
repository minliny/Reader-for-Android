# 主标签页顶部动作（Main Tab Top Actions）

## 统一原则（Unified Rule）

- 主标签页顶部动作属于 `MainTabShell.AppTopBar`，只处理当前根 Tab 的一级入口。
- 顶部动作必须使用显式动作名：`search`、`more`。
- `refresh` 不放在主 Tab 顶栏；刷新属于内容区局部动作，例如发现来源状态条、RSS 订阅概览或列表区。
- 未完成的次级流程只保留入口语义和轻量反馈，不生成临时全屏页面或临时菜单。
- `more` 不是全局同一菜单；每个主 Tab 必须按该页面文档声明自己的入口边界。

## 页面规划（Page Plan）

| 页面（Page） | 顶部动作（Top Actions） | `more` 原文档状态（Original Planning Status） | 当前 demo 允许行为（Allowed Demo Behavior） | 不允许（Not Allowed） |
|---|---|---|---|---|
| 书架（Bookshelf） | `search`、`more` | `书架/文字稿.md` 明确“更多图标：打开书架级更多菜单，后续定义”。 | 可打开书架级更多菜单；当前只承载已规划书架链路入口：批量管理、分组管理、本地书导入、排序与筛选。书架显示设置仍由“我的书架”右侧齿轮进入。 | 不把书架显示设置塞进顶部更多；不打开书籍单本操作底表；不临时生成未规划菜单项。 |
| 发现（Discover） | `search`、`more` | `发现/文字稿.md` 明确顶部有更多，但交互表写“保留入口，菜单暂缓”；范围边界写来源选择、更多分类、发现设置、榜单详情、来源检测暂缓。 | 只能显示轻量反馈，说明发现更多入口保留；不展开菜单。 | 不生成来源选择底表、更多分类页、发现设置页或榜单详情页；刷新仍在内容区。 |
| RSS | `search`、`more` | `RSS/文字稿.md` 明确顶部有更多，但交互表写“保留入口，菜单暂缓”；RSS 搜索、订阅管理、添加订阅源、条目详情、分组管理暂缓。 | 只能显示轻量反馈，说明 RSS 顶部更多入口保留；不展开菜单。 | 不生成订阅管理、添加订阅源、条目详情、分组管理菜单；行内三点菜单也只保留入口。 |
| 设置（Settings） | `search`、`more` | `设置/文字稿.md` 明确“更多图标：保留入口，导入导出/恢复默认后续设计”；状态表写“搜索/更多当前只保留入口，不补弹层状态”。 | 只能显示轻量反馈，说明设置更多入口保留；不展开菜单。 | 不在设置首页直接执行导入、导出、恢复默认、清空、删除等操作；破坏性操作必须进入二级页或确认弹窗。 |

## 搜索动作（Search Actions）

| 页面（Page） | `search` 当前行为（Current Behavior） |
|---|---|
| 书架（Bookshelf） | 进入书籍搜索页。 |
| 发现（Discover） | 进入书籍搜索页。 |
| RSS | 仅保留 RSS 搜索入口，RSS 搜索完整页暂缓。 |
| 设置（Settings） | 仅保留设置内搜索入口，设置搜索完整页暂缓。 |
