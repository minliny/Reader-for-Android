# 阅读控制层交互模型（Reader Control Interaction Model）

本文只定义阅读控制层里的交互分级、页面归属和返回关系。它用于避免把阅读控制层做成静态截图、PPT 页面或所有按钮都同级跳转。

## 总原则（Overall Rule）

- 阅读正文底层（Reading Surface）始终归 `ReaderShell` 持有，控制层、快捷面板、模块面板和完整控制页都不能重建正文上下文。
- 阅读控制层按钮不是统一跳页面；必须先判断是即时动作、快捷面板、模块面板、完整控制页还是流程页。
- 只有“常用项可快速改、复杂项需要展开”的功能，才采用“控制层按钮 -> 快捷窗口 -> 完整控制页”的二段式逻辑。
- 目录、朗读、界面、设置四个模块按钮采用“快捷控制窗 + 可展开完整页”模型：首次点击进入对应快捷控制窗，重复点击当前 active 模块关闭快捷控制窗；点击底栏顶部小横条并向上拖动，将底栏展开为该模块的完整控制页。
- 控制层打开后，点击正文中部热区不是进入模块，而是隐藏整个控制层并回到沉浸阅读。
- 换源（Source Switch）不是设置模块，但也必须遵守“短窗口先处理、复杂内容再展开”的原则：默认打开阅读内换源快捷窗口；需要完整来源对照时再进入 `FlowShell`。

## 快捷控制窗与完整设置页关系（Quick Window and Full Page Relationship）

新的阅读控制层交互以底部控制层为主轴，而不是以页面跳转为主轴。

```text
ReaderShell
├─ ReadingSurface                        阅读正文底层，始终保持同一份 DOM / Compose 节点
├─ ReaderTopBar                           顶部阅读控制栏
├─ ReaderQuickWindow                      快捷控制窗，位于正文可用区内
├─ BottomControlLayer                     底部控制层
│  ├─ DragHandle                          顶部小横条，点击或上拉展开
│  ├─ QuickSummary / ChapterControls       默认章节与进度控制
│  └─ ReaderModuleNavigation              目录、朗读、界面、设置四个按钮
└─ ReaderFullControlSheet                 由 BottomControlLayer 上拉展开的完整控制页
```

状态关系：

| 状态（State） | 进入方式（Entry） | 显示内容（Visible Content） | 退出方式（Exit） |
|---|---|---|---|
| 沉浸阅读（Immersive） | 打开书籍或隐藏控制层 | 只有 `ReadingSurface` | 点击正文中部打开控制层 |
| 默认控制层（Control Default） | 沉浸页点击中部 | 顶部栏、章节进度、快捷操作、亮度、底部四按钮和小横条 | 点击正文中部回沉浸；返回退出阅读 |
| 快捷控制窗（Quick Window） | 点击目录 / 朗读 / 界面 / 设置 / 搜索 / 自动翻页 / 替换 / 换源 | 当前模块的 P0 常用项；底栏和四按钮保持原位 | 重复点击 active 模块、点击正文空白或系统返回 |
| 完整控制页（Full Control Sheet） | 点击小横条或上拉底栏 | 当前模块的 P0 + P1 完整配置；底栏展开为全高 sheet | 下拉小横条回快捷窗；系统返回先折叠回快捷窗 |

布局规则：

- 快捷控制窗只占正文可用区，不压到底部四按钮，不与右侧亮度栏重叠。
- 底栏顶部小横条永远属于 `BottomControlLayer`，不属于快捷控制窗；它表示“当前底栏可展开”。
- 小横条点击等价于展开当前 active 模块；如果没有 active 模块，则展开默认章节控制完整页。
- 完整控制页由底栏向上展开，覆盖正文大部分区域，但不重建 `ReadingSurface`，只改变覆盖层高度。
- 完整控制页保留同一个 `ReaderModuleNavigation`，四个模块按钮的位置、尺寸、间距和 active 规则不变。
- 快捷控制窗和完整控制页共用同一模块状态，例如朗读播放状态、外观字号、设置开关必须同步。

动画规则：

- 打开快捷控制窗：快捷窗从底栏上方淡入并轻微上移，时长 160ms，正文不横向重载、不重新排版。
- 切换快捷模块：旧快捷窗内容交叉淡出，新内容交叉淡入，底栏和四按钮不移动，时长 120ms。
- 上拉展开完整页：底栏从当前高度插值到展开高度，时长 220ms，缓动 `cubic-bezier(0.2, 0.8, 0.2, 1)`。
- 下拉折叠：完整页回到快捷窗高度，时长 180ms；如果下拉距离超过 96px 或速度超过阈值则折叠，否则回弹。
- 拖动过程中只改变 sheet 的 `translateY` / 高度和遮罩透明度，不能触发页面路由重建。
- 完整页内部滚动优先；只有内容滚动到顶部后继续下拉，才折叠 sheet。

## 沉浸态运行胶囊（Immersive Runtime Capsule）

自动翻页（Auto Page）和朗读（Read Aloud）从快捷控制窗启动后，不能继续停留在完整控制层上，也不能把快捷控制窗悬在正文中间。启动后的沉浸页只允许显示最小运行胶囊，用于表达运行状态和提供暂停 / 继续 / 停止入口。

```text
Immersive Reading
├─ ReadingSurface                         正文排版层，尺寸和分页不因胶囊出现而变化
├─ ReaderHotZones                         上一页 / 打开控制层 / 下一页热区
├─ FooterProgressInfo                     页脚弱进度，可被运行胶囊临时降级或隐藏
└─ RuntimeCapsuleHost                     自动翻页 / 朗读运行胶囊宿主
```

胶囊类型：

| 胶囊（Capsule） | 触发来源（Entry） | 必须显示（Must Show） | 允许动作（Actions） |
|---|---|---|---|
| 自动翻页运行胶囊（Auto Page Capsule） | `auto-page` 快捷窗点击开始 | 自动翻页状态、速度或模式、暂停 / 继续、停止 | `pauseAutoPage`、`continueAutoPage`、`stopAutoPage`、`openAutoPageQuickWindow` |
| 朗读运行胶囊（TTS Capsule） | `tts` 快捷窗点击开始 | 朗读状态、当前句或章节范围、暂停 / 继续、停止 | `pauseReadAloud`、`continueReadAloud`、`stopReadAloud`、`openTTSQuickWindow` |

位置规则：

- 胶囊默认不放在正文中央，也不放在底部页脚正上方；默认锚定到正文安全空区的右上角或右下角。
- 优先位置为右上角：`top = safeAreaTop + 12px`，`right = readerMarginRight + 8px`，但不得覆盖章节标题、首段首行和顶部 2 行正文。
- 如果当前页顶部正文密度高，胶囊改为右下角：`bottom = footerHeight + 12px`，`right = readerMarginRight + 8px`，但不得覆盖页脚进度和最后 2 行正文。
- 如果右侧空间不足，胶囊进入边缘收起态，只显示 36-44px 的圆形状态按钮；点击后展开为完整胶囊，3 秒无操作后自动收起。
- 胶囊不得跨过正文左右内边距；不得压住翻页热区的中心打开控制层区域。
- 胶囊出现不触发正文重新分页，不改变 `ReadingSurface` 的宽高、字号、行距、段距、字距和段落分布。

避让算法：

1. 计算 `ReadingSurface` 内容安全矩形：排除状态栏、页脚、系统安全区和阅读页边距。
2. 计算当前页首段、末段和页脚进度的可见矩形。
3. 先尝试右上角胶囊；若与标题 / 首段首两行重叠超过 8px，则尝试右下角。
4. 若右下角与末段末两行或页脚重叠超过 8px，则进入边缘收起态。
5. 用户拖动胶囊后记录本页临时锚点；翻页后重新执行避让，不把上一页的遮挡位置强行沿用。

交互规则：

- 点击胶囊主按钮只做暂停 / 继续，不打开完整控制层。
- 点击胶囊的设置 / 更多区域才打开对应快捷控制窗，且打开后进入 `ReaderShell` 控制层态。
- 长按胶囊允许拖动位置，但只能停靠在四个边缘锚点：右上、右下、左上、左下。
- 点击正文中心仍然打开阅读控制层；如果运行胶囊已展开，先收起胶囊，再显示控制层。
- 系统返回优先级：展开胶囊 -> 收起胶囊；快捷窗 -> 关闭快捷窗；运行中沉浸页 -> 提示或停止当前运行功能；普通沉浸页 -> 返回来源。

显示密度规则：

- 展开胶囊最大宽度不超过正文安全宽度的 42%，高度不超过 44px。
- 圆形收起态为 36-44px，图标来自素材库，不能手绘 SVG。
- 胶囊文字最多一行，状态文案使用短句：`自动翻页中`、`已暂停`、`朗读中`。
- 朗读胶囊不得展示示例正文；只允许显示当前句序号或章节范围。
- 自动翻页胶囊不得展示完整模式说明；只允许显示速度或模式摘要。

## 入口分级（Entry Tiers）

| 级别（Tier） | 交互内容（Interactions） | 所属 Shell | 首次点击结果（First Click Result） | 是否进入完整页（Full Page） |
|---|---|---|---|---|
| 即时动作（Immediate Action） | 上一页、下一页、隐藏控制层、退出阅读、亮度拖动、章节进度拖动 | ReaderShell | 直接更新当前阅读运行态 | 否 |
| 快捷控制窗（Quick Window） | 目录、朗读、界面、设置、内容搜索、自动翻页、内容替换、换源 | ReaderShell | 在正文可用区打开短窗口，底栏和四按钮保持原位 | 可通过底栏小横条展开 |
| 完整控制页（Full Control Sheet） | 目录与书签、阅读外观、朗读、阅读设置、自动翻页、内容搜索、内容替换 | ReaderShell | 由底栏上拉展开，保留 ReadingSurface 和模块导航 | 本身就是完整页 |
| 流程页（Flow Page） | 换源完整流程 | FlowShell | 从换源快捷窗或上拉展开进入来源对照流程 | 不进入 SettingsShell |
| 设置页（Settings Page） | 书源管理、书源编辑、批量检测、App 设置 | SettingsShell | 进入设置栈 | 不属于阅读控制层 |

|---|---|---|---|
| 目录与书签（TOC and Bookmarks） | 当前章节附近目录；目录每行只显示章节名；书签 tab 可显示书签摘要和位置 | 完整目录、书签列表、目录搜索、章节更多操作 | 是 |
| 朗读（Read Aloud） | 播放/暂停图标按钮、上一句/下一句、语速、声音、范围、定时；不展示示例正文 | 朗读范围、声音管理、后台朗读、定时策略 | 是 |
| 阅读外观（Reading Appearance） | 4 个纯色主题色块、字号、行距、3 个字体、背景色和页边距入口，实时预览；主题色块内不放图标 | 主题编辑、字体管理、段距、字距、对齐方式、翻页动画、完整排版设置 | 是 |
| 阅读设置（Reading Settings） | 内部滚动列表：自动翻页、点击翻页方式、音量键翻页、翻页动画、横屏锁定、屏幕常亮、页脚进度、触摸反馈、自动缓存后续章节 | 分组设置、预设应用、恢复默认、更多阅读行为 | 是 |
| 内容搜索（Content Search） | 搜索输入、上一条/下一条、少量结果预览 | 全量结果、筛选范围、键盘避让、结果定位 | 是 |
| 自动翻页（Auto Page） | 速度、模式、开始/暂停 | 运行状态、退出路径、模式详情、停止策略 | 是 |
| 内容替换（Content Replacement） | 总开关、最近规则、新增入口、临时保存 | 规则编辑、测试结果、生效范围、持久化保存 | 是 |
| 换源（Source Switch） | 当前来源、候选来源两行列表、延迟和最新章节；默认按延迟排序 | FlowShell 来源对照，仅在需要完整来源流程时展开；仍保留 ReaderContext | 是，但完整页归 FlowShell |

## 快捷窗与完整页内容边界（Quick and Full Content Boundary）

| 功能（Feature） | 快捷控制窗只展示（Quick Window Shows） | 完整控制页才展示（Full Page Only） |
|---|---|---|
| 目录与书签（TOC and Bookmarks） | 目录 / 书签切换；当前章节附近 6-8 行；目录每章只显示章节名；书签只显示标题和位置。 | 全量目录滚动、目录搜索、书签批量管理、章节更多操作、缓存状态、定位当前章节。 |
| 朗读（Read Aloud） | 上一句、播放 / 暂停图标、下一句；语速、音色、范围、定时四个常用参数。 | 声音管理、朗读引擎、后台朗读、定时策略、断句策略、朗读高亮、更多高级配置。 |
| 阅读外观（Reading Appearance） | 4 个纯色主题色块；字号、行距；字体三选一；背景色和页边距入口；即时影响正文。 | 自定义主题、导入字体、段距、字距、对齐方式、翻页动画、背景纹理、保存 / 重置预设。 |
| 阅读设置（Reading Settings） | 内部滚动列表展示常用行为项：自动翻页、点击翻页方式、音量键翻页、翻页动画、横屏锁定、屏幕常亮、页脚进度、触摸反馈、自动缓存后续章节。 | 手势分区、状态栏显示细则、底部进度格式、屏幕超时、按键策略、更多阅读行为和恢复默认。 |
| 内容搜索（Content Search） | 输入框、清除、上一条 / 下一条、前 3-5 条结果。 | 全量结果列表、结果筛选、键盘避让、章节范围选择、搜索历史。 |
| 自动翻页（Auto Page） | 速度、模式、开始 / 暂停。 | 模式说明、边缘触发、停止条件、运行状态、退出策略。 |
| 内容替换（Content Replacement） | 总开关、最近 2-3 条规则；每条显示规则名、配置摘要、执行范围摘要、启用状态和进入编辑入口。 | 完整规则编辑、匹配表达、替换文本、执行范围细则、测试替换、规则排序、删除、持久化保存。 |
| 换源（Source Switch） | 当前来源、候选来源；每行两行：书源名 + 延迟 / 最新章节；无筛选排序。 | 来源对照、失败原因、重试检测、切换确认和返回阅读上下文；书源编辑仍归 SettingsShell。 |

## 内容替换信息分层（Content Replacement Information Split）

内容替换（Content Replacement）的信息分为三类：规则名（Rule Name）、规则具体配置（Rule Configuration）和执行范围（Execution Scope）。快捷控制窗负责让用户快速判断“当前有哪些规则、是否启用、作用到哪里”，完整设置页负责编辑、测试和持久化。

| 信息（Information） | 快捷控制窗展示（Quick Window） | 完整设置页展示（Full Page） |
|---|---|---|
| 规则名（Rule Name） | 必须展示。单行，最多 10-14 个中文字符，超出省略；用于快速识别规则。 | 必须展示。可编辑，支持较长名称，两行或输入框内横向滚动。 |
| 规则具体配置（Rule Configuration） | 只展示摘要：`匹配词 -> 替换词` 或 `正则 -> 替换`，最多一行；复杂正则用 `正则规则` + 命中数摘要，不直接铺满。 | 展示完整配置：匹配模式、原文 / 正则、替换文本、大小写 / 全词 / 多行等选项、测试输入和测试结果。 |
| 执行范围（Execution Scope） | 必须展示短标签：`当前章节`、`当前书`、`全部阅读`、`本次临时`；最多一个主范围 + 一个状态标签。 | 展示完整范围：当前章节 / 当前书 / 指定章节 / 全局、是否临时生效、是否保存到规则库、排除章节或来源条件。 |
| 启用状态（Enabled State） | 必须展示开关或状态点，可即时切换。 | 必须展示，并可批量启停、排序、删除。 |
| 命中反馈（Match Feedback） | 可展示短文本：`命中 3 处`、`未命中`、`格式错误`。 | 展示完整测试反馈、错误位置、替换前后预览。 |

快捷控制窗布局规则：

- 顶部显示总开关和全局状态：`启用内容替换`、`当前书 2 条`、`命中 3 处`。
- 规则列表最多首屏展示 2-3 条；超过后使用内部滚动，不扩大快捷窗，不压住底部控制层。
- 每条规则使用三段式紧凑行：第一行 `规则名 + 启用状态`；第二行 `配置摘要`；第三行用小标签显示 `执行范围` 和 `命中反馈`。
- 配置摘要不能超过一行；复杂表达不在快捷窗展开，点击规则进入完整设置页。
- 新增规则入口固定在列表后或底部操作区，文案保持 `新增规则`，不把编辑表单塞进快捷窗。
- 快捷窗内不展示大段预览正文，避免变成小号完整页。

完整设置页布局规则：

- 完整设置页显示规则列表、规则编辑表单、测试替换、保存、删除和排序。
- 规则编辑页必须同时有：规则名输入、匹配模式、替换前、替换后、执行范围、测试结果、保存。
- 执行范围的详细配置只放在完整页，不放在快捷窗中展开。
- 保存和临时关闭必须分离：临时关闭只影响当前阅读会话，保存写入规则库。

## 换源流程（Source Switch Flow）

换源必须按流程页处理，不按“快捷窗口 -> 完整控制页”处理。

```text
ReaderShell.ReaderTopBar
└─ 换源（Source Change）
   ├─ ReaderShell.SourceQuickWindow：正文可用区域内的换源快捷窗口和候选来源列表
   └─ FlowShell.SourceSwitch：完整来源对照和切换流程
      ├─ StepRegion：延续进入前的阅读控制层，包含顶部栏、正文、底部控制面板、亮度栏和模块导航
      └─ ComparisonRegion：候选来源对照、失败原因、重试检测和切换确认
```

换源交互规则：

- 点击换源按钮后先打开 `ReaderShell.SourceQuickWindow`；只有需要完整来源对照、失败原因或切换确认时，才进入 `FlowShell.SourceSwitch`。
- 手机端仍使用与其他页面一致的 `390 x 844` 应用画布；大屏或横屏设备可以把 FlowShell 展开为横向对照布局。
- 换源窗口必须保持在正文可用区域内，不与顶部阅读栏、底部控制面板、亮度栏或四模块导航重叠。
- 换源窗口样式沿用阅读控制层面板，不使用暗色遮罩、独立模态阴影或蓝色高亮聚焦。
- 候选来源行只显示两行：第一行书源名和延迟/状态，第二行最新章节。
- 候选来源默认按延迟从小到大排序；离线、超时等无延迟状态排在有延迟来源之后。
- 不展示筛选、检测条、结果保存面板或底部状态摘要。
- 点击候选来源只更新当前候选；是否点击即切源由真实业务实现决定，设计稿不额外展示确认结果区。
- 切源后返回 `ReaderShell`，并保留 `ReaderContext.bookId / chapterId / progress / appearance`。
- 关闭或返回不更新来源，只恢复进入换源前的阅读控制层或沉浸阅读。
- 书源管理、书源编辑、批量检测属于 `SettingsShell.SourceManagement`，不是换源窗口的一部分。
