## 8. P1 可后续修复问题

1. 左侧亮度条底部图标已从屏幕图标替换为箭头类图标，但 `keyboard_tab_rtl` 的方向和圆润箭头语义建议在后续 handoff 前再确认或微调为明确指向屏幕右侧的圆润箭头。
2. 快捷方式区当前视觉为横向 4 项分布，但 HTML 采用 `justify-between`，后续 Android handoff 可补充明确的等宽约束，避免实现侧出现非四等分。

## 9. 是否允许进入后续弹层派生

允许。

原因：目标 screen 已满足控制层参照模板的核心结构要求，包括顶部两行新结构、左侧悬浮亮度条、底部三层控制面板、正确的快捷入口和底栏入口、底栏无选中态，以及主要区域边界标注。未发现 P0 阻塞问题；仅存在图标方向/等宽约束这类 P1 级后续修复项。
