## 10. P0 剩余问题

1. 4 个底栏派生 screen 的快捷方式栏视觉规格不一致：目录为 plain grid，朗读为 `flex justify-between` + `opacity-60` 小虚线按钮，界面/设置为另一套虚线 grid，小图标、字号、按钮盒、间距和高度均不统一。
2. 4 个底栏派生 screen 的底栏功能区图标和入口尺寸不一致：目录/朗读与界面/设置使用不同图标语义，当前项强调规则也不一致，无法作为统一 Android Compose handoff 基准。
3. 4 个底栏派生 screen 的底部控制面板和展开面板尺寸体系不一致：目录使用 fixed bottom nav，朗读使用独立 Bottom Panel Group，界面使用 integrated 3-layer，设置将控制层嵌入 `h-[60vh]` panel，视觉大小和容器关系不统一。
4. 阅读页 - 朗读底栏功能展开 (最终修正版) 的朗读信息 UI 不合理：朗读状态、章节时间、播放控制、slider、定时关闭和音色选择被压缩在同一紧凑块，播放区 `skip_previous` 与章节跳转层语义混淆，主次操作层级不清。
5. 阅读页 - 朗读底栏功能展开 (最终修正版) 的「定时关闭 / 朗读音色」排版不合理：两张操作卡片并排后宽度不足，信息过挤；「定时关闭」有 chevron，「朗读音色」没有同等尾部 affordance，两个同级操作的交互暗示不一致。

## 11. P1 剩余问题

1. 阅读页 - 目录底栏功能展开 (最终修正版)：未看到明确可见的右侧快速滑动条或索引条。
2. 阅读页 - 设置底栏功能展开 (最终修正版)：未看到明确的「自动缓存本章或后续章节」设置项。
3. 阅读页 - 设置底栏功能展开 (最终修正版)：未看到明确命名的「阅读页手势设置」分组或入口，当前更接近翻页方式设置。
4. 同一章节的章节进度条状态不一致：目录 / 朗读约为 25%，界面 / 设置约为 45%，作为同一阅读页派生状态应保持一致。
5. 边界标注风格过重且不统一：目录使用 handoff label，朗读同时有 Playback Overlay / Control Buttons / Bottom Panel Group 标签，界面和设置更多依赖虚线边界，视觉干扰程度不同。

## 12. 是否允许进入下一阶段阅读页整体 handoff 审计

不允许。

原因：虽然上一轮残留的文本与正文类 P0 多数已经修复，但 4 个底栏派生 screen 的快捷方式栏、底栏图标体系、控制面板尺寸和展开区域容器关系没有统一，且朗读信息 UI 存在明显不合理，特别是「定时关闭 / 朗读音色」两项排版和交互暗示不一致。当前状态不适合作为统一的 Reader for Android 阅读页 handoff 基准，必须先统一底部控制层组件规格并重做朗读展开区的信息结构。
